! Context-Free Grammars

<img src="assets/context_free_grammars.svg" alt="Context-Free Grammars" width="800">

Imagine you are assembling a sentence from a set of recipe cards. One card says "a sentence is a subject followed by a predicate." Another says "a subject is either a name or a pronoun." Each card tells you how to expand one ingredient into smaller ingredients, until you reach actual words. You never need to peek at what came before or after the ingredient you are expanding — each card works in isolation, regardless of ''context''. That is the essence of a ''context-free grammar'' (CFG): a set of rewriting rules where every rule replaces exactly one symbol, independent of its surroundings. CFGs sit at the heart of how computers understand the structure of programming languages, how linguists model sentence structure, and how parsers in NLP break text into meaningful hierarchies.

!! The Core Insight

A context-free grammar generates strings by repeatedly replacing a single ''non-terminal'' symbol with a sequence of symbols (terminals and non-terminals), regardless of what surrounds it. The key constraint is //context-freedom//: the rule $A \to \alpha$ applies wherever $A$ appears, no matter what is to its left or right. This single restriction makes CFGs powerful enough to describe nested, recursive structures (like matched parentheses or nested if-else blocks) while remaining tractable enough to parse efficiently. Every CFG has an equivalent ''pushdown automaton'' — a finite-state machine augmented with a stack — which is exactly the extra memory needed to track nesting depth. This correspondence between grammars and machines is one of the foundational results of theoretical computer science.

!! The Chomsky Hierarchy and Where CFGs Fit

Noam Chomsky's 1956 classification arranges formal grammars into four types by the form of their production rules:

''Type 3 — Regular grammars.'' Rules have the form $A \to aB$ or $A \to a$. These generate ''regular languages'', recognized by finite automata. They cannot handle nesting: the language $\{a^n b^n \mid n \geq 1\}$ (equal numbers of $a$'s then $b$'s) is beyond their reach.

''Type 2 — Context-free grammars.'' Rules have the form $A \to \alpha$, where $A$ is a single non-terminal and $\alpha$ is any string of terminals and non-terminals. These generate ''context-free languages'' (CFLs), recognized by pushdown automata. The language $\{a^n b^n\}$ is context-free, captured by $S \to aSb \mid ab$.

''Type 1 — Context-sensitive grammars.'' Rules may condition on surrounding symbols: $\alpha A \beta \to \alpha \gamma \beta$. These generate context-sensitive languages, recognized by linear-bounded automata.

''Type 0 — Unrestricted grammars.'' Any rewriting rule is allowed. These generate recursively enumerable languages, recognized by Turing machines.

Each level strictly contains the one below it. CFGs occupy the sweet spot: expressive enough for most programming language syntax and many natural language constructions, yet efficient enough for polynomial-time parsing.

!! What the Math Is Actually Doing

A context-free grammar is a 4-tuple $G = (V, \Sigma, R, S)$ where:

- $V$ is a finite set of ''non-terminal'' symbols (variables)
- $\Sigma$ is a finite set of ''terminal'' symbols (the alphabet), with $V \cap \Sigma = \emptyset$
- $R \subseteq V \times (V \cup \Sigma)^*$ is a finite set of ''production rules''
- $S \in V$ is the ''start symbol''

A rule $(A, \alpha) \in R$ is written $A \to \alpha$. A ''derivation'' applies rules to rewrite the start symbol into a string of terminals:

$$S \Rightarrow \alpha_1 \Rightarrow \alpha_2 \Rightarrow \cdots \Rightarrow w \in \Sigma^*$$

The ''language'' generated by $G$ is:

$$L(G) = \{ w \in \Sigma^* \mid S \Rightarrow^* w \}$$

''Chomsky Normal Form'' (CNF) restricts all rules to either $A \to BC$ (two non-terminals) or $A \to a$ (one terminal). Every CFG can be converted to CNF, and this form enables the ''CYK algorithm'' for parsing in $O(n^3 |G|)$ time, where $n$ is the input length and $|G|$ is the grammar size.

The CYK algorithm fills a triangular table. For substring $w_{i..j}$, it checks whether any rule $A \to BC$ has $B$ generating $w_{i..k}$ and $C$ generating $w_{k+1..j}$ for some split point $k$:

$$T[i,j] = \{ A \mid \exists\, A \to BC,\; k: B \in T[i,k],\; C \in T[k+1,j] \}$$

The string $w$ is in $L(G)$ if and only if $S \in T[1,n]$.

''Ambiguity'' arises when a string has more than one parse tree (leftmost derivation). The sentence "I saw the man with the telescope" can attach "with the telescope" to the verb or the noun — two distinct parse trees from the same grammar. A grammar is ''ambiguous'' if any string in its language has multiple parse trees. Some context-free languages are //inherently ambiguous//: every grammar generating them is ambiguous.

!! CFGs in Practice: Parsing and NLP

''Programming language compilers'' rely on CFGs. The syntax of nearly every programming language (C, Python, Java) is specified by a CFG or a close variant. Parser generators like Yacc, Bison, and ANTLR take a CFG specification and produce efficient parsers. Most practical parsers use restricted CFG subclasses — LL(k) or LR(k) grammars — that enable linear-time $O(n)$ parsing by looking ahead $k$ tokens.

''Natural language processing'' historically used ''probabilistic context-free grammars'' (PCFGs), which assign probabilities to each production rule. Given a sentence, a PCFG parser finds the most likely parse tree using a probabilistic variant of CYK. PCFGs powered early syntactic parsers in NLP and remain a foundation for understanding constituency parsing, even as neural parsers (using [[Transformers|Transformers]]) have largely supplanted them in practice.

''Abstract syntax trees'' (ASTs) — the structured representations that compilers and code analysis tools manipulate — are parse trees pruned of syntactic sugar. The grammar defines what trees are possible; the parser builds the right one.

''Formal verification and security.'' Input validation in web browsers, XML/JSON parsers, and protocol handlers all rely on CFG-based parsing. The grammar defines what inputs are structurally valid, and the parser rejects anything that does not conform.

!! When Context-Free Grammars Break Down

''1. Cross-serial dependencies in natural language.'' Natural languages contain constructions that CFGs cannot capture. Swiss German and Dutch have cross-serial verb constructions whose structure matches $\{a^n b^n c^n\}$ — a language provably not context-free (by the ''pumping lemma'' for CFLs). This motivated mildly context-sensitive formalisms like tree-adjoining grammars (TAGs) and combinatory categorial grammar (CCG) in computational linguistics.

''2. Ambiguity without resolution.'' CFGs generate structure but provide no mechanism to choose among multiple valid parse trees. In NLP, "Time flies like an arrow" has at least three parses. Pure CFGs give all of them equal standing. PCFGs and neural parsers address this by scoring trees, but the grammar alone is blind to semantic plausibility. In programming languages, dangling-else ambiguity and operator precedence require extra disambiguation rules beyond the CFG itself.

''3. Semantic constraints beyond syntax.'' CFGs describe form, not meaning. A CFG for a programming language will happily parse a program that uses an undeclared variable or passes the wrong number of arguments to a function — these are //semantic// errors invisible to the grammar. Similarly, natural language sentences like "colorless green ideas sleep furiously" are syntactically valid under a CFG but semantically nonsensical. Ensuring well-formedness beyond structure requires additional machinery: type systems, attribute grammars, or learned semantic representations.

!! Technical Details

''Pumping lemma for CFLs:'' If $L$ is context-free, then for sufficiently long strings $s \in L$, $s = uvxyz$ where $|vy| \geq 1$, $|vxy| \leq p$, and $uv^i xy^i z \in L$ for all $i \geq 0$. Used to prove languages like $\{a^n b^n c^n\}$ and $\{ww \mid w \in \Sigma^*\}$ are not context-free.

''Closure properties:'' CFLs are closed under union, concatenation, Kleene star, and intersection with regular languages. They are //not// closed under intersection or complement — a critical difference from regular languages.

''Earley parser (1970):'' Parses any CFG in $O(n^3)$ worst case, $O(n^2)$ for unambiguous grammars, and $O(n)$ for many practical grammars (including all LR(k) grammars). More general than LL/LR parsers but with higher constant factors.

''Chomsky-Schutzenberger theorem:'' Every CFL can be expressed as a homomorphic image of the intersection of a Dyck language (matched parentheses) with a regular language. This reveals that nesting (the Dyck language) is the essential feature that separates CFLs from regular languages.

''Equivalence with pushdown automata (PDA):'' A language is context-free if and only if some PDA recognizes it. The PDA's stack provides exactly the memory to track recursive nesting that a finite automaton lacks.
