! Convolutional Neural Networks
<img src="assets/convolutional_neural_networks.svg" style="width:100%;max-width:800px;display:block;margin:0.5em auto 1em;">

Imagine you are looking for a friend's face in a crowd. You do not examine the entire scene as a single flattened list of pixels. Instead, your visual system scans small local regions — an eyebrow here, the curve of a jaw there — and assembles these local features into a coherent whole. A ''convolutional neural network'' (CNN) works exactly this way: small learned filters slide across an image, each one detecting a specific local pattern, and successive layers compose those patterns into increasingly abstract representations — edges become textures, textures become object parts, parts become whole objects.

!! The Core Insight

The fundamental idea behind CNNs is ''parameter sharing'' combined with ''local connectivity''. Rather than connecting every input pixel to every neuron (as in a fully connected layer), a CNN uses a small filter — say 3×3 — that is applied at every spatial position in the image. This single set of weights is shared across the entire input, which means the network can detect a feature (like a vertical edge) regardless of where it appears. This property is called ''translation equivariance'': if the input shifts, the output shifts by the same amount but retains the same pattern of activations. The result is dramatically fewer parameters, built-in spatial structure, and the ability to scale to high-resolution images that would be intractable for fully connected architectures. !! Convolutions, Pooling, and Feature Hierarchies

A CNN is built from three types of building blocks stacked in sequence:

''Convolutional layers'' apply a bank of learned filters to the input. Each filter produces one ''feature map'' — a 2D grid of activations indicating where and how strongly that filter's pattern was detected. With $K$ filters of size $f \times f$ applied to an input with $C$ channels, the layer learns $K \times C \times f \times f$ weights plus $K$ biases. A non-linearity (almost always ReLU) follows each convolution.

''Pooling layers'' downsample the spatial dimensions. ''Max pooling'' takes the maximum value in each local window (typically 2×2), reducing width and height by half. This provides a small degree of translation invariance and reduces computation in later layers. Some modern architectures replace pooling with strided convolutions.

''Fully connected layers'' appear at the end of the network. After convolution and pooling have reduced the image to a compact feature vector, one or more dense layers map this vector to class scores or other outputs.

The ''feature hierarchy'' is the key architectural principle: early layers detect low-level features (edges, color gradients), middle layers detect mid-level features (textures, corners, simple shapes), and deep layers detect high-level semantic features (eyes, wheels, buildings). This hierarchical composition is what gives CNNs their extraordinary representational power. See also: <<link "Pooling" "Pooling">><</link>>, <<link "Feature Maps" "Feature Maps">><</link>>.

!! What the Math Is Actually Doing

The discrete 2D convolution for a single filter applied to a single input channel is:

$$(I * K)(i, j) = \sum_{m=0}^{f-1} \sum_{n=0}^{f-1} I(i+m,\, j+n) \cdot K(m, n)$$

where $I$ is the input, $K$ is the filter of size $f \times f$, and the output is computed at every valid spatial position $(i, j)$. For multi-channel inputs with $C$ channels and $K$ output filters, the full operation is:

$$Y_k(i, j) = b_k + \sum_{c=1}^{C} \sum_{m=0}^{f-1} \sum_{n=0}^{f-1} I_c(i+m,\, j+n) \cdot W_{k,c}(m, n)$$

The output spatial size is determined by the ''stride'' $s$ (step size between filter positions) and ''padding'' $p$ (zeros added around the border):

$$\text{output size} = \left\lfloor \frac{n_{\text{in}} + 2p - f}{s} \right\rfloor + 1$$

With stride 1 and padding $p = \lfloor f/2 \rfloor$ (called ''same'' padding), the spatial dimensions are preserved. With stride 2, they are halved. During <<link "Backpropagation" "Backpropagation">><</link>>, the gradient with respect to the filter weights is itself a convolution of the input with the upstream gradient, which is why CNNs are efficient to train — the same computational primitive is reused in both the forward and backward passes.

!! CNN Architectures Through History

''LeNet-5'' (LeCun et al., 1998) applied two convolutional layers followed by fully connected layers to handwritten digit recognition — the architecture that proved CNNs worked in practice. ''AlexNet'' (Krizhevsky, Sutskever, Hinton, 2012) scaled this pattern to ImageNet with five convolutional layers, ReLU activations, dropout, and GPU training, winning the ImageNet Large Scale Visual Recognition Challenge by a decisive margin and igniting the deep learning revolution. ''VGGNet'' (Simonyan and Zisserman, 2014) showed that depth matters: 16–19 layers of uniform 3×3 convolutions outperformed shallower networks with larger filters. ''GoogLeNet/Inception'' (Szegedy et al., 2015) introduced parallel filter sizes within a single layer (inception modules), achieving efficiency through 1×1 convolutions that reduce channel dimensionality. ''ResNets'' (He et al., 2015) solved the degradation problem in very deep networks with skip connections, enabling architectures with 50, 101, or even 152 layers: <<link "ResNets" "ResNets">><</link>>. More recent work includes EfficientNet (compound scaling of depth, width, and resolution), ConvNeXt (modernizing pure CNN designs with Transformer-era training recipes), and the ongoing competition between CNNs and Vision Transformers.

!! When CNNs Break Down

''Spatial invariance is limited:'' CNNs achieve approximate translation invariance through pooling, but they are not inherently invariant to rotation, scale changes, or viewpoint shifts. A CNN trained on upright faces may fail on rotated faces unless augmentation or equivariant architectures are used. This is a fundamental limitation of the standard convolutional structure.

''Global context is expensive:'' Because each filter sees only a small local receptive field, capturing long-range dependencies requires either very deep stacks (expanding the effective receptive field layer by layer) or architectural additions like self-attention. For tasks requiring global reasoning — such as understanding scene layout or long-range pixel relationships — pure CNNs struggle compared to Transformers, which attend to all positions simultaneously.

''Small or noisy datasets cause overfitting:'' CNNs have millions of parameters. Without sufficient training data, regularization, or transfer learning, they memorize training examples rather than learning generalizable features. Data augmentation, dropout, weight decay, and pre-trained initialization are essential when data is limited.

!! Technical Details

''Parameter count examples:'' A 3×3 conv layer with 64 input and 128 output channels has $128 \times 64 \times 3 \times 3 + 128 = 73{,}856$ parameters. AlexNet: ~60M parameters. VGG-16: ~138M. ResNet-50: ~25.6M (skip connections allow going deeper with fewer parameters per layer). ''Receptive field:'' for a stack of $L$ layers each with $3 \times 3$ filters and stride 1, the theoretical receptive field is $(2L+1) \times (2L+1)$. ''1×1 convolutions:'' these perform channel-wise linear combinations without spatial mixing and are used in bottleneck blocks (ResNets), Inception modules, and channel attention (SE-Nets). ''Depthwise separable convolutions:'' factorize a standard convolution into a depthwise spatial convolution (one filter per channel) followed by a 1×1 pointwise convolution, reducing computation by roughly a factor of $K$ (number of output channels). Used in MobileNets and EfficientNets. ''Dilated (atrous) convolutions:'' insert gaps between filter elements to expand the receptive field without increasing parameters or reducing resolution; widely used in semantic segmentation (DeepLab). ''Batch normalization:'' normalizing activations per mini-batch after each conv layer stabilizes training and enables higher learning rates — nearly universal in modern CNNs. ''Typical training:'' SGD with momentum 0.9, learning rate 0.1 with cosine or step decay, weight decay $10^{-4}$, extensive data augmentation (random crop, horizontal flip, color jitter). Training ImageNet-scale models takes hours on multi-GPU setups.
