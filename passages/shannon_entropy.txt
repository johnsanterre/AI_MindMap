! Shannon Entropy

<img src="assets/shannon_entropy.svg" alt="Shannon entropy curve and cross-entropy decomposition diagram" style="width:100%;max-width:620px;display:block;margin:16px auto 24px auto;">

A coin flip is the purest unit of surprise. If you know the coin is biased 99% heads, each toss tells you almost nothing new — the outcome is nearly predetermined. If the coin is perfectly fair, each flip is maximally informative — you genuinely have no idea what will happen next. ''Shannon entropy'' quantifies this intuition mathematically, and it turns out to govern everything from optimal data compression to how well neural networks learn. It is the fundamental currency of information.

!! The Core Insight

The entropy of a random variable $X$ is defined as:

$$H(X) = -\sum_{x} p(x) \log p(x)$$

Three key facts follow immediately from this formula:

''Maximum entropy at the uniform distribution.'' When all outcomes are equally likely, you are maximally uncertain — entropy is highest. For a fair coin, $H = 1$ bit. For a fair six-sided die, $H = \log_2 6 \approx 2.58$ bits.

''Zero entropy when the outcome is certain.'' If $p(x) = 1$ for some $x$, then $H(X) = 0$. There is no surprise in a guaranteed outcome. (By convention, $0 \log 0 = 0$.)

''Entropy as average surprise.'' The term $-\log p(x)$ is the ''surprisal'' of outcome $x$ — rare events are surprising, common events are not. Entropy is the //expected surprisal// across the distribution. Equivalently, it is the minimum average number of bits needed to encode a message drawn from that distribution.

!! Entropy, KL Divergence, and Cross-Entropy

These three quantities form a tightly related family that appears throughout machine learning.

''KL divergence'' $D_{KL}(P \| Q)$ measures how different distribution $Q$ is from a reference distribution $P$:

$$D_{KL}(P \| Q) = \sum_{x} p(x) \log \frac{p(x)}{q(x)}$$

It is //asymmetric// — $D_{KL}(P \| Q) \neq D_{KL}(Q \| P)$ in general — and always non-negative. It equals zero only when $P = Q$. Think of it as the extra bits you waste by using code $Q$ when the true distribution is $P$.

''Cross-entropy'' $H(P, Q)$ is the average code length when you encode data from $P$ using a code optimized for $Q$:

$$H(P, Q) = -\sum_{x} p(x) \log q(x) = H(P) + D_{KL}(P \| Q)$$

This decomposition is crucial: cross-entropy equals true entropy plus the KL penalty for using the wrong distribution. Since $H(P)$ is fixed (it is a property of the data), minimizing cross-entropy loss is //identical// to minimizing $D_{KL}(P \| Q)$ — driving the model $Q$ toward the data distribution $P$.

''Mutual information'' $I(X; Y)$ measures how much knowing $Y$ reduces uncertainty about $X$:

$$I(X; Y) = H(X) - H(X \mid Y) = D_{KL}(P(X,Y) \| P(X)P(Y))$$

It is symmetric and non-negative. Zero mutual information means $X$ and $Y$ are independent.

!! What the Math Is Actually Doing

In ''nats'' (using natural logarithm), $H(X) = -\mathbb{E}[\ln p(X)]$. In ''bits'' (using $\log_2$), entropy counts the minimum average binary questions needed to identify an outcome.

''Shannon's source coding theorem'' (1948) is the foundational result: no lossless compression scheme can achieve an average code length below $H(X)$ bits per symbol, and Huffman coding comes arbitrarily close. Entropy is not just a measure — it is an achievable bound.

For ''continuous distributions'', differential entropy replaces the sum with an integral:

$$h(X) = -\int p(x) \ln p(x) \, dx$$

Unlike discrete entropy, $h(X)$ can be negative (e.g., a very narrow Gaussian has low differential entropy that can go below zero). This makes raw comparisons between discrete and continuous quantities treacherous.

The ''maximum entropy principle'' states: among all distributions consistent with known constraints (e.g., fixed mean and variance), the distribution with highest entropy is the //least biased// choice. This recovers familiar distributions — Gaussian maximizes entropy for fixed mean and variance; exponential maximizes it for fixed mean on $[0, \infty)$.

$$H(X) = -\mathbb{E}[\log p(X)]$$

!! Entropy in ML

''Cross-entropy loss for classification'' is the workhorse of supervised learning. Given true labels (one-hot distribution $P$) and model softmax outputs ($Q$), minimizing $H(P, Q)$ is exactly minimizing $D_{KL}(P \| Q)$. See <<link "cross entropy loss" "cross entropy loss">><</link>>.

''Entropy regularization in RL'' encourages exploration by adding $\alpha H(\pi(\cdot \mid s))$ to the reward. The Soft Actor-Critic (SAC) algorithm optimizes a maximum-entropy objective, producing policies that are robust and exploratory. See <<link "Reinforcement Learning" "Reinforcement Learning">><</link>>.

''Mutual information maximization'' in representation learning (InfoNCE, MINE) trains encoders to produce representations that maximally preserve information about inputs or labels.

''Decision trees'' select splits by ''information gain'' — the reduction in entropy after observing a feature. A split that perfectly separates classes reduces entropy to zero; an uninformative split leaves it unchanged.

''VAE ELBO'': the Evidence Lower BOund decomposes as reconstruction loss minus a KL term $D_{KL}(q(z|x) \| p(z))$, which regularizes the latent space toward the prior. See <<link "Variational Inference" "Variational Inference">><</link>>.

''Bits-per-dimension'' (BPD) is the standard metric for evaluating generative models on images: how many bits does the model need per pixel? Lower is better; it is directly derived from the cross-entropy between data and model.

!! When Entropy Analysis Breaks Down

''Differential entropy can be negative.'' Unlike discrete entropy (which is always $\geq 0$), differential entropy for continuous distributions has no lower bound and can be negative. A Gaussian with very small variance has $h(X) = \frac{1}{2} \ln(2\pi e \sigma^2)$, which goes to $-\infty$ as $\sigma \to 0$. Comparing discrete and continuous entropies directly — or treating negative differential entropy as "less information than nothing" — is a conceptual error with no clean fix.

''KL divergence is not a distance.'' It is asymmetric: $D_{KL}(P \| Q) \neq D_{KL}(Q \| P)$. It can be infinite when $Q(x) = 0$ for any $x$ where $P(x) > 0$ (the model assigns zero probability to a real event). It violates the triangle inequality. Most consequentially, //which direction// you minimize changes behavior qualitatively: minimizing $D_{KL}(P \| Q)$ (reverse KL, as in VI) produces //mode-seeking// behavior; minimizing $D_{KL}(Q \| P)$ (forward KL, as in MLE) produces //mean-seeking// behavior. See <<link "KL Divergence" "KL Divergence">><</link>>.

''Entropy assumes IID samples.'' Shannon's source coding theorem requires that symbols are drawn independently from the same distribution. For correlated sequences — time series, natural language, audio — the per-symbol entropy overestimates the true information content. The correct quantity is the ''entropy rate'' $\bar{H} = \lim_{n\to\infty} \frac{1}{n} H(X_1, \ldots, X_n)$, which accounts for dependencies. Applying IID entropy to correlated data gives overly pessimistic compression bounds and misleading information estimates.

!! Technical Details

''Foundational paper:'' Shannon, C.E. (1948). "A Mathematical Theory of Communication." //Bell System Technical Journal//, 27(3), 379–423. This paper simultaneously defined entropy, channel capacity, and the source and channel coding theorems.

''Huffman coding'' (1952) constructs an optimal prefix-free code achieving the entropy bound to within 1 bit per symbol. ''Arithmetic coding'' achieves the bound asymptotically with no 1-bit overhead.

''Channel capacity:'' $C = \max_{p(x)} I(X; Y)$ is the maximum mutual information over all input distributions. Shannon's channel coding theorem guarantees reliable communication at any rate below $C$.

''Data processing inequality:'' If $X \to Y \to Z$ forms a Markov chain, then $I(X; Y) \geq I(X; Z)$. Processing can only reduce or preserve information, never increase it. This is the theoretical basis for why deep features can be no more informative than the raw input.
