! Message Passing Neural Networks

<img src="assets/message_passing_nn.svg" alt="Message Passing Neural Networks: gather messages from neighbors, aggregate, update node representations" style="width:100%;max-width:620px;display:block;margin:16px auto 24px auto;">

Imagine a neighborhood meeting where every household sends a note to each of its neighbors describing what it knows. Each household reads all the notes it received, summarizes them, and then updates its own understanding of the world. After several rounds, every household knows something about the entire neighborhood — even houses that never directly communicated. ''Message Passing Neural Networks'' (MPNNs) formalize exactly this process on graphs: nodes exchange ''messages'' with their neighbors, aggregate incoming information, and update their own representations. This framework, introduced by Gilmer et al. (2017), unifies virtually all spatial <<link "Graph Neural Networks" "Graph Neural Networks">><</link>> — including <<link "Graph Convolutional Networks" "Graph Convolutional Networks">><</link>>, GraphSAGE, GATs, and GINs — under a single mathematical umbrella.

!! The Core Insight

Every MPNN layer performs three operations for each node $v$ in the graph:

1. ''Message computation:'' For each neighbor $u \in \mathcal{N}(v)$, compute a message $m_{u \to v}$ that depends on the sender's features $h_u$, the receiver's features $h_v$, and optionally the edge features $e_{uv}$.

2. ''Aggregation:'' Combine all incoming messages into a single fixed-size summary $M_v$ using a permutation-invariant function — sum, mean, or max — because the neighbors have no natural ordering.

3. ''Update:'' Combine the aggregated message $M_v$ with the node's current state $h_v$ to produce a new representation $h_v'$.

After $K$ layers of message passing, each node's representation encodes information from its $K$-hop neighborhood. This is the graph analog of expanding a receptive field in a CNN: each additional layer lets information travel one more hop across the graph.

!! The Message Passing Framework

Gilmer et al. showed that many existing GNN architectures are special cases of a single framework. The differences lie in how each architecture defines its message function, aggregation, and update.

''GCN'' (Kipf & Welling, 2017): The message is simply the neighbor's features scaled by degree — $m_{u \to v} = \frac{1}{\sqrt{d_u d_v}} W h_u$. Aggregation is summation. This corresponds to a first-order Chebyshev approximation of spectral graph convolution, connecting spatial message passing to the <<link "Graph Laplacian" "Graph Laplacian">><</link>>.

''GraphSAGE'' (Hamilton et al., 2017): Samples a fixed number of neighbors and aggregates via mean, LSTM, or max pooling. The update concatenates the aggregated neighborhood with the node's own features, then applies a linear layer. Designed for inductive learning on large, evolving graphs.

''GAT'' (Velickovic et al., 2018): Introduces learned <<link "Attention" "Attention">><</link>> weights between neighbors. The message from $u$ to $v$ is weighted by an attention coefficient $\alpha_{uv}$ computed via a shared attention mechanism. Different neighbors contribute differently — the graph equivalent of self-attention in Transformers.

''GIN'' (Xu et al., 2019): Proves that sum aggregation with an injective update function is maximally expressive among MPNNs, matching the power of the Weisfeiler-Lehman graph isomorphism test. Mean and max aggregation lose information about multiset structure.

!! What the Math Is Actually Doing

The general MPNN layer is:

$$m_{u \to v}^{(k)} = \text{MSG}^{(k)}\!\left(h_u^{(k-1)},\, h_v^{(k-1)},\, e_{uv}\right)$$

$$M_v^{(k)} = \bigoplus_{u \in \mathcal{N}(v)} m_{u \to v}^{(k)}$$

$$h_v^{(k)} = \text{UPD}^{(k)}\!\left(h_v^{(k-1)},\, M_v^{(k)}\right)$$

where $\bigoplus$ is a permutation-invariant aggregation (sum, mean, max). In the simplest GCN case, this reduces to:

$$h_v^{(k)} = \sigma\!\left(\sum_{u \in \mathcal{N}(v) \cup \{v\}} \frac{1}{\sqrt{d_u d_v}}\, W^{(k)} h_u^{(k-1)}\right)$$

For graph-level tasks (molecular property prediction, graph classification), a ''readout'' function pools all node representations into a single graph vector:

$$h_G = \text{READOUT}\!\left(\{h_v^{(K)} \mid v \in V\}\right)$$

Common readouts include sum, mean, or attention-weighted pooling. The readout must also be permutation invariant — the graph representation should not depend on how nodes are numbered.

The ''Weisfeiler-Lehman (WL) connection:'' Xu et al. proved that no MPNN can distinguish graphs that the 1-WL test cannot distinguish. The 1-WL test iteratively refines node labels by hashing multisets of neighbor labels — exactly what message passing does. GIN matches 1-WL power; GCN and GraphSAGE are strictly weaker because mean/max aggregation fails to distinguish certain multisets.

!! Edge Features and Heterogeneous Graphs

Real-world graphs often carry rich edge information. In molecular graphs, edges encode bond types (single, double, aromatic). In <<link "Knowledge Graphs" "Knowledge Graphs">><</link>>, edges represent different relation types (''born_in'', ''works_at'', ''is_a''). The MPNN framework naturally incorporates edge features $e_{uv}$ into the message function, allowing the network to produce different messages depending on the relationship between nodes.

''Heterogeneous message passing'' extends the framework to graphs with multiple node types and edge types. Each (source_type, edge_type, target_type) triple can have its own message function and weight matrices. This is critical for real-world applications: citation networks have papers and authors; biological networks have genes, proteins, and diseases; social networks have users, posts, and groups.

''Edge updates:'' Some architectures (e.g., MPNN for molecules, PaiNN for 3D point clouds) also update edge representations alongside node representations, enabling bidirectional information flow between node and edge features.

!! When Message Passing Breaks Down

''(1) Over-smoothing: deep MPNNs make all nodes look identical.'' Each message-passing layer mixes a node's features with its neighbors'. After $K$ layers, every node's representation is influenced by its entire $K$-hop neighborhood. For small-diameter graphs, even 5-6 layers cause all node representations to converge to a single vector — the graph equivalent of blurring an image until everything is the same gray. Residual connections (as in DeepGCN) and normalization help, but fundamentally limit practical MPNN depth to 2-4 layers for most tasks.

''(2) Over-squashing: long-range information gets exponentially compressed.'' When a node needs information from a distant node, that information must travel through a chain of intermediate nodes, each of which aggregates messages from //all// its neighbors. The distant signal gets diluted exponentially with distance. Alon and Yahav (2021) showed this is related to the graph's Cheeger constant — bottleneck edges create information chokepoints. Graph rewiring (adding virtual edges) and graph Transformers partially address this.

''(3) Limited expressiveness: MPNNs cannot distinguish all non-isomorphic graphs.'' The 1-WL limitation means standard MPNNs fail on graphs with symmetric local structure. For example, they cannot count triangles or detect certain subgraph patterns. Higher-order GNNs (k-WL), subgraph GNNs, and equivariant architectures break past this barrier but at significantly higher computational cost.

!! Technical Details

''Original paper:'' Gilmer, Schoenholz, Riley, Vinyals, Dahl (ICML 2017) — "Neural Message Passing for Quantum Chemistry." Applied to molecular property prediction on the QM9 dataset, achieving state-of-the-art results by unifying prior GNN variants.

''Complexity:'' One message-passing layer costs $O(|E| \cdot d^2)$ where $|E|$ is the number of edges and $d$ is the feature dimension. The $d^2$ factor comes from the linear transformation $W h_u$. For sparse graphs, this is efficient; for dense graphs, it approaches $O(N^2 d^2)$.

''Mini-batching:'' Unlike images, graphs vary in size and structure. Mini-batching requires either (a) padding to a maximum size (wasteful), (b) block-diagonal adjacency matrices combining multiple graphs, or (c) neighborhood sampling (GraphSAGE). Libraries like PyTorch Geometric and DGL implement efficient batching.

''Relation to Transformers:'' A Transformer's self-attention layer is message passing on a //fully connected// graph where the attention weights serve as learned edge weights. Graph Transformers add structural encodings (e.g., Laplacian eigenvectors, random walk features) to inject graph topology into the attention mechanism.

''Virtual nodes:'' Adding a virtual node connected to all real nodes gives every pair of nodes a 2-hop path, mitigating over-squashing. The virtual node acts as a global memory that all nodes read from and write to during message passing.
