! Tensor Algebra

<img src="assets/tensor_algebra.svg" alt="Diagram showing the progression from scalar to vector to matrix to 3rd-order tensor with labeled axes" style="width:100%;max-width:620px;display:block;margin:16px auto 24px auto;">

Think of a spreadsheet. A single cell holds one number — that is a scalar. A row of cells is a vector. An entire sheet with rows and columns is a matrix. Now imagine stacking many sheets into a thick book: each entry requires three coordinates (row, column, page) to locate. That book is a third-order tensor. Tensor algebra is the mathematics of these multi-indexed objects and the operations that connect them — a natural extension of [[Core Linear Algebra]] into higher dimensions, and the backbone of how modern machine learning frameworks actually store, move, and transform data.

---

!! The Core Insight

A ''tensor'' is not just a multi-dimensional array — it is a ''multilinear map''. The word "multilinear" means linear in each argument separately while holding the others fixed. A matrix multiplies one vector on the left and one on the right; a third-order tensor accepts three vectors and returns a scalar. This perspective explains why tensors appear everywhere in ML: the core computation of a neural network — applying a weight matrix to an input vector — is a bilinear operation, and the moment you batch inputs, add channels, or stack attention heads, you are working in genuinely higher-order tensor spaces whether you realise it or not.

The order (sometimes called ''rank'' in physics, though this conflicts with the linear-algebraic meaning of rank) counts the number of independent indices. A scalar is order 0, a vector is order 1, a matrix is order 2, and a feature map in a [[CNN|Convolutional Neural Networks]] with batch, channel, height, and width dimensions is an order-4 tensor. Every operation in [[Singular Value Decomposition]], [[Matrix Factorization]], and [[Non-Negative Matrix Factorization]] has a tensor generalisation, and those generalisations unlock structure that flat matrix methods simply cannot see.

---

!! Tensor Operations and Decompositions

''Mode-n product.'' The mode-$n$ product multiplies a tensor by a matrix along its $n$-th mode. For a third-order tensor $\mathcal{X} \in \mathbb{R}^{I \times J \times K}$ and a matrix $\mathbf{U} \in \mathbb{R}^{M \times J}$, the mode-2 product $\mathcal{Y} = \mathcal{X} \times_2 \mathbf{U}$ gives $\mathcal{Y} \in \mathbb{R}^{I \times M \times K}$ by contracting the second index of $\mathcal{X}$ with the column index of $\mathbf{U}$. This is the tensor generalisation of ordinary matrix multiplication $\mathbf{Y} = \mathbf{X}\mathbf{U}^\top$ and is fundamental to every decomposition below.

''CP decomposition (CANDECOMP/PARAFAC).'' CP expresses an order-$N$ tensor as a sum of rank-one tensors — each being an [[Outer Product]] of $N$ vectors:

$$\mathcal{X} \approx \sum_{r=1}^{R} \lambda_r \, \mathbf{a}_r^{(1)} \circ \mathbf{a}_r^{(2)} \circ \cdots \circ \mathbf{a}_r^{(N)}$$

The number of terms $R$ is the ''tensor rank''. Unlike matrix rank, tensor rank is NP-hard to compute in general. CP decomposition is uniquely identifiable under mild conditions — a property matrices do not have without additional constraints — making it especially powerful for source separation and latent factor recovery.

''Tucker decomposition.'' Tucker decomposes a tensor into a small ''core tensor'' $\mathcal{G}$ multiplied by a factor matrix along each mode:

$$\mathcal{X} \approx \mathcal{G} \times_1 \mathbf{U}^{(1)} \times_2 \mathbf{U}^{(2)} \times_3 \mathbf{U}^{(3)}$$

This is the tensor analogue of truncated [[Singular Value Decomposition|SVD]]: the factor matrices $\mathbf{U}^{(n)}$ capture the principal subspace along each mode, and the core $\mathcal{G}$ captures the interactions between modes. Tucker is more flexible than CP (CP is the special case where the core is superdiagonal) but requires more parameters.

''Tensor train (TT) decomposition.'' TT represents a high-order tensor as a chain of three-dimensional cores $\mathcal{G}_1, \mathcal{G}_2, \dots, \mathcal{G}_N$, each connected to the next by a shared ''bond dimension'' $r_k$. Element-wise:

$$\mathcal{X}(i_1, i_2, \dots, i_N) = \mathbf{G}_1(i_1) \, \mathbf{G}_2(i_2) \cdots \mathbf{G}_N(i_N)$$

where each $\mathbf{G}_k(i_k)$ is an $r_{k-1} \times r_k$ matrix slice. The total parameter count scales linearly in $N$ rather than exponentially, making TT the method of choice for very high-order tensors that arise in quantum chemistry, probabilistic graphical models, and [[Dimensionality Reduction]] of enormous parameter spaces.

---

!! What the Math Is Actually Doing

A tensor of order $N$ over vector spaces $V_1, V_2, \dots, V_N$ is an element of the ''tensor product space'' $V_1 \otimes V_2 \otimes \cdots \otimes V_N$. Concretely, if $\{e_i^{(k)}\}$ is a basis for $V_k$, then $\mathcal{X}$ has components:

$$\mathcal{X} = \sum_{i_1, i_2, \dots, i_N} x_{i_1 i_2 \cdots i_N} \; e_{i_1}^{(1)} \otimes e_{i_2}^{(2)} \otimes \cdots \otimes e_{i_N}^{(N)}$$

''Contraction'' is the fundamental operation: summing over a shared index between two tensors, generalising matrix multiplication. If $\mathcal{A} \in \mathbb{R}^{I \times J}$ and $\mathcal{B} \in \mathbb{R}^{J \times K}$, contracting over $J$ yields $C_{ik} = \sum_j A_{ij} B_{jk}$ — ordinary matrix multiplication. For higher-order tensors, contracting mode $p$ of $\mathcal{A}$ with mode $q$ of $\mathcal{B}$ yields a tensor whose order is $\text{order}(\mathcal{A}) + \text{order}(\mathcal{B}) - 2$.

The ''mode-n unfolding'' (matricisation) rearranges a tensor into a matrix by mapping mode $n$ to the rows and all other modes to the columns. For $\mathcal{X} \in \mathbb{R}^{I_1 \times I_2 \times I_3}$, the mode-1 unfolding $\mathbf{X}_{(1)} \in \mathbb{R}^{I_1 \times (I_2 I_3)}$ stacks the "fibers" along mode 1 as columns. Unfoldings connect tensor decompositions to matrix decompositions: the columns of the factor matrix $\mathbf{U}^{(n)}$ in Tucker decomposition span the column space of $\mathbf{X}_{(n)}$.

The ''$n$-rank'' of a tensor is the tuple $(r_1, r_2, \dots, r_N)$ where $r_n = \text{rank}(\mathbf{X}_{(n)})$. This is distinct from the CP rank $R$. For matrices, both notions coincide, but for higher-order tensors they diverge — a fact that makes tensor algebra richer and more subtle than its matrix counterpart.

The ''Kruskal rank'' of a set of vectors is the maximum $k$ such that every subset of $k$ vectors is linearly independent. The celebrated ''Kruskal uniqueness theorem'' states that a CP decomposition is essentially unique when the sum of Kruskal ranks of the factor matrices exceeds $2R + N - 1$. This uniqueness without orthogonality constraints is why tensor methods outperform matrix methods for blind source separation.

---

!! Tensors in Machine Learning

''Tensor networks for model compression.'' Large weight matrices in neural networks can be reshaped into higher-order tensors and compressed via TT or Tucker decomposition, reducing parameter counts by 10-100x with minimal accuracy loss. The [[Inner Product]] between compressed weight tensors and input tensors is computed without ever materialising the full weight matrix, enabling deployment on memory-constrained devices.

''Higher-order statistics.'' While a covariance matrix captures second-order statistics, many distributions require third-order (skewness) or fourth-order (kurtosis) tensors to characterise fully. Independent Component Analysis (ICA) can be formulated as a simultaneous diagonalisation of the fourth-order cumulant tensor — a problem that CP decomposition solves directly.

''Attention as tensor contraction.'' In a [[Transformers|Transformer]], multi-head attention computes $\text{softmax}(QK^\top / \sqrt{d_k})V$. The queries, keys, and values across all heads form a third-order tensor (batch $\times$ heads $\times$ sequence $\times$ dimension), and the attention computation is a sequence of tensor contractions. Explicit tensor decomposition of attention weights has been used to study what Transformers learn and to compress multi-head attention into fewer parameters.

''Tensorised neural networks.'' Replacing a fully connected layer's weight matrix $\mathbf{W} \in \mathbb{R}^{M \times N}$ with a TT-decomposed tensor reduces the parameter count from $O(MN)$ to $O((MN)^{1/2} r^2)$ where $r$ is the TT-rank. This idea, introduced by Novikov et al. (2015), has been extended to convolutional layers, embedding layers (critical for large-vocabulary language models), and recurrent networks.

---

!! When Tensor Methods Break Down

''Computational intractability of rank and decomposition.'' Determining the rank of a tensor is NP-hard, and computing the best rank-$R$ CP approximation is ill-posed: unlike matrices, the set of tensors of rank at most $R$ is not closed, meaning a sequence of rank-$R$ tensors can converge to a tensor of lower rank while the component vectors diverge to infinity. This "degeneracy" problem means CP fitting algorithms can fail to converge or return numerically meaningless solutions, requiring careful initialisation and regularisation.

''Curse of dimensionality in storage and computation.'' A dense tensor of order $N$ with $I$ entries per mode requires $I^N$ elements — exponential in the order. Even for modest dimensions (e.g., $I = 100$, $N = 6$), the tensor has $10^{12}$ entries and cannot be stored, let alone decomposed. Structured formats like TT mitigate this, but they impose their own constraints: not every tensor admits a low-TT-rank representation, and operations between two TT tensors can cause rank explosion that requires expensive rounding (truncation) to control.

''Lack of mature, general-purpose tooling.'' While matrix operations enjoy decades of optimised BLAS/LAPACK support and GPU-accelerated routines in every major framework, tensor decomposition libraries (TensorLy, TensorNetwork, ITensor) are comparatively immature. GPU support is uneven, automatic differentiation through tensor decompositions is fragile, and there is no standardised API — meaning practitioners often fall back to reshaping tensors into matrices and applying familiar SVD or NMF tools, losing the multi-modal structure that tensor methods are specifically designed to exploit.

!! Technical Details

''Key references:'' Kolda & Bader (2009) "Tensor Decompositions and Applications" (SIAM Review) is the foundational survey. Oseledets (2011) introduced the TT decomposition. Novikov et al. (2015) "Tensorizing Neural Networks" demonstrated TT-compressed fully connected layers.

''Tensor rank vs matrix rank:'' For a matrix, rank equals the minimum number of rank-1 terms in any decomposition. For tensors of order $\geq 3$, the analogous ''border rank'' (minimum rank achievable in the limit) can be strictly less than the rank — a phenomenon with no matrix analogue.

''HOSVD (Higher-Order SVD):'' Computes the Tucker decomposition by performing SVD on each mode-$n$ unfolding independently. The resulting factor matrices are orthogonal, but the core tensor $\mathcal{G}$ is not diagonal (unlike the matrix case). HOSVD is not optimal in the least-squares sense but provides a good initialisation for iterative methods like HOOI (Higher-Order Orthogonal Iteration).

''Einstein summation notation:'' Modern frameworks (NumPy's `einsum`, PyTorch's `torch.einsum`) use index-based notation to express arbitrary tensor contractions concisely, e.g., `torch.einsum('bijk,bkl->bijl', A, B)` contracts the last mode of $A$ with the second mode of $B$. This notation eliminates transpositions, reshapes, and explicit loops, and the backends can optimise contraction order for minimal FLOPs.

''Tensor rank over different fields:'' The rank of a real tensor can differ from the rank of the same tensor viewed over the complex numbers. The maximum rank of a real $2 \times 2 \times 2$ tensor is 3, but over $\mathbb{C}$ it is 2. This subtlety matters in signal processing applications where the underlying model is real but complex decomposition tools are used.

''Multilinear rank and HOOI:'' The multilinear rank $(r_1, r_2, \dots, r_N)$ determines the minimal core size for a Tucker decomposition. HOOI alternates between updating each factor matrix via the leading $r_n$ left singular vectors of the mode-$n$ unfolding of the current residual, converging to a local minimum of the reconstruction error.
