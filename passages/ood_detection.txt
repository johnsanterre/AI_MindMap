! Out-of-Distribution Detection

<img src="assets/ood_detection.svg" alt="Diagram showing OOD detection methods: softmax scores, energy-based, Mahalanobis distance, and contrastive approaches" style="width:100%;max-width:620px;display:block;margin:16px auto 24px auto;">

Imagine a doctor who has trained exclusively on X-rays of adult lungs. One day, a veterinary scan of a parrot arrives in the queue. The doctor has never seen anything like it, yet confidently announces a diagnosis — because nothing in the training said "you are allowed to say I don't know." ''Out-of-distribution (OOD) detection'' is the discipline of teaching models to recognize when an input is unlike anything they were trained on, so they can flag it rather than produce a dangerously confident but meaningless prediction. It sits at the intersection of <<link "Model Calibration" "Model Calibration">><</link>>, <<link "Anomaly Detection" "Anomaly Detection">><</link>>, and <<link "Robustness" "Robustness">><</link>>, and it is one of the most important unsolved problems in deploying machine learning safely.

---

!! The Core Insight

Standard classifiers are trained with a closed-world assumption: every input at test time belongs to one of the training classes. The <<link "Softmax" "Softmax">><</link>> function at the output layer always produces a valid probability distribution — it must sum to one — so every input, no matter how alien, receives a confident-looking class assignment. A model trained on cats and dogs will assign a goldfish image 62% "cat" and 38% "dog" without blinking.

OOD detection reframes classification as a two-stage pipeline. ''Stage one:'' is this input from the same distribution as my training data? ''Stage two:'' if yes, classify it normally. The challenge is that stage one receives no labeled OOD examples during training — you cannot enumerate everything the model has //not// seen. Instead, OOD detectors extract a ''scoring function'' from the model's internal representations or outputs, and threshold it: inputs scoring above the threshold are treated as in-distribution (ID), while those below are flagged as OOD.

The fundamental tension is that a good OOD score must separate all possible OOD inputs from ID inputs, using only knowledge of the ID data. This is an open-set recognition problem, and no single method solves it perfectly. Different scoring functions exploit different properties — output confidence, feature-space geometry, energy landscapes, or learned representations — each with distinct failure modes.

---

!! Methods and Scoring Functions

The simplest approach is the ''Maximum Softmax Probability (MSP)'' baseline proposed by Hendrycks & Gimpel (2017). The score is simply $S(x) = \max_k p(y = k \mid x)$. ID inputs tend to produce higher maximum softmax values than OOD inputs. Despite its simplicity, MSP is a surprisingly strong baseline. Its weakness is that softmax is not calibrated for OOD — neural networks can produce high-confidence predictions on inputs far from the training manifold because the softmax function compresses logit differences into probabilities without regard to absolute magnitude.

''ODIN'' (Liang et al., 2018) improves on MSP by applying two tricks: temperature scaling of the softmax (dividing logits by a temperature $T > 1$ before softmax) and small input perturbations in the direction that increases the maximum softmax score. The perturbation amplifies the gap between ID and OOD scores. ODIN requires tuning $T$ and the perturbation magnitude $\epsilon$, typically using a held-out OOD validation set.

''Energy-based'' scoring (Liu et al., 2020) bypasses softmax entirely. The energy of an input is defined as:

$$E(x) = -\log \sum_{k=1}^{K} e^{f_k(x)}$$

where $f_k(x)$ are the logits. ID inputs have lower (more negative) energy; OOD inputs have higher energy. Energy scoring is theoretically motivated by the connection to the data likelihood under an energy-based model interpretation: $p(x) \propto e^{-E(x)}$. It dominates MSP because it uses the full logit vector, not just the maximum.

''Mahalanobis distance'' (Lee et al., 2018) operates in the feature space of intermediate network layers. For each class $c$, compute the empirical mean $\hat{\mu}_c$ and a shared covariance $\hat{\Sigma}$ of features from training data. The score for a test input $x$ with feature vector $f(x)$ is:

$$M(x) = \min_c \; (f(x) - \hat{\mu}_c)^T \hat{\Sigma}^{-1} (f(x) - \hat{\mu}_c)$$

ID inputs cluster near class means; OOD inputs lie far away in Mahalanobis distance. This method assumes class-conditional features are approximately Gaussian — a reasonable assumption in deep feature spaces due to the tendency toward <<link "Neural Collapse" "Neural Collapse">><</link>>.

''Contrastive'' approaches (e.g., CSI by Tack et al., 2020; SSD by Sehwag et al., 2021) learn representations where ID data is tightly clustered. Using contrastive learning objectives, the model pulls ID examples together and pushes augmented views apart, creating a feature space where OOD inputs naturally fall far from ID clusters. The OOD score is typically a distance metric — cosine similarity or $k$-nearest neighbor distance — in the learned representation space.

---

!! What the Math Is Actually Doing

All OOD scoring functions are attempting to estimate, directly or indirectly, a quantity related to the data density $p(x)$ or the density ratio $p(x_{\text{ID}}) / p(x_{\text{OOD}})$. The difficulty is that neural classifiers learn $p(y \mid x)$, which carries no information about $p(x)$.

For the energy score, the connection is explicit. If we treat the classifier logits as an energy-based model, the marginal data density is:

$$p(x) = \frac{e^{-E(x)}}{Z} = \frac{\sum_k e^{f_k(x)}}{Z}$$

where $Z = \int \sum_k e^{f_k(x')} \, dx'$ is the partition function. Since $Z$ is constant for a fixed model, $-E(x) = \log \sum_k e^{f_k(x)}$ is a monotonic function of $p(x)$. Thresholding the energy is equivalent to thresholding an unnormalized estimate of $p(x)$.

The Mahalanobis score takes a different route. Under the assumption that class-conditional features $f(x) \mid y = c$ follow $\mathcal{N}(\mu_c, \Sigma)$, the Mahalanobis distance to the nearest class mean is proportional to $-\log p(f(x) \mid y = c^*)$. Minimizing over classes selects the most favorable Gaussian component, approximating $-\log \max_c \, p(f(x) \mid y = c)$. In-distribution inputs have small distances (high likelihood under some class); OOD inputs have large distances (low likelihood under all classes).

The softmax score $\max_k \, p(y = k \mid x)$ is related to $p(x)$ only indirectly and unreliably. By Bayes' theorem, $p(y \mid x) = p(x \mid y) p(y) / p(x)$. A high $p(y \mid x)$ could come from high $p(x \mid y)$ (the input is typical of that class) or from low $p(x)$ (the input is unusual but the class prior or likelihood ratio still favors one class). This is why MSP can produce high confidence on OOD inputs — it conflates typicality with <<link "Distribution Shift" "Distribution Shift">><</link>>.

Temperature scaling modifies the sharpness of the softmax: $p_T(y = k \mid x) = e^{f_k(x)/T} / \sum_j e^{f_j(x)/T}$. For $T > 1$, the distribution flattens, reducing the gap between the highest and second-highest logits. This flattening disproportionately affects OOD inputs (whose logits tend to be more uniform), widening the score gap between ID and OOD. The perturbation step in ODIN exploits the gradient $\nabla_x \max_k \log p_T(y = k \mid x)$ to nudge the input toward higher confidence, and ID inputs respond more strongly to this nudge because they lie in regions of steeper loss landscape.

---

!! Practical Considerations

''Choosing the threshold'' is the central practical decision. The threshold on the OOD score determines the trade-off between two errors: falsely rejecting ID inputs (reduced accuracy) and falsely accepting OOD inputs (unsafe predictions). The standard evaluation metric is the area under the ROC curve (AUROC) and the false positive rate at 95% true positive rate (FPR95). A good detector achieves FPR95 below 5% — meaning fewer than 5% of OOD inputs sneak through when 95% of ID inputs are correctly accepted.

''Near-OOD vs. far-OOD'' is a critical distinction. Far-OOD detection (CIFAR-10 vs. SVHN, or text vs. images) is relatively easy — most methods achieve near-perfect AUROC. Near-OOD detection (CIFAR-10 vs. CIFAR-100, or different hospital scanners for the same pathology) is far harder, because near-OOD inputs share low-level features with ID data. Practical deployment almost always involves near-OOD, which is why benchmark results on far-OOD datasets can be misleading.

''Post-hoc vs. training-time methods:'' MSP, ODIN, energy, and Mahalanobis are all post-hoc — they work with any pre-trained classifier and require no retraining. Contrastive methods and Outlier Exposure (Hendrycks et al., 2019) modify the training objective, producing better representations at the cost of requiring OOD-aware training. Post-hoc methods are popular because retraining large models is expensive, but training-time methods generally produce stronger detectors.

''Multi-layer aggregation'' improves detection. Rather than using only the final layer's features or logits, methods like the Mahalanobis detector compute scores at multiple intermediate layers and combine them (e.g., via a simple logistic regression). Different layers capture different aspects of typicality: early layers capture texture and low-level patterns; later layers capture semantic content. OOD inputs may look normal at one layer but anomalous at another.

---

!! When OOD Detection Breaks Down

''Adversarial OOD inputs:'' An adversary who knows your OOD detector can craft inputs that are genuinely out-of-distribution but produce in-distribution scores. For example, small perturbations to an OOD image can push the Mahalanobis distance below the threshold or raise the softmax confidence above it. All score-based detectors are vulnerable to adaptive adversaries, because the scoring function is a known, differentiable function that can be optimized against. Adversarial <<link "Robustness" "Robustness">><</link>> of the detector itself — not just the classifier — is an open problem.

''Overconfident softmax on structured OOD:'' Neural networks can assign extremely high confidence to OOD inputs that happen to align with learned features. A model trained on handwritten digits will confidently classify a photo of a house number as a digit — correctly in some sense, but dangerously so when the input context is wrong. The <<link "Softmax" "Softmax">><</link>> function compresses logits into probabilities in a way that destroys information about whether the input is "in scope." Energy and Mahalanobis methods partially address this, but any method relying on classifier features inherits the classifier's blind spots.

''Distribution shift without semantic shift:'' OOD detection assumes that distribution shift implies semantic novelty — that a shifted input should be flagged. But many real-world shifts are benign: a photo taken at a different time of day, a medical image from a different scanner, or text in a slightly different dialect. These cause <<link "Distribution Shift" "Distribution Shift">><</link>> in pixel or token space without changing the semantic content. OOD detectors that are too sensitive will flag these harmless shifts, creating an unacceptable false-positive rate in production. Separating //semantic// OOD from //covariate// shift remains a major unsolved challenge.

!! Technical Details

''Key papers:'' Hendrycks & Gimpel (2017), //A Baseline for Detecting Misclassified and Out-of-Distribution Examples in Neural Networks// — established the MSP baseline and the standard evaluation protocol (AUROC, FPR95). Liang et al. (2018), //Enhancing the Reliability of Out-of-Distribution Image Detection in Neural Networks// — introduced ODIN with temperature scaling and input perturbations. Lee et al. (2018), //A Simple Unified Framework for Detecting Out-of-Distribution Samples and Adversarial Attacks// — proposed the Mahalanobis distance detector using class-conditional Gaussians in feature space. Liu et al. (2020), //Energy-based Out-of-Distribution Detection// — introduced the energy score and provided theoretical justification via the energy-based model interpretation.

''Outlier Exposure (OE)'' by Hendrycks et al. (2019) fine-tunes the model on a large auxiliary dataset of diverse images (e.g., 80 Million Tiny Images or Textures), training the network to produce uniform softmax outputs on these auxiliary examples. This teaches the model to be uncertain on non-ID data without requiring access to the specific OOD distribution at test time. OE is simple and effective but assumes the auxiliary dataset covers enough OOD variation to generalize.

''ReAct'' (Sun et al., 2021) discovered that OOD inputs tend to produce abnormally large activations in the penultimate layer. By simply truncating activations above a threshold (rectifying the feature space), ReAct dramatically improves energy-based OOD detection with zero retraining. This finding suggests that overconfidence on OOD inputs is partly driven by activation magnitudes rather than activation patterns.

''OpenOOD benchmark'' standardizes evaluation across near-OOD and far-OOD settings, multiple architectures, and both post-hoc and training-time methods. Standard benchmarks include CIFAR-10 vs. CIFAR-100 (near-OOD), CIFAR-10 vs. SVHN/Textures/Places365 (far-OOD), and ImageNet-1K vs. iNaturalist/SUN/Places/Textures.

The connection to <<link "Model Calibration" "Model Calibration">><</link>> is important but subtle. A well-calibrated model assigns accurate confidence to in-distribution predictions, but calibration says nothing about OOD behavior. A model can be perfectly calibrated on ID data while remaining dangerously overconfident on OOD inputs. OOD detection and calibration address different failure modes and require different solutions. <<link "Anomaly Detection" "Anomaly Detection">><</link>> is the unsupervised cousin: it identifies unusual inputs within a single unlabeled distribution, while OOD detection specifically identifies inputs that fall outside the //training// distribution of a supervised classifier.
