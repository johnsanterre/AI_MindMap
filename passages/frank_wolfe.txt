! Frank-Wolfe Algorithm
<img src="assets/frank_wolfe.svg" alt="Diagram showing the Frank-Wolfe algorithm iterating over a convex feasible set by solving linear subproblems and stepping toward vertices" style="max-width:100%; margin:1em 0;">

Imagine you need to find the lowest point in a valley, but you are only allowed to walk within a fenced park. Most optimization methods would, at each step, walk downhill and then project back inside the fence — an expensive correction. The Frank-Wolfe algorithm (also called the ''Conditional Gradient'' method) takes a cleverer approach: instead of projecting, it asks "which corner of the fence is most downhill from here?" and then takes a step toward that corner. Because the step is a convex combination of the current point and a vertex, the result is automatically inside the fence — no projection needed.

!! The Core Insight

Frank and Wolfe (1956) observed that minimizing a smooth convex function over a convex set can be reduced to solving a sequence of ''linear'' minimization problems over the same set. At each iteration, you linearize the objective around the current point and find the vertex of the constraint set that minimizes this linear approximation. Then you step toward that vertex. The update is always a convex combination of feasible points, so feasibility is maintained for free.

This is transformative when projection onto the constraint set is expensive but linear minimization over it is cheap. Think of the ''nuclear norm ball'' (where projection requires a full SVD, but linear minimization requires only the top singular vector), or ''combinatorial polytopes'' (where projection is NP-hard, but linear optimization is polynomial). !! Why Iterates Are Naturally Sparse

Each Frank-Wolfe iterate is a convex combination of at most $t+1$ vertices of the constraint set after $t$ steps. When the feasible set is a polytope (e.g., the simplex, the $\ell_1$ ball, or the Birkhoff polytope), vertices are extremely sparse — the vertices of the $\ell_1$ ball are the standard basis vectors $\pm e_i$, for instance.

This means Frank-Wolfe builds up solutions as sparse combinations of atoms without any explicit sparsity penalty or thresholding. In matrix completion and low-rank problems over the nuclear norm ball, each "vertex" is a rank-1 matrix, so after $t$ steps the iterate has rank at most $t+1$. This ''structural sparsity'' is a major practical advantage.

!! What the Math Is Actually Doing

''Problem:'' Minimize $f(x)$ subject to $x \in \mathcal{C}$, where $f$ is convex and differentiable with $L$-Lipschitz continuous gradient, and $\mathcal{C}$ is a compact convex set.

''Linear Minimization Oracle (LMO):'' At iteration $t$, solve:

$$s_t = \arg\min_{s \in \mathcal{C}} \langle \nabla f(x_t), s \rangle$$

This replaces the objective with its first-order Taylor expansion at $x_t$ and minimizes over $\mathcal{C}$ — a linear program.

''Update:'' The descent direction is $d_t = s_t - x_t$. With step size $\gamma_t \in [0, 1]$:

$$x_{t+1} = x_t + \gamma_t (s_t - x_t) = (1 - \gamma_t) x_t + \gamma_t s_t$$

''Step size choices:''
* Fixed schedule: $\gamma_t = \frac{2}{t + 2}$
* Line search: $\gamma_t = \arg\min_{\gamma \in [0,1]} f(x_t + \gamma d_t)$

''Convergence rate:'' With the $2/(t+2)$ schedule, the ''Frank-Wolfe gap'' $g_t = \langle \nabla f(x_t), x_t - s_t \rangle$ satisfies:

$$f(x_t) - f(x^*) \leq \frac{2LD^2}{t + 2}$$

where $D = \text{diam}(\mathcal{C})$ and $L$ is the Lipschitz constant of $\nabla f$. This is an $O(1/t)$ sublinear rate — slower than projected gradient descent's $O(1/t^2)$ with acceleration, but each iteration may be dramatically cheaper.

''Duality gap certificate:'' The Frank-Wolfe gap $g_t$ is always an upper bound on the suboptimality $f(x_t) - f(x^*)$, giving a built-in stopping criterion at no extra cost.

!! Practical Applications

''Low-rank matrix problems:'' For matrix completion or matrix regression over the nuclear norm ball $\{X : \|X\|_* \leq \tau\}$, the LMO requires computing only the ''top singular vector pair'' of $-\nabla f(X_t)$, which is $O(mn)$ via power iteration — far cheaper than the full SVD needed for projection.

''Structured prediction in ML:'' In structured SVMs and CRFs, the feasible set is a marginal polytope. Frank-Wolfe reduces optimization to repeated MAP inference (a combinatorial problem with efficient solvers), avoiding the expensive projection onto the marginal polytope.

''Traffic assignment:'' The original motivation. Frank and Wolfe studied network flow problems where the feasible set is a flow polytope. The LMO is a shortest-path computation — fast and well-understood.

''Video co-localization and tracking:'' The flow polytope formulation of multi-object tracking problems allows Frank-Wolfe to decompose a joint optimization into successive shortest-path queries.

!! When Frank-Wolfe Breaks Down

''Non-smooth objectives:'' Frank-Wolfe requires $\nabla f$ to be Lipschitz continuous. If the objective has kinks or is non-differentiable (e.g., the hinge loss without smoothing), the algorithm may oscillate or fail to converge. Subgradient variants exist but lose the clean convergence guarantees.

''Slow tail convergence:'' The $O(1/t)$ rate means high-accuracy solutions require many iterations. For problems where projected gradient descent achieves $O(1/t^2)$ or linear convergence, Frank-Wolfe can be orders of magnitude slower in the final digits. Away-step and pairwise variants partially mitigate this, achieving linear convergence for strongly convex objectives over polytopes.

''Zig-zagging near the boundary:'' When the optimum lies on a face of $\mathcal{C}$ rather than at a vertex, Frank-Wolfe iterates can zig-zag between vertices of that face without converging efficiently. The duality gap decreases but the iterates themselves oscillate. Fully corrective or in-face variants address this by optimizing over the convex hull of previously visited vertices.

!! Technical Details

''Original paper:'' Frank, M. and Wolfe, P. (1956), "An algorithm for quadratic programming." Published in Naval Research Logistics Quarterly.

''Variants:''
- ''Away-step Frank-Wolfe (AFW):'' Can move away from bad vertices, achieving linear convergence for strongly convex functions over polytopes.
- ''Pairwise Frank-Wolfe:'' Transfers weight between two vertices per step, also achieving linear convergence.
- ''Fully corrective:'' At each step, re-optimizes over the convex hull of all visited vertices.
- ''Stochastic Frank-Wolfe:'' Uses stochastic gradient estimates in the LMO; convergence rate degrades to $O(1/\sqrt{t})$.
- ''Lazy / MFW (Momentum Frank-Wolfe):'' Reuses previous LMO solutions when the gradient has not changed much, reducing the number of LMO calls.

''Relation to boosting:'' Boosting in machine learning can be viewed as Frank-Wolfe on the exponential loss over the convex hull of weak learners. Each weak learner is a "vertex," and boosting greedily selects the one most aligned with the negative gradient.

''Affine-invariance:'' Unlike projected gradient descent, Frank-Wolfe convergence depends on the ''curvature constant'' $C_f$ of $f$ over $\mathcal{C}$, not on the Lipschitz constant in a particular norm. This makes the algorithm affine-invariant — its behavior does not change under invertible linear transformations.

''Memory:'' Iterates can be stored as a sparse combination of at most $t$ vertices, which is $O(t)$ in the number of atoms. Fully corrective variants periodically prune inactive vertices to control memory.
