! Sparse Autoencoders
<img src="assets/sparse_autoencoders.svg" alt="Sparse autoencoder diagram showing dense residual stream expanded to sparse feature vector and decoded back" style="max-width:100%;margin:1em 0;">

A symphony orchestra recorded onto a single audio track is nearly impossible to analyze — all instruments are superimposed on one another. A sparse autoencoder is like a demixing algorithm: it separates the orchestra into individual instrument tracks, even though each track is mostly silence. Applied to neural networks, it separates ''polysemantic'' neurons (playing many roles at once) into ''monosemantic'' features (each with one clear meaning).

!! The Core Insight

''Sparse autoencoders'' (SAEs) are a technique from [[Mechanistic Interpretability]] for decomposing the superposed representations inside neural networks. The ''superposition hypothesis'' (Elhage et al. 2022) explains why this is necessary: neural network models store far more features than they have dimensions. A 512-dimensional residual stream cannot hold 50,000 truly orthogonal feature directions, so the model encodes features as nearly-orthogonal directions that partially overlap. This causes individual neurons to respond to many unrelated concepts — a phenomenon called ''polysemanticity''. SAEs exploit the fact that features are sparse at any given moment to recover a large dictionary of monosemantic features from the compressed representation.

Related pages: [[Mechanistic Interpretability]], [[Autoencoders]], [[Embeddings]], [[Large Language Models]], [[Probe Identification]].

!! The Superposition Problem

A 512-dimensional residual stream cannot represent 50,000 features as orthogonal basis vectors — there simply are not enough dimensions. Instead, the model uses ''interference'' as a controlled tradeoff: each feature is assigned a direction in 512-d space, features are approximately orthogonal (angle close to 90°) but not exactly. When features are ''sparse'' — meaning only a few are active at once for any given input — the interference from non-active features is small, and the model can still correctly reconstruct any sparse combination of active features.

SAEs exploit this structure. They learn a large dictionary of directions (atoms) such that any input activation can be expressed as a sparse sum of a small number of dictionary elements. The sparsity constraint is what forces the dictionary to align with the true underlying features rather than arbitrary linear combinations.

!! What the Math Is Actually Doing

The SAE has three components: an encoder, a sparse code, and a decoder.

Given an input activation vector $x \in \mathbb{R}^d$ (e.g., $d = 512$):

''Encoder:'' $z = \text{ReLU}(W_{\text{enc}}\, x + b_{\text{enc}})$ where $z \in \mathbb{R}^m$ and $m \gg d$ (e.g., $m = 32{,}768$). ReLU forces most entries to zero.

''Decoder:'' $\hat{x} = W_{\text{dec}}\, z + b_{\text{dec}}$, reconstructing the original vector.

''Training loss:''
$$\mathcal{L} = \underbrace{\|x - \hat{x}\|^2}_{\text{reconstruction}} + \underbrace{\lambda \|z\|_1}_{\text{sparsity}}$$

The $\ell_1$ penalty on $z$ is the key: it pushes most feature activations to exactly zero, keeping only the features that genuinely explain the input. The hyperparameter $\lambda$ controls the reconstruction–sparsity tradeoff.

''TopK-SAE'' (Gao et al. 2024): instead of an $\ell_1$ penalty, constrain exactly $k$ features to be active — $\|z\|_0 = k$ — using a hard top-$k$ selection. This eliminates the need to tune $\lambda$ and gives more predictable sparsity levels.

''Dictionary size'' is typically 4×–64× larger than the input dimension. A 512-d residual stream might be expanded to 2,048, 16,384, or 65,536 features.

!! Findings from Scaling

Large-scale SAE experiments have found genuinely interpretable, monosemantic features:

''Anthropic (Templeton et al. 2024):'' Applied SAEs to the residual stream of Claude Sonnet. Found features for "The Eiffel Tower" (activates for French text, images of Paris, and English mentions of Paris), "DNA repair proteins," and "code syntax error in line 3." Many features are ''multimodal and multilingual'' — organized by concept, not by input language or modality. Feature steering experiments showed that clamping the "Golden Gate Bridge" feature caused the model to describe itself as the bridge.

''Gurnee and Tegmark (2023):'' Found linear representations of factual associations (countries, capitals, facts) in GPT-2 using sparse probing and SAE-style analysis.

''OpenAI (Gao et al. 2024):'' Scaled SAE training to GPT-4, finding millions of interpretable features. Introduced the TopK-SAE variant.

!! When Sparse Autoencoders Break Down

''Absorption:'' one intended monosemantic feature — for example, "starts with the letter K" — is not represented as a single clean feature but is instead absorbed into many other features in unexpected ways. The SAE may correctly activate most features for a concept while silently missing a component.

''Dead features:'' a large fraction of learned SAE features never activate on any real input. They represent wasted dictionary capacity and suggest the model has not yet found a meaningful direction for that slot. Typical SAEs have 5–30% dead features.

''Reconstruction vs. sparsity tradeoff:'' enforcing high sparsity produces cleaner, more interpretable features but degrades reconstruction fidelity, causing the SAE to miss activations that carry meaningful information. Tuning $\lambda$ or $k$ is non-trivial and dataset-dependent.

!! Technical Details

''Key papers:'' Cunningham et al. 2023 (first large-scale SAE applied to LLM internals); Templeton et al. 2024 (Anthropic, Claude Sonnet features); Gao et al. 2024 (OpenAI, TopK-SAE, scaling to GPT-4); Elhage et al. 2022 (superposition hypothesis, toy models).

''Tools:'' SAELens library (Joseph Bloom et al.) — open-source PyTorch library for training and evaluating SAEs on transformer models.

''Dictionary learning connection:'' SAEs are closely related to classical sparse coding (Olshausen and Field 1996) and the k-SVD algorithm. The key difference is scale: modern SAEs operate on millions of activation samples and dictionaries of tens of thousands of atoms.

''Evaluation methods:'' (1) probing accuracy — do SAE features predict known ground-truth labels? (2) automated interpretability — query a large LM to generate a natural-language description of each feature from its top activating examples, then score coherence; (3) reconstruction loss and $\ell_0$ sparsity on held-out data.

''Feature steering:'' clamping or adding multiples of a feature direction to the residual stream can causally test whether a feature controls the corresponding model behavior, linking SAE features to mechanistic explanations.
