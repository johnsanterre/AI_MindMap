! Activation Patching

<img src="assets/activation_patching.svg" alt="Diagram illustrating activation patching: clean activations are transplanted into a corrupted forward pass to identify causally important sites" style="width:100%;max-width:620px;display:block;margin:16px auto 24px auto;">

Imagine two identical assembly lines producing the same car. On the first line, everything works perfectly. On the second, someone has sabotaged one station — the engine block comes out wrong, and the car does not start. Now you want to find where the sabotage happened. So you start swapping parts from the good line into the bad one, station by station. The moment you transplant a part and the car starts working again, you have found the station that matters. ''Activation patching'' does exactly this inside a neural network: it runs two forward passes — one clean, one corrupted — then transplants activations from the clean run into the corrupted one, site by site, to discover which internal components are causally responsible for the model's behavior.

---

!! The Core Insight

The fundamental problem in neural network interpretability is distinguishing //correlation// from //causation//. A neuron might activate strongly whenever the model produces a correct answer, but that does not mean the neuron //caused// the correct answer — it might be a downstream echo of the real computation happening elsewhere. Activation patching resolves this by performing a controlled intervention: if replacing one specific activation in a corrupted run with the clean version restores the correct output, that site is not merely correlated with the behavior — it is //causally necessary// for it.

This technique is also called ''causal tracing'' (Meng et al. 2022) and is the primary experimental tool of <<link "Mechanistic Interpretability" "Mechanistic Interpretability">><</link>>. It connects deeply to the logic of <<link "Causal Inference" "Causal Inference">><</link>> — specifically the interventionist framework where causation is defined by what happens when you surgically change a variable while holding everything else fixed. In the language of causal graphs, each layer-position pair in the network is a node, and activation patching performs a do-calculus intervention on that node.

The technique has been used to localize factual recall in <<link "Transformers" "Transformers">><</link>> (Meng et al. 2022), reverse-engineer the Indirect Object Identification circuit in GPT-2 (Wang et al. 2022), and identify the components responsible for in-context learning. It is the backbone of model editing methods like <<link "ROME" "ROME">><</link>>, which first use causal tracing to find //where// a fact is stored, then surgically update that location.

---

!! How the Procedure Works

The activation patching procedure has four steps. First, choose a ''clean input'' — a prompt where the model produces the behavior of interest (e.g., "The Eiffel Tower is in" → "Paris"). Second, create a ''corrupted input'' by perturbing the clean input in a way that disrupts the behavior — typically by replacing the subject token with noise or a different entity ("The [random noise] is in" → garbage output). Third, run both inputs through the model and cache all intermediate activations: the residual stream states, <<link "Attention" "Attention">><</link>> head outputs, and MLP outputs at every layer and token position. Fourth, perform the ''patching sweep'': for each site $(l, t)$ — layer $l$, token position $t$ — replace the corrupted activation with the corresponding clean activation and measure how much the output recovers toward the clean answer.

The result is a two-dimensional ''causal map'' — a heatmap over layers and positions showing which sites, when patched, restore the correct behavior. Bright spots on this map are the network's causal bottlenecks: the places where the information needed for the behavior flows through.

''Path patching'' extends this idea by patching not just a site's output but the specific connection from one component to another. Instead of asking "is layer 8, position 3 important?" it asks "is the connection from attention head 7.2 to attention head 9.1 important?" This reveals not just //where// information is, but //how// it flows between components — enabling full circuit discovery.

---

!! What the Math Is Actually Doing

Let a transformer have $L$ layers, sequence length $T$, and hidden dimension $d$. The residual stream at layer $l$ and position $t$ is $h_t^{(l)} \in \mathbb{R}^d$. Denote the full forward pass on the clean input as $h_t^{(l, \text{clean})}$ and on the corrupted input as $h_t^{(l, \text{corrupt})}$.

The ''patched forward pass'' at site $(l^*, t^*)$ replaces one activation:

$$\tilde{h}_t^{(l)} = \begin{cases} h_t^{(l, \text{clean})} & \text{if } l = l^* \text{ and } t = t^* \\ f_l(\tilde{h}^{(l-1)}) & \text{otherwise} \end{cases}$$

where $f_l$ is the transformer layer function. After the patch, all subsequent layers run normally on the modified residual stream.

The ''patching effect'' at site $(l^*, t^*)$ is the change in the model's output probability for the correct token:

$$\Delta(l^*, t^*) = P(\text{correct} \mid \text{patched at } (l^*, t^*)) - P(\text{correct} \mid \text{fully corrupted})$$

Normalized by the clean-corrupted gap, this gives a ''recovery fraction'':

$$R(l^*, t^*) = \frac{\Delta(l^*, t^*)}{P(\text{correct} \mid \text{clean}) - P(\text{correct} \mid \text{corrupt})}$$

A recovery fraction near 1.0 means that site alone carries essentially all the causal information. A value near 0 means patching that site has no effect.

For ''path patching'' between components $C_i$ and $C_j$, the intervention is more surgical. Let $a_i$ be the output of component $C_i$ that feeds into $C_j$. Only the contribution of $C_i$ to $C_j$ is patched — all other inputs to $C_j$ remain corrupted:

$$\text{input to } C_j = a_i^{\text{clean}} + \sum_{k \neq i} a_k^{\text{corrupt}}$$

This isolates the specific edge in the computational graph, revealing directed information flow.

!! Practical Applications and Variants

''Causal tracing for factual recall:'' Meng et al. (2022) applied activation patching to GPT-2 and GPT-J to localize where factual associations are stored. They found that for prompts like "The Eiffel Tower is in ___," the critical site is the MLP output at the //subject token position// ("Eiffel Tower") in //early-to-mid layers//. This finding directly motivated <<link "ROME" "ROME">><</link>>, which edits facts by modifying the MLP weight matrices at these identified layers.

''Circuit discovery:'' Wang et al. (2022) used activation patching and path patching to reverse-engineer the 28-component Indirect Object Identification circuit in GPT-2. By systematically patching each attention head and MLP layer, they identified name-mover heads, S-inhibition heads, duplicate token heads, and the connections between them.

''Automated circuit discovery:'' Recent work automates the patching procedure using attribution methods. ''Attribution patching'' (Neel Nanda, 2023) approximates the full patching effect using gradients: $\Delta(l,t) \approx \nabla_{h_t^{(l)}} \mathcal{L} \cdot (h_t^{(l, \text{clean})} - h_t^{(l, \text{corrupt})})$, reducing the cost from one forward pass per site to a single forward-backward pass. This makes it feasible to run on larger models where exhaustive patching is prohibitively expensive.

''Activation patching with <<link "Sparse Autoencoders" "Sparse Autoencoders">><</link>>:'' Rather than patching at the level of full residual stream vectors, researchers now patch individual SAE features — interpretable directions extracted by sparse autoencoders. This provides feature-level causal attribution: not just "layer 8, position 3 matters," but "the 'capital city' feature at layer 8, position 3 matters."

---

!! When Activation Patching Breaks Down

''Distributed representations defeat localization:'' Activation patching assumes that causal responsibility is //localizable// — concentrated at a small number of sites. But some computations are genuinely distributed across many layers and positions, with no single site carrying more than a few percent of the causal effect. In these cases, the patching heatmap shows diffuse, weak effects everywhere, and the technique fails to produce actionable insights. This is particularly problematic for emergent capabilities that arise from the interaction of many components rather than a discrete circuit.

''Nonlinear interactions create misleading maps:'' Patching one site at a time assumes that causal contributions are approximately additive — that the effect of patching sites A and B together is roughly the sum of patching each alone. When components interact nonlinearly (e.g., one attention head gates another's output), single-site patching can miss the interaction entirely, or worse, can attribute causal importance to a site that only matters //in combination// with another. Path patching partially addresses this, but exhaustive interaction testing is combinatorially intractable.

''Corruption method biases results:'' The choice of how to corrupt the input is not neutral — different corruption strategies (Gaussian noise on embeddings, token replacement, token deletion) can produce different causal maps. Gaussian noise may overstate the importance of sites that are sensitive to noise generally, while token replacement may fail to disrupt computations that are robust to synonym substitution. There is no principled way to choose the "right" corruption, and results that appear crisp under one method may dissolve under another, raising questions about the robustness of causal tracing findings.

---

!! Technical Details

''Foundational papers:'' Vig et al. 2020 ("Causal Mediation Analysis for Interpreting Neural NLP") introduced causal mediation analysis to NLP models, adapting Pearl's causal framework. Meng et al. 2022 ("Locating and Editing Factual Associations in GPT") developed the causal tracing technique for localizing factual recall, directly motivating ROME. Wang et al. 2022 ("Interpretability in the Wild") used activation and path patching extensively for circuit analysis. Goldowsky-Dill et al. 2023 ("Localizing Model Behavior with Path Patching") formalized path patching as a tool for edge-level circuit discovery.

''Computational cost:'' Exhaustive activation patching requires $O(L \times T)$ forward passes per example, where $L$ is the number of layers and $T$ is the sequence length. For GPT-2 (48 layers, context 1024), this is approximately 49,000 forward passes. ''Attribution patching'' reduces this to $O(1)$ forward-backward passes at the cost of approximation quality — it is a first-order Taylor approximation and can miss nonlinear effects.

''Granularity levels:'' Patching can target the full residual stream $h_t^{(l)}$, individual attention head outputs $a_t^{(l,h)}$, MLP layer outputs $m_t^{(l)}$, or individual SAE features. Finer granularity provides more precise localization but requires more forward passes and is more susceptible to noise.

''Relation to <<link "Probe Identification" "Probe Identification">><</link>>:'' Probing tests whether information //exists// in a representation; activation patching tests whether the model //uses// that information. A representation can contain information (detectable by a probe) that the model ignores for the task at hand — only patching reveals causal usage. The two methods are complementary: probing surveys what is encoded; patching reveals what is causally active.

''Tools:'' TransformerLens (Neel Nanda) provides built-in hooks for activation patching at all residual stream sites. The ``patchscopes`` library extends patching to cross-model interventions.
