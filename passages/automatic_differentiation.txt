! Automatic Differentiation
<img src="assets/automatic_differentiation.svg" alt="Diagram showing a computational graph for f(x,y) = (x+y)*y with forward mode propagating derivatives left-to-right and reverse mode propagating gradients right-to-left" style="max-width:100%; margin:1em 0;">

Imagine you are following a recipe with many steps, and you want to know how changing the amount of salt affects the final dish. You could try to write out the entire recipe as one giant formula and simplify it (symbolic differentiation), or you could wiggle the salt amount slightly and taste the difference (numerical differentiation). Automatic differentiation does something cleverer: it follows the recipe step by step, tracking exactly how each tiny change propagates through every intermediate step, giving you the exact derivative with no approximation error and no formula explosion.

!! The Core Insight

Every computer program, no matter how complex, is ultimately a sequence of elementary operations — additions, multiplications, exponentials, logarithms. Each of these has a known derivative. ''Automatic differentiation'' (AD) exploits this by decomposing a program into its elementary steps, recording them in a ''computational graph'', and then applying the <<link "Chain Rule" "Backpropagation and Chain Rule">><</link>> mechanically through that graph. The result is exact (to floating-point precision), efficient, and fully automated.

There are two modes. ''Forward mode'' pushes derivatives forward alongside the computation, computing $\partial f / \partial x_i$ for one input $x_i$ per pass. ''Reverse mode'' first runs the computation forward, records the graph, then sweeps backward from the output to compute $\partial f / \partial x_i$ for ''all'' inputs in a single backward pass — this is exactly what we call <<link "Backpropagation" "Backpropagation and Chain Rule">><</link>>.

!! Forward Mode vs Reverse Mode

''Forward mode'' associates each intermediate variable $v_i$ with a ''tangent'' $\dot{v}_i = \partial v_i / \partial x$. You seed one input with $\dot{x} = 1$ and propagate forward. At the end you have $\dot{f} = \partial f / \partial x$. If you have $n$ inputs, you need $n$ forward passes — one per input.

''Reverse mode'' associates each intermediate variable with an ''adjoint'' $\bar{v}_i = \partial f / \partial v_i$. You seed the output with $\bar{f} = 1$ and propagate backward. One backward pass gives you $\partial f / \partial x_i$ for every input simultaneously. This is why reverse mode dominates deep learning: a neural network may have millions of parameters (inputs) but produces a single scalar loss (output).

The choice is driven by the shape of the <<link "Jacobian" "jacobian matrices">><</link>>. For a function $f : \mathbb{R}^n \to \mathbb{R}^m$, forward mode costs $O(n)$ passes and reverse mode costs $O(m)$ passes. Deep learning: $m = 1$ (scalar loss), $n$ is enormous — reverse mode wins decisively.

!! What the Math Is Actually Doing

Consider a simple computation $f(x, y) = (x + y) \cdot y$. We decompose it:

$$v_1 = x + y, \quad v_2 = v_1 \cdot y = f$$

''Forward mode'' (seeding $\dot{x} = 1, \dot{y} = 0$ to get $\partial f / \partial x$):

$$\dot{v}_1 = \dot{x} + \dot{y} = 1 + 0 = 1$$
$$\dot{v}_2 = \dot{v}_1 \cdot y + v_1 \cdot \dot{y} = 1 \cdot y + v_1 \cdot 0 = y$$

So $\partial f / \partial x = y$. Correct.

''Reverse mode'' (seeding $\bar{v}_2 = 1$):

$$\bar{v}_1 = \bar{v}_2 \cdot \frac{\partial v_2}{\partial v_1} = 1 \cdot y = y$$
$$\bar{x} = \bar{v}_1 \cdot \frac{\partial v_1}{\partial x} = y \cdot 1 = y$$
$$\bar{y} = \bar{v}_1 \cdot \frac{\partial v_1}{\partial y} + \bar{v}_2 \cdot \frac{\partial v_2}{\partial y} = y \cdot 1 + 1 \cdot v_1 = y + x + y = x + 2y$$

One backward pass yields both $\partial f / \partial x = y$ and $\partial f / \partial y = x + 2y$. This generalises: for a loss $L$ with millions of parameters $\theta_i$, one forward pass plus one backward pass gives every $\partial L / \partial \theta_i$.

The <<link "Jacobian" "jacobian matrices">><</link>> $J \in \mathbb{R}^{m \times n}$ of a function $f : \mathbb{R}^n \to \mathbb{R}^m$ connects the two modes: forward mode computes $J \dot{x}$ (Jacobian-vector product), reverse mode computes $J^T \bar{y}$ (vector-Jacobian product).

!! How Frameworks Implement AD

Modern deep learning frameworks implement reverse-mode AD via ''tape-based'' or ''operator-overloading'' approaches:

''PyTorch'' uses dynamic computational graphs. Every tensor operation is recorded on a tape at runtime. Calling {{{loss.backward()}}} traverses this tape in reverse, applying local <<link "Chain Rule" "Backpropagation and Chain Rule">><</link>> rules at each node. The tape is discarded after each backward pass (unless {{{retain_graph=True}}}).

''JAX'' takes a functional approach: {{{jax.grad(f)}}} returns a new function that computes the gradient of $f$. It uses ''tracing'' — running the function once with abstract values to build a computation graph (jaxpr), then transforming that graph. This enables composable transformations: {{{jax.grad(jax.grad(f))}}} gives second derivatives.

''Dual numbers'' implement forward mode elegantly. Extend reals to $a + b\epsilon$ where $\epsilon^2 = 0$. Evaluating $f(x + \epsilon)$ gives $f(x) + f'(x)\epsilon$ — the derivative rides along automatically. This is the mathematical basis for forward-mode AD.

!! When Automatic Differentiation Breaks Down

''Non-differentiable operations:'' AD requires every elementary operation to have a defined local derivative. Functions with hard discontinuities (e.g., {{{if x > 0}}} branches, integer rounding, argmax) produce zero or undefined gradients at the discontinuity. Frameworks silently return the gradient of whichever branch was taken, which can be misleading. Workarounds include straight-through estimators and softmax relaxations.

''Exploding computational graphs:'' Naive AD on iterative or recursive programs (long unrolled RNNs, optimization loops) creates enormous graphs that exhaust memory. The graph size grows linearly with the number of operations. Solutions include gradient checkpointing (recompute instead of store), truncated backpropagation through time, and implicit differentiation for fixed-point iterations.

''Numerical precision:'' While AD gives exact symbolic derivatives in theory, floating-point arithmetic introduces rounding errors that can accumulate through deep graphs. Catastrophic cancellation in subtraction-heavy computations (e.g., finite-difference-like patterns within an AD-traced program) can degrade gradient quality. Mixed-precision training exacerbates this by performing the forward pass in float16 while accumulating gradients in float32.

!! Technical Details

''Historical context:'' AD predates deep learning by decades. Forward mode was described by Wengert (1964). Reverse mode was independently discovered by Linnainmaa (1970) and later popularised for neural networks by Rumelhart, Hinton, and Williams (1986) as <<link "backpropagation" "Backpropagation and Chain Rule">><</link>>.

''Complexity:'' The ''cheap gradient principle'' (Griewank) states that computing the gradient of a scalar function costs at most 4-5x the cost of evaluating the function itself, regardless of the number of parameters. This is why training billion-parameter models is feasible.

''Higher-order derivatives:'' AD can be nested. Applying reverse mode to a reverse-mode computation yields Hessian-vector products in $O(n)$ time (Pearlmutter, 1994), avoiding the $O(n^2)$ cost of forming the full Hessian. This is used in second-order optimisation and meta-learning.

''Forward vs reverse in practice:'' Forward mode is preferred for functions $f : \mathbb{R} \to \mathbb{R}^m$ (few inputs, many outputs) — common in ODE solvers, sensitivity analysis, and physics simulations. Reverse mode dominates neural network training where $f : \mathbb{R}^n \to \mathbb{R}$.

''Relation to <<link "Gradient Descent" "Stochastic Gradient Descent">><</link>>:'' AD is not an optimisation algorithm. It is the ''mechanism'' that computes the gradients that gradient descent then uses to update parameters. Without efficient AD (specifically reverse mode), training deep networks would be computationally intractable.
