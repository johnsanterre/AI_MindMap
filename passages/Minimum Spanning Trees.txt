! Minimum Spanning Trees

<img src="assets/minimum_spanning_trees.svg" alt="Minimum Spanning Tree: weighted graph with MST edges highlighted in red, non-MST edges dashed" style="width:100%;max-width:620px;display:block;margin:16px auto 24px auto;">

Imagine you are an engineer tasked with connecting seven villages with roads. You could pave every possible route, but budgets are tight — you need every village reachable from every other, using the least total pavement. The ''Minimum Spanning Tree'' (MST) solves exactly this problem: given a connected, weighted graph, find the subset of edges that connects all vertices with the smallest possible total weight. The result is always a tree — no cycles, exactly $N-1$ edges for $N$ vertices — and it is //spanning// because it touches every vertex.

!! The Core Insight

The MST obeys a beautifully simple principle: ''safe edges can be added greedily''. At every step, the cheapest edge that does not create a cycle is guaranteed to belong to //some// MST. This is not a heuristic — it is provably optimal. The reason is the ''cut property'': for any partition of the vertices into two non-empty sets, the lightest edge crossing the cut must be in every MST (assuming unique weights). Conversely, the ''cycle property'' says that in any cycle, the heaviest edge cannot belong to any MST. These two properties are duals of each other, and they make the MST one of the rare optimization problems where greedy algorithms produce globally optimal solutions.

This greedy optimality is no accident — it arises because the set of forests on a graph forms a ''matroid'', an algebraic structure where the greedy algorithm is guaranteed to find the optimal independent set. The MST problem is, in a precise sense, the //prototypical// matroid optimization problem.

!! Algorithms for Finding MSTs

''Kruskal's algorithm'' (1956) sorts all edges by weight and adds them one by one, skipping any edge that would create a cycle. Cycle detection uses a [[Union-Find|Union-Find]] data structure, giving $O(E \log E)$ total time. The algorithm is conceptually the simplest and works well on sparse graphs.

''Prim's algorithm'' (1957) grows a single tree from an arbitrary starting vertex, always adding the cheapest edge that connects a tree vertex to a non-tree vertex. With a binary heap, this runs in $O(E \log V)$; with a Fibonacci heap, $O(E + V \log V)$. Prim's is often faster on dense graphs where $E$ approaches $V^2$.

''Borůvka's algorithm'' (1926) — the oldest of the three — works in parallel rounds. In each round, every connected component simultaneously selects its cheapest outgoing edge and merges. The number of components at least halves each round, so the algorithm terminates in $O(\log V)$ rounds, each costing $O(E)$ time, for $O(E \log V)$ total. Borůvka's parallelism makes it the foundation of modern parallel and distributed MST algorithms.

All three algorithms are correct because they exploit the cut property in different orderings: Kruskal's considers edges globally by weight, Prim's considers edges locally from the growing tree, and Borůvka's considers edges locally from every component simultaneously.

!! What the Math Is Actually Doing

Given a connected undirected graph $G = (V, E)$ with edge weights $w: E \to \mathbb{R}$, a ''spanning tree'' $T$ is a connected acyclic subgraph containing all vertices. The MST minimizes:

$$w(T) = \sum_{e \in T} w(e)$$

''Cut property:'' For any cut $(S, V \setminus S)$, let $e$ be the unique minimum-weight edge crossing the cut. Then $e$ belongs to every MST of $G$.

//Proof sketch:// Suppose MST $T^*$ does not contain $e$. Adding $e$ to $T^*$ creates a cycle that must cross the cut at least once more via some edge $f$. Since $w(e) < w(f)$, replacing $f$ with $e$ gives a lighter spanning tree — contradicting optimality.

''Cycle property:'' For any cycle $C$ in $G$, let $e$ be the unique maximum-weight edge in $C$. Then $e$ belongs to no MST of $G$.

''Matroid structure:'' Define the ''graphic matroid'' $M = (E, \mathcal{I})$ where $\mathcal{I}$ is the collection of acyclic edge subsets (forests). This satisfies the matroid axioms: (1) $\emptyset \in \mathcal{I}$, (2) subsets of independent sets are independent, (3) the exchange axiom holds. On any [[Matroid Theory|matroid]], the [[Greedy Algorithms|greedy algorithm]] optimally solves the weighted optimization problem, which is why Kruskal's algorithm works.

''Complexity:'' Kruskal's runs in $O(E \log E) = O(E \log V)$. Prim's with a Fibonacci heap achieves $O(E + V \log V)$. The fastest known deterministic algorithm (Chazelle, 2000) runs in $O(E \cdot \alpha(V))$ where $\alpha$ is the inverse Ackermann function — effectively linear.

!! MSTs in Machine Learning

''Single-linkage hierarchical [[Clustering|clustering]]'' is equivalent to computing the MST and then removing edges in order of decreasing weight. Each edge removal splits a cluster. The resulting dendrogram is exactly the MST sorted by edge weight. This connection means that efficient MST algorithms directly accelerate hierarchical clustering on large datasets.

''Network design and feature graphs:'' In graph-based semi-supervised learning, constructing a $k$-nearest-neighbor graph from data and then computing its MST identifies the "skeleton" of the data manifold. The MST preserves connectivity while removing redundant edges, giving a sparse graph that often captures cluster structure better than the full $k$-NN graph.

''TSP approximation:'' The MST provides a 2-approximation for the ''Traveling Salesman Problem'' (TSP) on metric spaces: since deleting any edge from an optimal TSP tour yields a spanning tree, the MST weight is a lower bound on the optimal tour. Doubling the MST and shortcutting gives a tour at most twice the optimal. Christofides' algorithm improves this to a 1.5-approximation by combining the MST with minimum-weight perfect matching.

''Borůvka in distributed ML:'' Borůvka's algorithm naturally parallelizes — each component independently selects its cheapest edge — making it ideal for computing MSTs on graphs distributed across multiple machines, a common setting in large-scale [[Graph Theory|graph analytics]].

!! When MSTs Break Down

''(1) Edge weights must be meaningful for MST to be useful.'' If the edge weights do not reflect true similarity or cost, the MST will faithfully find the cheapest tree for garbage distances. In high-dimensional spaces, distance metrics often become meaningless (the "curse of dimensionality"), and MST-based clustering degrades because all pairwise distances converge. Dimensionality reduction or learned metrics are needed first.

''(2) The MST is sensitive to outliers and noise.'' A single outlier point with a spuriously short edge to one cluster and a long edge to another can bridge clusters that should remain separate. Unlike $k$-means or spectral clustering, MST-based methods have no mechanism to "absorb" noise — every edge matters. Robust variants prune edges above a threshold, but choosing the threshold requires domain knowledge.

''(3) MST is undefined or ill-posed for directed, disconnected, or dynamic graphs.'' The classic MST assumes undirected, connected graphs with static weights. For directed graphs, the related problem (minimum-cost arborescence) requires entirely different algorithms (Edmonds/Chu-Liu). For dynamic graphs where edges change over time, recomputing the MST from scratch is expensive; online MST algorithms exist but add significant complexity.

!! Technical Details

''Uniqueness:'' If all edge weights are distinct, the MST is unique. With tied weights, multiple MSTs may exist, but all have the same total weight and the same //sorted// sequence of edge weights (the "weight spectrum" is unique).

''Relationship to shortest paths:'' The MST minimizes total edge weight but does //not// minimize the distance between any particular pair of vertices. The path between $u$ and $v$ in the MST is the ''minimax path'' — the path that minimizes the maximum edge weight encountered. This property is useful in image segmentation (Felzenszwalb-Huttenlocher algorithm).

''Matrix tree theorem:'' The number of spanning trees of a graph equals any cofactor of its [[Graph Theory|graph Laplacian]] $L$, connecting MSTs to spectral graph theory.

''Steiner trees:'' If only a subset $S \subset V$ must be connected (not all vertices), the problem becomes the ''Steiner tree problem'', which is NP-hard. The MST of the subgraph induced by $S$ gives a 2-approximation.

''Random MSTs:'' When edge weights are i.i.d. uniform on $[0,1]$, the expected MST weight of the complete graph $K_n$ converges to $\zeta(3) = \sum_{k=1}^{\infty} k^{-3} \approx 1.202$ as $n \to \infty$ (Frieze, 1985).

See also: [[Graph Theory|Graph Theory]], [[Greedy Algorithms|Greedy Algorithms]], [[Matroid Theory|Matroid Theory]], [[Clustering|Clustering]], [[Union-Find|Union-Find]].
