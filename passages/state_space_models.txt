! State Space Models
<img src="assets/state_space_models.svg" alt="State Space Models diagram" style="max-width:100%;margin:1em 0;">

Imagine you are tracking a submarine with sonar. You cannot see the sub directly — you only hear noisy pings. But you have a physics model of how submarines move: speed, heading, inertia. At every timestep you update your best guess of where the sub is by combining the physics prediction with the latest ping. That predict-then-correct loop — a ''state space model'' — is exactly what S4 and Mamba bring to sequence modelling. Instead of letting every token attend to every other token (the expensive <<link "Attention" "Attention">><</link>> mechanism in <<link "Transformers" "Transformers">><</link>>), these models maintain a compact hidden state that evolves through a ''linear recurrence'', processing sequences in linear time.

!! The Core Insight

A state space model borrows directly from <<link "Control Theory" "Control Theory">><</link>>. The continuous-time system is defined by four matrices:

$$h'(t) = \mathbf{A}\,h(t) + \mathbf{B}\,u(t), \quad y(t) = \mathbf{C}\,h(t) + \mathbf{D}\,u(t)$$

Here $h(t)$ is the hidden state, $u(t)$ is the input, and $y(t)$ is the output. The key discovery behind ''S4'' (Structured State Spaces for Sequence Modeling, Gu et al. 2021) is that if you initialise $\mathbf{A}$ with a special structure called the ''HiPPO matrix'' — designed to optimally compress the history of the input signal — and then discretise the system, the resulting model can remember dependencies over tens of thousands of timesteps. Classical <<link "RNN" "RNN">><</link>> architectures lose this signal to vanishing gradients; S4 sidesteps the problem entirely because its recurrence is ''linear''.

!! From Continuous to Discrete: The Dual View

The continuous system must be discretised for digital computation. Using a zero-order hold or bilinear transform with step size $\Delta$, you get discrete matrices $\bar{\mathbf{A}}, \bar{\mathbf{B}}$:

$$\bar{\mathbf{A}} = \exp(\Delta \mathbf{A}), \quad \bar{\mathbf{B}} = (\Delta \mathbf{A})^{-1}(\bar{\mathbf{A}} - \mathbf{I}) \cdot \Delta \mathbf{B}$$

The discrete recurrence $h_k = \bar{\mathbf{A}}\,h_{k-1} + \bar{\mathbf{B}}\,u_k$ can be unrolled into a ''global convolution'':

$$y = \bar{\mathbf{K}} * u, \quad \text{where } \bar{K}_i = \mathbf{C}\,\bar{\mathbf{A}}^i\,\bar{\mathbf{B}}$$

This is the dual view that makes S4 fast: during training, the convolution can be computed in $O(N \log N)$ via FFT. During inference, you switch to the recurrence form and process tokens one-by-one in $O(1)$ per step — something Transformers cannot do without caching all previous keys and values.

!! What the Math Is Actually Doing

The HiPPO matrix $\mathbf{A}$ is constructed so that the hidden state $h(t)$ stores optimal polynomial coefficients approximating the input history under a specific measure. For the ''HiPPO-LegS'' variant:

$$A_{nk} = -\begin{cases} (2n+1)^{1/2}(2k+1)^{1/2} & \text{if } n > k \\ n+1 & \text{if } n = k \\ 0 & \text{if } n < k \end{cases}$$

This gives $h(t)$ the property that its $n$-th component tracks the $n$-th Legendre polynomial coefficient of $u$ over $[0, t]$. The result is a hidden state that compresses arbitrarily long histories without any learned forgetting gate — the structure of $\mathbf{A}$ handles it by construction.

''Mamba'' (Gu & Dao, 2023) breaks the convolution shortcut deliberately. It makes $\mathbf{B}$, $\mathbf{C}$, and $\Delta$ ''input-dependent'' — functions of the current token. This means the system is no longer linear time-invariant, so you cannot use the FFT trick. Instead, <<link "Mamba" "Mamba">><</link>> uses a ''hardware-aware parallel scan'' algorithm on GPU, achieving $O(N)$ time with excellent memory efficiency. The payoff: the model can now selectively remember or forget information based on content, much like an LSTM gate, but within the SSM framework.

!! Practical Advantages Over Transformers

The computational profile is the headline. Self-<<link "Attention" "Attention">><</link>> costs $O(N^2)$ in sequence length; SSMs cost $O(N)$ or $O(N \log N)$. For a 16k-token document, that is the difference between 256 million attention operations and 16 thousand recurrence steps. This makes SSMs natural candidates for long-context tasks: genomics (DNA sequences of 100k+ bases), audio waveforms (16kHz signals over minutes), and long-document understanding.

Mamba-based models match or exceed Transformer performance at the same parameter count on language modelling benchmarks up to around 3B parameters. Inference is particularly efficient: generation requires only a constant-size state per layer, versus the growing key-value cache of autoregressive Transformers.

The architecture stacks SSM layers with normalization, residual connections, and gated MLPs, forming a ''Mamba block'' analogous to a Transformer block but without any attention sublayer.

!! When X Breaks Down

''Retrieval and precise copying.'' SSMs compress history into a fixed-size state vector. If a task requires retrieving an exact token from 10,000 steps ago — like looking up a specific name mentioned once in a long document — the compressed state may have discarded it. Transformers with full attention can look back directly. This is the fundamental information bottleneck of any fixed-width recurrence.

''In-context learning with arbitrary patterns.'' Transformers excel at in-context learning partly because attention can dynamically form arbitrary retrieval patterns at inference time. SSMs, even selective ones like Mamba, route information through structured linear dynamics that may struggle with tasks requiring ad-hoc, content-based lookup across distant positions. Hybrid architectures (Mamba + a few attention layers) often outperform pure SSMs for this reason.

''Scaling beyond 7B parameters.'' As of early 2025, the largest pure SSM language models trail the largest Transformers. Whether this is a fundamental limitation or simply reflects less engineering investment remains an open question, but the empirical gap at scale is real.

!! Technical Details

| Property | Detail |
|---|---|
| Core formulation | Linear time-invariant (S4) or time-varying (Mamba) state space model |
| State equation | $h_k = \bar{A}\,h_{k-1} + \bar{B}\,u_k;\; y_k = C\,h_k + D\,u_k$ |
| Training complexity | $O(N \log N)$ via FFT (S4) or $O(N)$ via parallel scan (Mamba) |
| Inference complexity | $O(1)$ per step (constant-size hidden state) |
| Key initialisation | HiPPO matrix (Legendre, Laguerre variants) |
| Selective mechanism | Mamba: $B$, $C$, $\Delta$ are functions of input $u_k$ |
| State dimension | Typically $N = 16$ to $64$ per channel |
| Discretisation | Zero-order hold (ZOH) or bilinear transform |
| Key papers | S4 (Gu et al., 2021), Mamba (Gu & Dao, 2023), S5 (Smith et al., 2023) |
| Strengths | Linear-time sequences, constant-memory generation, long-range dependencies |
| Weaknesses | Exact retrieval, in-context learning, scaling beyond ~7B |
| Relation to classical SSM | Identical formulation to <<link "Control Theory" "Control Theory">><</link>> state observers; HiPPO adds optimal history compression |
