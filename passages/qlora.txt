! QLoRA
<img src="assets/qlora.svg" alt="QLoRA pipeline diagram showing 4-bit NF4 quantization combined with LoRA adapters for memory-efficient fine-tuning" style="max-width:100%;margin:0.5em 0 1em;">

Imagine you want to renovate a mansion but can only afford a studio apartment's worth of tools. QLoRA is the trick that lets you compress the mansion's blueprints into a tiny filing cabinet (4-bit quantization), then make surgical upgrades using a lightweight toolkit (LoRA adapters) — all without ever needing to unfold the full-size blueprints. The result: you fine-tune a 65-billion-parameter language model on a single consumer GPU that would normally need a server rack.

!! The Core Insight

''Dettmers et al. (2023)'' showed that you can quantize a pretrained model down to 4-bit precision, freeze those quantized weights, and layer <<link "LoRA" "LoRA">><</link>> adapters on top — with //no measurable loss// in fine-tuning quality compared to full 16-bit fine-tuning. The key was not just any 4-bit format, but a new data type called ''NormalFloat4 (NF4)'' that is information-theoretically optimal for normally distributed weights.

QLoRA combines three innovations: (1) NF4 quantization that matches the weight distribution's quantiles, (2) ''double quantization'' that compresses the quantization constants themselves, and (3) ''paged optimizers'' that use NVIDIA unified memory to prevent out-of-memory crashes during gradient spikes.

Related topics: <<link "LoRA" "LoRA">><</link>>, <<link "Quantization" "Quantization">><</link>>, <<link "Fine-tuning" "Fine-tuning">><</link>>, <<link "Large Language Models" "Large Language Models">><</link>>, <<link "Transfer Learning" "Transfer Learning">><</link>>.

!! How NF4 Quantization Works

Standard <<link "Quantization" "Quantization">><</link>> maps floating-point values to a fixed grid of integers — say, 16 evenly spaced levels for 4 bits. But neural network weights are not uniformly distributed. They follow an approximately ''normal distribution'' centered at zero, with most values clustered near the mean and few values out in the tails.

NF4 exploits this by placing the 16 quantization levels at the ''quantiles'' of the standard normal distribution $N(0,1)$. More levels sit near zero (where most weights live) and fewer levels sit in the tails (where few weights live). Each block of 64 weights is normalized by an ''absmax'' scale factor $s = \max(|w_i|)$, then each normalized weight $w_i / s$ is snapped to the nearest NF4 level.

''Double quantization'' goes one step further: the FP32 scale factors $s$ themselves are quantized to FP8 using a second level of block-wise quantization (groups of 256 scale factors sharing one FP32 second-level constant). This saves an additional 0.37 bits per parameter — seemingly tiny, but at 65 billion parameters it frees roughly 3 GB of GPU memory.

!! What the Math Is Actually Doing

Given a pretrained weight matrix $W \in \mathbb{R}^{d \times k}$, QLoRA first quantizes it:
$$W_{\text{NF4}} = \text{nf4}\!\left(\frac{W}{s}\right), \quad s = \max(|W|)$$
where $\text{nf4}(\cdot)$ maps each value to the nearest of $2^4 = 16$ fixed quantile points $\{q_1, \ldots, q_{16}\}$ derived from the inverse CDF of $N(0,1)$.

During forward passes, the quantized weights are ''dequantized'' back to BF16 on the fly:
$$\tilde{W} = s \cdot \text{dequant}(W_{\text{NF4}})$$

The LoRA update adds trainable low-rank matrices. The effective computation for an input $x$ becomes:
$$y = \tilde{W}x + \frac{\alpha}{r} B A x$$
where $B \in \mathbb{R}^{d \times r}$, $A \in \mathbb{R}^{r \times k}$, rank $r \ll \min(d,k)$, and $\alpha$ is a scaling hyperparameter. Only $A$ and $B$ receive gradients — the frozen NF4 weights contribute to the forward pass but require no gradient storage.

The gradient for the LoRA parameters flows through the dequantized weights:
$$\frac{\partial \mathcal{L}}{\partial A} = \frac{\alpha}{r} B^\top \frac{\partial \mathcal{L}}{\partial y} x^\top, \quad \frac{\partial \mathcal{L}}{\partial B} = \frac{\alpha}{r} \frac{\partial \mathcal{L}}{\partial y} (Ax)^\top$$

Memory savings come from three sources: (i) storing base weights in 4 bits instead of 16 saves $\sim$75%, (ii) gradients exist only for the small LoRA matrices, and (iii) optimizer states (Adam's $m$ and $v$) are maintained only for LoRA parameters.

!! Practical Usage and Guanaco

The authors demonstrated QLoRA by training ''Guanaco'', a family of chatbot models fine-tuned from LLaMA. A Guanaco 65B model trained on a single 48 GB GPU for 24 hours reached ''97.8%'' of ChatGPT's performance level on the Vicuna benchmark — using only 0.2% trainable parameters.

''Typical hyperparameters:'' LoRA rank $r = 64$, $\alpha = 16$, applied to all linear layers in the <<link "Transformers" "Transformers">><</link>> (not just query and value projections as in standard LoRA). Using all layers was found to be important at 4-bit precision.

''Memory budget for a 65B model:'' approximately 41 GB for the NF4 base weights, plus ~2 GB for LoRA parameters and optimizer states in BF16, plus activations. This fits on a single A100-80GB or two consumer RTX 3090s.

''Paged optimizers:'' during training, long sequences can cause activation memory spikes that exceed available VRAM. QLoRA uses NVIDIA's CUDA unified memory to automatically page optimizer states between GPU and CPU memory, preventing OOM errors at the cost of minor throughput reduction during spikes.

!! When QLoRA Breaks Down

''Inference latency overhead:'' at serving time, the 4-bit weights must be dequantized to BF16 for every matrix multiply. This dequantization step adds computational overhead, making QLoRA-quantized models 10–20% slower at inference than natively BF16 models of the same size. For latency-critical deployments, you may need to merge the LoRA weights and requantize using a production-optimized scheme like GPTQ or AWQ.

''Quantization error propagation at extreme compression:'' NF4 is near-optimal for normally distributed weights, but some layers — particularly the first embedding layer and the final output projection — have weight distributions that deviate significantly from Gaussian. At 4-bit precision these layers accumulate quantization noise that LoRA cannot fully compensate for, occasionally causing degraded performance on tasks requiring precise numerical reasoning.

''Training instability with very low rank:'' when both the quantization precision //and// the LoRA rank $r$ are simultaneously pushed to their minimums (e.g., 4-bit with $r = 4$), the adapter lacks sufficient capacity to correct for quantization-induced errors. The training loss may plateau early or oscillate. In practice, QLoRA requires higher LoRA ranks ($r \geq 16$, often $r = 64$) than standard full-precision LoRA to compensate for the lossy base representation.

!! Technical Details

''Paper:'' Dettmers et al. 2023, "QLoRA: Efficient Finetuning of Quantized LLMs" (NeurIPS 2023).

''NF4 quantization levels:'' the 16 levels are computed as $q_i = \Phi^{-1}\!\bigl(\frac{2i-1}{32}\bigr)$ for $i = 1, \ldots, 16$, where $\Phi^{-1}$ is the inverse CDF of the standard normal. After normalization, zero is represented exactly (an asymmetric set: 8 negative values, zero, and 7 positive values).

''Block size:'' each block of 64 weights shares one FP32 absmax scale factor. With double quantization, groups of 256 such scale factors share one FP32 second-level constant, and each first-level constant is stored as FP8. Net cost: $4 + 32/64 + 8/64 + 32/(64 \times 256) \approx 4.63$ bits per parameter.

''Comparison:'' RTN (Round-to-Nearest) 4-bit quantization loses 1–2 perplexity points; NF4 loses $< 0.2$ perplexity points on LLaMA-7B through LLaMA-65B.

''Paged optimizer memory:'' uses cudaMallocManaged to allocate optimizer states. Pages are 2 MB. GPU-to-CPU transfer occurs only on memory pressure events.

''Guanaco results:'' Guanaco-33B outperforms all prior open-source chatbots on the Vicuna benchmark; Guanaco-7B (trainable in ~5 hours on a single GPU) outperforms a 26B Alpaca model.

''LoRA target modules:'' unlike standard LoRA which targets $W_Q$ and $W_V$ only, QLoRA applies adapters to all linear projections ($W_Q, W_K, W_V, W_O$, and the two feed-forward matrices) — this broader coverage is critical at 4-bit precision.
