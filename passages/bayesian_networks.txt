! Bayesian Networks

[img[assets/bayesian_networks.svg]]

Imagine you wake up sneezing. Is it the flu or allergies? You check: it is winter, your coworker was sick last week, and the pollen count is low. Without realizing it, you are running a ''Bayesian network'' in your head — combining multiple clues through a chain of cause-and-effect reasoning, where each piece of evidence updates your belief about what is actually going on. A Bayesian network formalizes this process as a graph: nodes are variables, arrows encode which variables directly influence which others, and attached to each node is a table describing //how much// parents influence their children.

!! The Core Insight

A Bayesian network is a ''directed acyclic graph'' (DAG) where each node represents a random variable and each directed edge represents a direct probabilistic dependency. The magic is in what the graph //leaves out//: missing edges encode ''conditional independencies''. If there is no arrow from node $A$ to node $B$, the network asserts that $A$ does not directly influence $B$ (though they may be indirectly related through other paths).

Each node $X_i$ carries a ''conditional probability table'' (CPT) specifying $P(X_i \mid \text{Parents}(X_i))$. The full joint distribution over all variables factors as a product of these local tables:

$$P(X_1, X_2, \ldots, X_n) = \prod_{i=1}^{n} P(X_i \mid \text{Parents}(X_i))$$

This factorization is the core computational trick. Instead of storing an exponentially large joint table, you store only the small local tables at each node. A network with $n$ binary variables and at most $k$ parents per node needs $O(n \cdot 2^k)$ parameters instead of $O(2^n)$. This is what makes Bayesian networks tractable for real-world problems. See <<link "Bayes Theorem" "Bayes Theorem">><</link>> for the foundational probability rule underlying all of this.

!! D-Separation and Conditional Independence

The graph structure encodes a rich set of ''conditional independence'' relationships through a criterion called ''d-separation''. Two variables $A$ and $B$ are d-separated given a set of observed variables $C$ if every path between them is "blocked." A path is blocked if it passes through a node that is either:

''Chain'' ($A \rightarrow M \rightarrow B$): Observing the middle node $M$ blocks information flow. If you know the mediator, the cause tells you nothing new about the effect.

''Fork'' ($A \leftarrow M \rightarrow B$): Observing the common cause $M$ blocks the path. Without conditioning on $M$, $A$ and $B$ appear correlated (they share a common cause), but once you know $M$, the correlation vanishes.

''Collider'' ($A \rightarrow M \leftarrow B$): Here the opposite happens. The path is //blocked by default// — $A$ and $B$ are independent. But if you observe $M$ (or any descendant of $M$), the path //opens// and $A$ and $B$ become dependent. This is the infamous "explaining away" effect: learning that one cause is present makes the other cause less likely, given that their shared effect occurred.

D-separation is complete: every conditional independence implied by the graph can be read off using d-separation, and every d-separation statement corresponds to a genuine independence in any distribution faithful to the graph. This connects deeply to <<link "Conditional Independence" "Conditional Independence">><</link>> in probability theory.

!! What the Math Is Actually Doing

The joint distribution factorizes according to the DAG structure:

$$P(X_1, \ldots, X_n) = \prod_{i=1}^{n} P(X_i \mid \text{Pa}(X_i))$$

where $\text{Pa}(X_i)$ denotes the parents of node $X_i$. This is the ''chain rule of probability'' applied in the topological order of the DAG, with conditional independence assumptions dropping terms.

''Inference'' means computing posterior probabilities given evidence $E = e$:

$$P(Q \mid E = e) = \frac{P(Q, E = e)}{P(E = e)} = \frac{\sum_{\text{hidden}} \prod_i P(X_i \mid \text{Pa}(X_i))}{\sum_{Q} \sum_{\text{hidden}} \prod_i P(X_i \mid \text{Pa}(X_i))}$$

For tree-structured networks, ''belief propagation'' computes exact marginals in $O(n)$ time by passing messages along edges:

$$m_{i \to j}(x_j) = \sum_{x_i} P(x_i \mid \text{Pa}(x_i)) \prod_{k \in \text{neighbors}(i) \setminus j} m_{k \to i}(x_i)$$

For general graphs, exact inference is NP-hard, so approximate methods like ''variational inference'' or ''MCMC sampling'' are used. The ''junction tree algorithm'' converts the DAG into a tree of cliques and runs exact message passing, but with cost exponential in the ''treewidth'' of the graph.

''Parameter learning'' with complete data uses maximum likelihood. For each CPT entry:

$$\hat{P}(X_i = x \mid \text{Pa}(X_i) = \pi) = \frac{\text{count}(X_i = x, \text{Pa}(X_i) = \pi)}{\text{count}(\text{Pa}(X_i) = \pi)}$$

With missing data, the ''EM algorithm'' iterates between inferring missing values (E-step) and re-estimating parameters (M-step).

!! Learning and Inference

''Structure learning'' — discovering the graph itself from data — is one of the hardest and most important problems. Three main approaches exist:

''Constraint-based methods'' (e.g., the PC algorithm) test pairs of variables for conditional independence, then orient edges based on detected v-structures (colliders). These scale well but are sensitive to errors in independence tests.

''Score-based methods'' search over possible graph structures, scoring each with metrics like ''BIC'' (Bayesian Information Criterion) or ''BGe'' (Bayesian Gaussian equivalent). The search space is super-exponential in the number of variables, so heuristics like greedy hill-climbing or dynamic programming over parent sets are essential.

''Hybrid methods'' combine both: use constraint-based tests to prune the search space, then apply score-based search over the reduced space.

A fundamental limitation is that different DAGs can encode the same set of conditional independencies — they belong to the same ''Markov equivalence class''. Without interventional data or strong prior knowledge, you cannot distinguish between them from observational data alone. This connects to the broader challenge of <<link "Causal Discovery" "Causal Discovery">><</link>>.

Bayesian networks are closely related to <<link "Probabilistic Graphical Models" "Probabilistic Graphical Models">><</link>>, which also include undirected models (Markov random fields). They also generalize <<link "Markov Chains" "Markov Chains">><</link>>, which are Bayesian networks with a simple linear chain structure.

!! When Bayesian Networks Break Down

''Intractable inference in dense graphs'': Exact inference is NP-hard in general, and the cost scales exponentially with ''treewidth'' — the size of the largest clique in the junction tree. Dense networks with many interconnected variables (treewidth > 20 or so) defeat exact methods, forcing reliance on approximate inference that may not converge or may give poor estimates. A medical diagnosis network with hundreds of interacting symptoms and diseases can easily exceed tractability limits.

''Structure learning from limited data'': With few observations relative to the number of variables, conditional independence tests have low statistical power and score-based methods overfit. The PC algorithm may fail to detect true edges or hallucinate false ones. Worse, the number of possible DAGs grows super-exponentially — for $n$ nodes, there are more than $2^{n(n-1)/2}$ possible graphs — making exhaustive search impossible beyond about 30 variables without strong structural priors.

''Assumption of causal sufficiency and no feedback'': Bayesian networks assume all relevant variables are included (no hidden confounders) and that relationships are acyclic (no feedback loops). In reality, many systems have unmeasured confounders — socioeconomic status affecting both health and behavior — and many real systems have cycles (predator-prey dynamics, economic feedback). Violating either assumption produces a model that misrepresents the true data-generating process, leading to incorrect inferences and dangerous overconfidence.

!! Technical Details

''History'': Judea Pearl introduced the modern framework of Bayesian networks in his 1988 book //Probabilistic Reasoning in Intelligent Systems//, unifying earlier work on influence diagrams and causal graphs.

''Markov blanket'': The ''Markov blanket'' of a node $X$ consists of its parents, children, and co-parents (other parents of its children). Given its Markov blanket, $X$ is conditionally independent of all other variables in the network. This minimal set is crucial for feature selection and local computation.

''Noisy-OR model'': A compact parameterization for nodes with many binary parents. Instead of $2^k$ parameters for $k$ parents, the noisy-OR uses $k$ parameters (one per parent), assuming each cause independently has a chance of producing the effect: $P(X = 0 \mid \text{Pa}) = \prod_{i: \text{Pa}_i = 1} (1 - p_i)$.

''Dynamic Bayesian networks'' (DBNs): Extend Bayesian networks to model temporal processes by "unrolling" the network across time steps. Each time slice is a Bayesian network, with inter-slice edges capturing temporal dependencies. Hidden Markov Models and Kalman filters are special cases of DBNs.

''Causal interpretation'': Pearl's ''do-calculus'' gives Bayesian networks a causal semantics. The ''intervention'' $do(X = x)$ differs from ''observation'' $X = x$: intervention removes all incoming edges to $X$, while observation does not. This distinction is the foundation of modern causal inference.

''Complexity'': Exact inference in Bayesian networks is NP-hard (by reduction from 3-SAT). Approximate inference (even with bounded error) is also NP-hard in the worst case, though practical algorithms work well for many real-world network structures.
