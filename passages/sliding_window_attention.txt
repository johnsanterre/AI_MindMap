! Sliding Window Attention

<img src="assets/sliding_window_attention.svg" alt="Diagram showing sliding window attention: banded attention matrix, receptive field growth across layers, and circular rolling KV cache buffer" style="width:100%;max-width:620px;display:block;margin:16px auto 24px auto;">

Imagine reading a book through a magnifying glass that only shows ''W'' words at a time. Each word can only see its W nearest neighbors directly — but because the magnifying glass slides forward with each word, information propagates through the full sequence across layers. After //k// layers, each token has indirect access to //k × W// tokens. The effective receptive field grows linearly with depth. This is ''sliding window attention'': a simple, powerful constraint that transforms the Transformer from a quadratic-cost architecture into one that can process arbitrarily long sequences with fixed memory.

!! The Core Insight

In standard <<link "self-attention" "Attention">><</link>>, every token attends to every other token — $O(L^2)$ complexity. Sliding window attention restricts each token to attend only to the ''W'' tokens immediately before it (or optionally $W/2$ tokens on each side in a symmetric window). This single constraint changes everything:

* ''Complexity'' drops from $O(L^2)$ to $O(L \times W)$ — linear in sequence length when $W$ is fixed
* ''KV cache'' is bounded: you only need to store $W$ key-value pairs per layer, not the entire sequence history
* ''Generation'' can run indefinitely with fixed memory — the cache acts as a rolling buffer that evicts the oldest entries

The key realization is that you don't //need// every token to see every other token in a single layer. Information can propagate through intermediate layers. A window of $W = 4096$ across 32 layers gives an effective receptive field of 131,072 tokens — far larger than most documents.

!! Receptive Field Growth

With window size $W$ and $L$ layers, the ''effective receptive field'' after all layers is $L \times W$ tokens. This is directly analogous to how <<link "convolutional neural networks" "Convolutional Neural Networks">><</link>> build large receptive fields from small kernels stacked in depth.

Consider ''Mistral-7B'': $W = 4096$ and 32 <<link "Transformer" "Transformers">><</link>> layers. At layer 1, each token sees 4,096 neighbors. At layer 2, each token's representation already encodes information from its neighbors' neighbors — an effective field of 8,192. By layer 32, the effective receptive field spans $32 \times 4096 = 131{,}072$ tokens.

This works because each layer's output is a //compressed summary// of its input window. When the next layer attends to that summary, it indirectly accesses everything that summary encoded. Information flows across windows through these intermediate representations — each layer extends the reach by another $W$ positions.

The critical caveat: this is an //effective// receptive field, not a guaranteed one. Information can degrade as it passes through many layers. Direct attention (as in full self-attention) is always stronger than indirect propagation for long-range dependencies.

!! What the Math Is Actually Doing

Standard self-attention computes attention over all $L$ positions. Sliding window attention restricts the key and value sets to a local window:

$$\text{Attn}_i = \text{softmax}\!\left(\frac{q_i \cdot K_{[i-W:i]}^\top}{\sqrt{d}}\right) \cdot V_{[i-W:i]}$$

Here $q_i$ is the query for token $i$, and $K_{[i-W:i]}$ and $V_{[i-W:i]}$ are the keys and values for the $W$ tokens in the window $[i - W, \ldots, i]$.

The ''KV cache memory'' per layer becomes:

$$\text{Memory} = O(W \times d)$$

This is //constant// regardless of total sequence length $L$. For the full model with $L_{\text{layers}}$ layers:

$$\text{Total KV cache} = O(W \times d \times L_{\text{layers}})$$

Compare this with full attention's KV cache of $O(L \times d \times L_{\text{layers}})$, which grows without bound as the sequence extends.

With <<link "Flash Attention" "Flash Attention">><</link>>, the sliding window is implemented via ''causal masking within tiles''. The attention computation is tiled into blocks, and the mask zeroes out entries outside the window band — achieving both memory efficiency (no materialization of the full $L \times L$ matrix) and computational efficiency (skipping tiles entirely outside the window).

!! Implementations and Variants

''Mistral (2023)'' uses pure sliding window attention with $W = 4096$ across all layers. Every layer sees exactly the same local window. This is the simplest implementation: a single hyperparameter controls the attention span.

''Gemma'' uses sliding window attention in ''alternating layers'' — some layers use local windowed attention while others use full attention. This hybrid approach captures both local patterns (windowed layers) and global dependencies (full layers) while reducing average cost.

''Longformer (2020)'' combines sliding window attention with ''global attention tokens''. Most tokens attend locally with window $W$, but designated tokens (e.g., the [CLS] token or task-specific tokens) attend to and are attended by //all// tokens. This gives $O(L \times W + L \times g)$ complexity where $g$ is the number of global tokens.

''StreamingLLM (2023)'' pairs the sliding window with ''attention sinks'' — keeping the first few tokens of the sequence (typically 4) permanently in the KV cache alongside the rolling window. These initial tokens accumulate disproportionate attention mass during generation and act as stabilizing anchors. Without them, perplexity spikes when the window slides past the beginning of the sequence.

The ''rolling buffer'' implementation uses a circular KV cache of size $W$. New key-value pairs overwrite the oldest entries at position $i \bmod W$. This means the cache never grows — it simply cycles, making infinite-length generation possible with fixed GPU memory.

!! When Sliding Window Breaks Down

''1. Long-range direct dependencies.'' If token 1 is crucial for predicting token 10,000 and $W = 4096$, the information from token 1 must propagate through approximately 3 intermediate layers before it can influence token 10,000. At each layer, the signal passes through a compression bottleneck — and if the information is too specific or subtle, it may be lost entirely. Tasks like copy-from-beginning (reproduce the first sentence at the end) or <<link "needle-in-a-haystack" "Benchmarks">><</link>> retrieval expose this weakness directly.

''2. Window size selection.'' Choosing $W$ is a consequential design decision with no universal answer. Too small a $W$ loses critical context — a model with $W = 512$ cannot directly access a function definition 1,000 tokens ago. Too large a $W$ removes the computational benefit, converging back toward full attention cost. The optimal $W$ is task-dependent: code generation benefits from larger windows than conversational text. There is no free lunch — the tradeoff between efficiency and context access is fundamental.

''3. Position encoding interaction.'' Sliding window attention interacts subtly with <<link "positional encodings" "Rotary Position Embeddings">><</link>>. Learned or absolute position encodings degrade for positions beyond the training sequence length, creating a hard extrapolation boundary. <<link "RoPE" "Rotary Position Embeddings">><</link>> extends better due to its relative nature, but still has extrapolation limits at extreme lengths. When the window slides to positions the model has never seen during training, attention patterns can become erratic — even if the relative distances within the window remain familiar.

!! Technical Details

''Key papers:''
* Jiang et al., 2023 — //Mistral 7B// — introduced pure sliding window attention with $W = 4096$ and rolling buffer KV cache in a production-grade open model
* Xiao et al., 2023 — //Efficient Streaming Language Models with Attention Sinks// (StreamingLLM) — discovered attention sink phenomenon and combined it with sliding window for stable infinite-length generation
* Beltagy et al., 2020 — //Longformer: The Long-Document Transformer// — sliding window plus global tokens for document-level tasks
