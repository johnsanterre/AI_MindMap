! System Identification

<img src="assets/system_identification.svg" alt="Diagram showing the system identification pipeline: input-output data from an unknown system feeds an identification algorithm that produces a learned model" style="width:100%;max-width:620px;display:block;margin:16px auto 24px auto;">

Imagine you buy a used car with no owner's manual. You don't know the engine specs, the gear ratios, or how the suspension responds to bumps. But you //can// drive it: press the gas, turn the wheel, hit the brakes, and observe what happens. After enough driving, you develop an internal model — "this car pulls left when braking," "it needs three seconds to reach highway speed." ''System identification'' is exactly this process made rigorous. Given a dynamic system you cannot open up and inspect, you apply known inputs, record the outputs, and fit a mathematical model that reproduces the observed behaviour. It is how engineers build the models that <<link "Control Theory" "Control Theory">><</link>> and <<link "Signal Processing" "Signal Processing">><</link>> need to work.

---

!! The Core Insight

Every controller, every filter, every predictor needs a ''model'' of the system it is working with. The <<link "Kalman Filter" "Kalman Filter">><</link>> needs state-space matrices. A PID controller needs to know the plant's transfer function. <<link "Optimal Control" "Optimal Control">><</link>> needs dynamics equations. But where do these models come from? Sometimes physics gives you the equations (first-principles modelling). More often, the system is too complex, too poorly understood, or too variable for physics alone. System identification fills the gap: it //learns// the model from data.

The idea is deceptively simple. You excite the system with a carefully chosen input signal $u(t)$, measure the output $y(t)$, choose a model structure (how many parameters, what form the equations take), and then find the parameter values $\theta$ that make the model's predicted output $\hat{y}(t|\theta)$ match the measured output as closely as possible. The result is a compact mathematical description of a system you may never fully understand from first principles — but one you can now simulate, predict, and control.

---

!! From Black Box to Mathematical Model

System identification methods fall along a spectrum from ''white-box'' (physics-derived equations with a few unknown constants) through ''grey-box'' (structure from physics, parameters from data) to ''black-box'' (structure and parameters both learned from data).

The classical ''parametric'' approach works like this. You choose a model structure — for linear time-invariant systems, this is typically an ''ARX'' (AutoRegressive with eXogenous input) model:

$$y(t) = a_1 y(t-1) + \cdots + a_n y(t-n) + b_1 u(t-1) + \cdots + b_m u(t-m) + e(t)$$

This says the current output depends linearly on past outputs, past inputs, and a noise term $e(t)$. The model orders $n$ and $m$ define how far back the memory reaches. Fitting the parameters $a_i, b_i$ is a <<link "Linear Regression" "Linear Regression">><</link>> problem — you can solve it with ordinary least squares.

More sophisticated structures include ''ARMAX'' (adds a moving-average noise model), ''Output-Error'' (models the noise separately from the dynamics), and ''Box-Jenkins'' (separate transfer functions for signal and noise). Each makes different assumptions about where the noise enters the system.

For ''nonparametric'' identification, you skip the model structure entirely. Frequency-response analysis applies sinusoidal inputs at different frequencies and measures the gain and phase shift at each — producing a ''Bode plot'' directly from data. Impulse-response estimation recovers how the system responds to a sharp kick. These methods make no structural assumptions but give less compact, less interpretable models.

---

!! What the Math Is Actually Doing

The workhorse is ''prediction error minimisation'' (PEM). Given a parametric model $\mathcal{M}(\theta)$ that predicts $\hat{y}(t|\theta)$, define the prediction error:

$$\varepsilon(t, \theta) = y(t) - \hat{y}(t|\theta)$$

The parameter estimate minimises a loss function over $N$ data points:

$$\hat{\theta}_N = \arg\min_\theta \frac{1}{N} \sum_{t=1}^{N} \ell\bigl(\varepsilon(t, \theta)\bigr)$$

For quadratic loss $\ell(\varepsilon) = \varepsilon^2$, this is least squares. For Gaussian noise, least squares is also the ''maximum likelihood'' estimate.

For linear models (ARX), the cost is quadratic in $\theta$ and the solution is closed-form:

$$\hat{\theta} = (\Phi^T \Phi)^{-1} \Phi^T Y$$

where $\Phi$ is the regression matrix of past inputs and outputs, and $Y$ is the vector of measured outputs. For nonlinear model structures, the cost surface is non-convex and you need iterative optimisation — gradient descent, Gauss-Newton, or Levenberg-Marquardt.

''Subspace methods'' take a different approach entirely. Instead of specifying a model structure and optimising parameters, they estimate the state-space matrices $(A, B, C, D)$ directly from input-output data using linear algebra — specifically, the <<link "SVD" "SVD">><</link>> of a block Hankel matrix of input-output data. The N4SID and MOESP algorithms are the standard implementations. They are non-iterative, require no initial guess, and naturally produce state-space models ready for controller design.

The ''persistency of excitation'' condition is critical: your input signal must be "rich enough" to probe all the system's modes. A constant input tells you nothing about dynamics. A single sinusoid reveals one frequency. A pseudo-random binary sequence (PRBS) or multisine excites a broad frequency range and is the standard choice.

---

!! System Identification in Practice

''Experiment design'' matters enormously. You must choose the input signal, the sampling rate, the experiment duration, and the operating conditions. A system identified around one operating point may behave completely differently elsewhere — linearisation is local. For aerospace and automotive applications, multisine signals with carefully chosen frequency content are standard.

''Model order selection'' — choosing $n$ and $m$ — is the bias-variance tradeoff in disguise. Too few parameters and the model cannot capture the true dynamics (underfitting). Too many and it fits noise (<<link "Overfitting" "Overfitting">><</link>>). Information criteria like AIC and BIC penalise complexity. <<link "Cross Validation" "Cross Validation">><</link>> on held-out data is the gold standard: if the model predicts well on data it was not trained on, you have the right complexity.

''Modern neural system identification'' replaces parametric model structures with neural networks — recurrent networks, temporal convolutional networks, or state-space-inspired architectures. These handle nonlinearities naturally but sacrifice interpretability and the guaranteed convergence of linear methods. They connect system identification directly to deep learning: the model is a neural network, the loss is prediction error, and training is backpropagation through time.

''Recursive identification'' updates the model online as new data arrives, using algorithms like recursive least squares (RLS) or the extended <<link "Kalman Filter" "Kalman Filter">><</link>>. This is essential for adaptive control, where the system's dynamics change over time and the model must track them.

---

!! When System Identification Breaks Down

''Insufficient excitation:'' If the input signal does not excite all the system's dynamic modes, the data contains no information about those modes and the identified model will be wrong in precisely the regimes that matter. A common failure: identifying a system during normal steady-state operation produces a model that is useless for predicting transient or fault behaviour. The fix is deliberate experiment design with broadband excitation, but in safety-critical systems (aircraft, nuclear plants) you may not be allowed to perturb the system enough.

''Nonlinearity and regime dependence:'' Linear identification methods assume the system obeys superposition. Real systems — chemical reactors, biological networks, turbulent flows — are deeply nonlinear. A linear model identified at one operating point can be dangerously wrong at another. Piecewise-linear models, Volterra series, and neural approaches help, but each brings its own complexity. The fundamental problem is that nonlinear systems have infinitely many possible structures, and choosing the right one requires domain knowledge that data alone cannot provide.

''Feedback confounds:'' When the system operates in closed loop — with a controller already adjusting the input based on the output — the input and output are correlated through the feedback path, not just through the system dynamics. Naive identification in closed loop produces biased estimates because the algorithm cannot distinguish the system's response from the controller's influence. Instrumental variable methods, indirect identification (identify the closed-loop transfer function and extract the plant), or careful experiment design with external perturbations are needed.

---

!! Technical Details

''Historical context:'' System identification emerged from control engineering in the 1960s, with foundational work by Lennart Ljung, Torsten Soderstrom, and Karl Johan Astrom. The textbook reference is Ljung, //System Identification: Theory for the User// (2nd ed., 1999). The MATLAB System Identification Toolbox, authored by Ljung, remains the standard software.

''Convergence:'' Under mild regularity conditions (the true system is in the model set, the input is persistently exciting of sufficient order, the noise is stationary), PEM estimates converge to the true parameters as $N \to \infty$ and are asymptotically efficient (achieve the Cramer-Rao lower bound).

''Model validation:'' After fitting, check that the residuals $\varepsilon(t, \hat{\theta})$ are white (uncorrelated). If they show structure — autocorrelation or cross-correlation with the input — the model has not captured all the dynamics. The ''whiteness test'' and ''cross-correlation test'' are standard diagnostics.

''Connections to ML:'' System identification //is// supervised learning where the target is a time series and the features include past values. ARX models are linear autoregressive models. Neural system identification is sequence-to-sequence learning. The entire field can be viewed as <<link "Linear Regression" "Linear Regression">><</link>> (for linear models) or nonlinear regression (for neural models) applied to dynamical systems.
