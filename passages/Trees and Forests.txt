! Trees and Forests

<img src="assets/trees_and_forests.svg" alt="Trees and Forests: rooted tree, binary tree, and forest of disconnected trees" style="width:100%;max-width:620px;display:block;margin:16px auto 24px auto;">

Think of a family tree. Every person has exactly one parent (except the ancestor at the top), there are no loops of ancestry, and you can trace a single unambiguous line from any person back to the root. A ''tree'' in mathematics captures exactly this structure: a connected graph with no cycles. Remove the requirement that everything be connected — allow several independent family trees side by side — and you have a ''forest''. These structures are among the most fundamental in all of computer science and mathematics, underpinning everything from file systems and parsing to [[Decision Trees|Decision Trees]] and [[Random Forests|Random Forests]] in machine learning.

!! The Core Insight

A tree on $n$ nodes has exactly $n - 1$ edges, and between any two nodes there is exactly //one// path. These three properties — ''connected'', ''acyclic'', and ''$n - 1$ edges'' — are equivalent: any two of them imply the third. This means a tree is the most economical way to connect $n$ nodes: remove any edge and the graph disconnects, add any edge and you create a cycle. A ''forest'' is simply an acyclic graph (possibly disconnected), which is the same as a disjoint union of trees. If a forest has $k$ connected components on $n$ total nodes, it has exactly $n - k$ edges.

This minimality is what makes trees so pervasive. They impose a strict hierarchy with no redundancy, which means algorithms on trees are typically simpler and faster than their general-graph counterparts — many problems that are NP-hard on graphs become linear-time on trees.

!! Rooted Trees, Unrooted Trees, and Binary Trees

An ''unrooted tree'' is simply a connected acyclic graph with no distinguished node. Choosing any node as a ''root'' imposes a parent-child hierarchy: every non-root node has exactly one parent (its neighbor on the path to the root) and zero or more children. The same unrooted tree can be rooted at any of its $n$ nodes, producing $n$ different rooted trees with different hierarchical structures.

A ''binary tree'' is a rooted tree where every node has at most two children, conventionally called the //left// child and //right// child. A ''full'' binary tree has every internal node with exactly two children. A ''complete'' binary tree fills every level except possibly the last, which is filled from left to right. A ''perfect'' binary tree is both full and complete — all leaves are at the same depth $d$, giving $2^{d+1} - 1$ total nodes.

Binary trees are the backbone of [[Binary Search Trees|Binary Search Trees]], heaps, expression trees, and Huffman coding. Their balanced variants (AVL trees, red-black trees) guarantee $O(\log n)$ search, insertion, and deletion.

!! Tree Traversals and Algorithms

Traversal means visiting every node in a systematic order. For rooted trees, the main strategies are:

''Depth-first search (DFS)'' explores as deep as possible before backtracking. It comes in three flavors for binary trees: //pre-order// (visit root, then left subtree, then right subtree), //in-order// (left, root, right — which yields sorted order for binary search trees), and //post-order// (left, right, root — useful for expression evaluation and deletion). DFS uses $O(h)$ stack space where $h$ is the tree height.

''Breadth-first search (BFS)'' visits nodes level by level, using a queue. It naturally finds shortest paths in unweighted trees and is used in level-order serialization.

For general graphs, DFS produces a ''DFS tree'' (or spanning tree), and BFS produces a ''BFS tree''. The edges not in the spanning tree reveal structure: ''back edges'' (DFS) indicate cycles, while ''cross edges'' (BFS) connect nodes at the same or adjacent levels.

!! What the Math Is Actually Doing

''Cayley's formula'' states that the number of labeled trees on $n$ nodes is $n^{n-2}$. For $n = 4$, there are $4^2 = 16$ distinct labeled trees. The proof via ''Prüfer sequences'' gives a bijection between labeled trees and sequences of length $n - 2$ with elements from $\{1, \ldots, n\}$.

The ''Prüfer sequence'' of a labeled tree is constructed iteratively: find the leaf with the smallest label, record its neighbor's label, remove the leaf, and repeat until two nodes remain. This produces a sequence $(a_1, a_2, \ldots, a_{n-2})$ where each $a_i \in \{1, \ldots, n\}$. The mapping is reversible: from any such sequence, you can reconstruct the unique tree. Since there are $n^{n-2}$ possible sequences, Cayley's formula follows.

For a rooted binary tree with $n$ internal nodes, the number of distinct shapes is the $n$-th ''Catalan number'':

$$C_n = \frac{1}{n+1}\binom{2n}{n} = \frac{(2n)!}{(n+1)!\,n!}$$

The first few values are $C_0 = 1, C_1 = 1, C_2 = 2, C_3 = 5, C_4 = 14$. Catalan numbers grow as $C_n \sim \frac{4^n}{n^{3/2}\sqrt{\pi}}$.

The ''height'' of a random binary tree with $n$ nodes is $O(\sqrt{n})$ on average, but a random binary //search// tree (built by random insertions) has expected height $O(\log n)$. Balanced BSTs guarantee worst-case $O(\log n)$ height by rebalancing after each operation.

!! Trees in Machine Learning

''Decision trees'' ([[Decision Trees|Decision Trees]]) recursively partition feature space by splitting on one feature at a time. Each internal node tests a condition, each branch represents an outcome, and each leaf assigns a class or value. The tree structure provides natural interpretability — you can read off the decision logic by tracing a root-to-leaf path.

''Random forests'' ([[Random Forests|Random Forests]]) are ensembles of decision trees trained on bootstrap samples with random feature subsets. The forest's prediction aggregates (majority vote or average) individual trees, reducing variance while maintaining low bias. The key insight is that decorrelated trees' errors cancel when averaged.

''Parse trees'' (syntax trees) represent the grammatical structure of sentences or code. In NLP, constituency parse trees decompose sentences hierarchically (sentence → noun phrase + verb phrase → ...), while dependency trees connect words directly by grammatical relations. Tree-structured neural networks (Tree-LSTMs, recursive neural networks) process these structures bottom-up, composing child representations into parent representations.

''Hierarchical clustering dendrograms'' are binary trees that represent nested cluster merges. Agglomerative clustering builds the tree bottom-up (each leaf is one data point, merging the two closest clusters at each step), while divisive clustering works top-down. Cutting the dendrogram at a chosen height produces a flat clustering at that granularity.

''Spanning trees'' ([[Minimum Spanning Trees|Minimum Spanning Trees]]) connect all nodes of a weighted graph with minimum total edge weight. Algorithms like Kruskal's and Prim's find them greedily. In ML, minimum spanning trees are used in single-linkage clustering and graph-based semi-supervised learning.

!! When Trees Break Down

''(1) Trees cannot represent shared structure or cycles.'' A tree forces every pair of nodes to have exactly one path between them. When the underlying structure involves sharing (as in directed acyclic graphs for computation graphs, or cycles in social networks), trees either duplicate subtrees (exponential blowup) or simply cannot represent the structure at all. [[Graph Theory|Graph Theory]] provides the necessary generalization.

''(2) Unbalanced trees degenerate to linear chains.'' A binary search tree built from sorted input has height $n$ instead of $\log n$, making search $O(n)$ — no better than a linked list. Similarly, decision trees grown without constraints overfit the training data, creating deep, irregular trees that memorize noise. Pruning, depth limits, and ensemble methods (random forests, gradient boosting) mitigate this.

''(3) Counting and enumeration blow up combinatorially.'' Cayley's formula gives $n^{n-2}$ labeled trees — for $n = 20$, that is over $10^{23}$ trees. Catalan numbers grow exponentially. This combinatorial explosion means exhaustive search over tree structures is infeasible for even moderate $n$, forcing greedy heuristics (greedy splitting in decision trees, greedy Prüfer decoding) or sampling approaches.

!! Technical Details

''Equivalent characterizations of a tree:'' A graph $G$ on $n$ nodes is a tree if and only if any of the following hold: (i) $G$ is connected and has $n-1$ edges; (ii) $G$ is acyclic and has $n-1$ edges; (iii) $G$ is connected and removing any edge disconnects it; (iv) $G$ is acyclic and adding any edge creates exactly one cycle; (v) every pair of vertices has a unique simple path.

''Kirchhoff's matrix tree theorem:'' The number of spanning trees of a graph $G$ equals any cofactor of the graph Laplacian $L = D - A$. Equivalently, $\tau(G) = \frac{1}{n}\prod_{i=2}^{n}\lambda_i$ where $\lambda_i$ are the nonzero eigenvalues of $L$.

''Euler tour technique:'' A rooted tree can be "flattened" into an array by recording each node when DFS enters and exits it. This gives a sequence of length $2n$ that reduces many tree queries (subtree sums, lowest common ancestor) to range queries on arrays, solvable in $O(1)$ with preprocessing.

''Tree isomorphism:'' Two unrooted trees are isomorphic if and only if they have the same ''center'' structure and matching rooted subtrees. Testing tree isomorphism is solvable in $O(n)$ time using canonical forms (AHU algorithm), unlike general graph isomorphism.

''Prüfer sequence properties:'' A node $v$ appears in the Prüfer sequence exactly $\deg(v) - 1$ times. Nodes with degree 1 (leaves) do not appear at all. This immediately implies Cayley's formula and gives a way to count trees with prescribed degree sequences.

See also: [[Graph Theory|Graph Theory]], [[Decision Trees|Decision Trees]], [[Binary Search Trees|Binary Search Trees]], [[Minimum Spanning Trees|Minimum Spanning Trees]], [[Random Forests|Random Forests]].
