! Bellman Equation
[img[assets/bellman_equation.svg]]

Imagine you are driving across the country and want to find the fastest route. You do not need to plan the entire trip from your driveway to the destination all at once. Instead, you can think about it one intersection at a time: from wherever you are right now, which turn gives you the best combination of immediate progress plus the best you could do from that next intersection onward? That recursive reasoning — "the value of where I am equals the best immediate payoff plus the value of where I end up" — is exactly what the ''Bellman Equation'' captures. It is the fundamental recursive decomposition that underlies nearly all of <<link "Reinforcement Learning" "Reinforcement Learning">><</link>> and <<link "Dynamic Programming" "Dynamic Programming">><</link>>.

!! The Core Insight

The Bellman Equation says something deceptively simple: //you can break any long sequential decision problem into a chain of one-step problems//. The total value of a state is the immediate reward you collect plus the discounted value of whatever state you transition to next. This means you never have to reason about the entire future at once. You only need to know (1) what reward you get right now and (2) how good the next state is. If every state correctly estimates its own value in terms of its successors, then the entire value function is self-consistent, and you have found the optimal strategy.

This ''principle of optimality'' — first articulated by Richard Bellman in the 1950s — transformed operations research, control theory, and eventually machine learning. It is the reason we can solve problems with billions of possible futures using tractable, iterative algorithms.

!! Optimality and Fixed Points

There are two flavors of the Bellman Equation that matter in practice:

The ''Bellman Expectation Equation'' describes the value of a state under a //fixed// policy $\pi$. It says: the value of state $s$ under policy $\pi$ is the expected immediate reward plus the discounted expected value of the next state, where expectations are taken over the actions chosen by $\pi$ and the transitions of the environment.

The ''Bellman Optimality Equation'' describes the value of a state under the //best possible// policy. Instead of averaging over actions, it takes the ''maximum'' — the action that yields the highest sum of immediate reward and future value.

A value function that satisfies the Bellman Optimality Equation is called a ''fixed point''. Once you find it, the optimal policy falls out immediately: at every state, just pick the action that achieves the max. Algorithms like <<link "Q-Learning" "Q-Learning">><</link>> and value iteration are essentially methods for converging to this fixed point through repeated application of the Bellman operator.

!! What the Math Is Actually Doing

The ''Bellman Expectation Equation'' for a policy $\pi$:

$$V^\pi(s) = \sum_a \pi(a|s) \left[ R(s,a) + \gamma \sum_{s'} P(s'|s,a)\, V^\pi(s') \right]$$

Here $V^\pi(s)$ is the value of state $s$ under policy $\pi$, $R(s,a)$ is the expected immediate reward, $\gamma \in [0,1)$ is the ''discount factor'' controlling how much we care about future rewards, and $P(s'|s,a)$ is the transition probability to state $s'$.

The ''Bellman Optimality Equation'' replaces the policy average with a max:

$$V^*(s) = \max_a \left[ R(s,a) + \gamma \sum_{s'} P(s'|s,a)\, V^*(s') \right]$$

For ''action-values'' (Q-values), the optimality equation becomes:

$$Q^*(s,a) = R(s,a) + \gamma \sum_{s'} P(s'|s,a) \max_{a'} Q^*(s',a')$$

This Q-form is what <<link "Q-Learning" "Q-Learning">><</link>> estimates directly, without needing a model of the transition dynamics $P$. The ''Bellman operator'' $\mathcal{T}$ defined by the right-hand side is a ''contraction mapping'' whenever $\gamma < 1$, which guarantees convergence to a unique fixed point by the Banach fixed-point theorem.

The ''temporal difference (TD) error'' used to drive learning is:

$$\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)$$

This is simply the difference between the Bellman target (right side) and the current estimate (left side). When $\delta_t = 0$ everywhere, the Bellman Equation is satisfied.

!! From Bellman to Deep RL

Classical Bellman-based methods — value iteration, policy iteration — require a complete table of state values or a known model of the environment. This works for <<link "Markov Decision Processes" "Markov Decision Processes">><</link>> with small state spaces but collapses when states number in the millions or are continuous.

Modern deep RL addresses this by ''approximating'' the value function or Q-function with neural networks. DQN (Deep Q-Network) famously used a convolutional network to approximate $Q^*(s,a)$ for Atari games, training by minimizing the squared Bellman error. The ''target network'' trick — using a slowly-updated copy of the network to compute the Bellman target — stabilizes training by preventing the target from shifting on every update.

<<link "Policy Gradient" "Policy Gradient">><</link>> methods take a different route: instead of solving the Bellman Equation explicitly, they optimize the policy directly. But even here, the Bellman Equation lurks in the background. The advantage function $A(s,a) = Q(s,a) - V(s)$, used in methods like PPO, relies on value estimates that satisfy (approximately) the Bellman Equation. Actor-critic methods maintain an explicit value function trained via TD learning — pure Bellman reasoning — alongside the policy.

!! When the Bellman Equation Breaks Down

''1. The curse of dimensionality.'' The Bellman Equation is exact, but solving it exactly requires iterating over all states and actions. When the state space is high-dimensional or continuous — robotic joint angles, pixel observations, financial portfolios — tabular methods are impossible. Function approximation introduces its own instabilities: the "deadly triad" of off-policy learning, function approximation, and bootstrapping can cause divergence rather than convergence.

''2. Non-stationarity and partial observability.'' The Bellman Equation assumes a stationary <<link "Markov Decision Processes" "Markov Decision Processes">><</link>> — transition probabilities and rewards do not change, and the current state contains all relevant information. In real-world settings where the environment shifts over time or the agent cannot observe the full state, the Bellman fixed point may not exist or may be constantly moving, making convergence unreliable.

''3. Discount factor sensitivity.'' The discount factor $\gamma$ controls the horizon of planning. Setting $\gamma$ close to 1 makes the agent far-sighted but slows convergence and amplifies estimation errors. Setting $\gamma$ too low makes the agent myopic. There is no universal recipe for choosing $\gamma$, and in problems with sparse, delayed rewards, no single discount factor may work well across all states.

!! Technical Details

''Contraction property:'' The Bellman optimality operator $\mathcal{T}$ satisfies $\|\mathcal{T}V_1 - \mathcal{T}V_2\|_\infty \leq \gamma \|V_1 - V_2\|_\infty$. This guarantees that value iteration converges at a geometric rate of $\gamma$ per iteration.

''Value iteration:'' Set $V_0$ arbitrarily. Compute $V_{k+1}(s) = \max_a [R(s,a) + \gamma \sum_{s'} P(s'|s,a) V_k(s')]$ for all $s$. Converges to $V^*$ as $k \to \infty$.

''Policy iteration:'' Alternate between (1) policy evaluation — solving the linear Bellman expectation equation for the current policy, and (2) policy improvement — setting $\pi(s) = \arg\max_a [R(s,a) + \gamma \sum_{s'} P(s'|s,a) V^\pi(s')]$. Converges in a finite number of iterations for finite MDPs.

''Linear system form:'' For a fixed policy $\pi$, the Bellman expectation equation is a system of $|S|$ linear equations: $V^\pi = R^\pi + \gamma P^\pi V^\pi$, which has closed-form solution $V^\pi = (I - \gamma P^\pi)^{-1} R^\pi$. Matrix inversion costs $O(|S|^3)$, feasible only for small state spaces.

''Bellman residual:'' The norm $\|V - \mathcal{T}V\|$ measures how far a value function is from satisfying the Bellman Equation. Minimizing the mean-squared Bellman error is one objective for fitted value methods, though it has known issues with double sampling.

''Connection to linear programming:'' The optimal value function $V^*$ is the component-wise smallest $V$ satisfying $V(s) \geq R(s,a) + \gamma \sum_{s'} P(s'|s,a) V(s')$ for all $s, a$. This can be formulated as a linear program.
