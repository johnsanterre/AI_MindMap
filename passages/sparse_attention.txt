! Sparse Attention

<img src="assets/sparse_attention.svg" alt="Sparse attention pattern matrices comparing full, local window, strided, and Longformer-style combined patterns" style="width:100%;max-width:620px;display:block;margin:16px auto 24px auto;">

Imagine you're in a conference room with 1,000 people. You don't need to listen to everyone equally -- you pay close attention to the five people sitting nearest you and the speaker at the podium, while tuning out the hundreds of murmured side conversations you can barely hear. ''Sparse attention'' applies exactly this idea to <<link "Transformers" "Transformers">><</link>>. In standard self-attention, every token attends to every other token, which costs $O(L^2)$ computation for a sequence of length $L$. Sparse attention restricts each token to attending only to a carefully chosen subset of other tokens -- nearby neighbors, periodic landmarks, or learned targets -- cutting the cost to $O(L\sqrt{L})$ or even $O(L)$. This is what makes it possible to process documents with tens of thousands of tokens where full attention would be computationally impossible.

!! The Core Insight

Full <<link "Attention" "Attention">><</link>> computes a dense $L \times L$ matrix of attention weights. For a sequence of 16,000 tokens, that's 256 million entries -- and it grows quadratically. Double the sequence length and you quadruple the cost. This is the fundamental bottleneck that limits how much context a transformer can see.

But here's the key observation: //most of those 256 million attention weights are near-zero anyway//. Empirical studies show that the attention matrix is naturally sparse -- tokens overwhelmingly attend to nearby tokens and a handful of globally important tokens (like [CLS] or punctuation). The vast majority of token pairs have negligible attention weights.

Sparse attention exploits this by defining a ''sparsity pattern'' -- a predetermined set of token pairs that are allowed to attend to each other. Everything outside the pattern is simply never computed. The art lies in choosing patterns that capture the important dependencies while discarding the irrelevant ones. The main families are local windows (nearby tokens), strided patterns (periodic long-range connections), global tokens (designated tokens that see everything), random connections, and learned routing.

!! Sparse Attention Patterns

''Local / sliding window attention'' is the simplest pattern: each token attends only to its $W$ nearest neighbors. This captures the intuition that most dependencies in language are local -- a word usually depends most on the words immediately around it. Mistral and Mixtral use sliding window attention as their default mechanism. Cost: $O(LW)$.

''Strided attention'' adds long-range reach by having each token attend to every $k$-th token in addition to its local window. Think of it like reading a book by looking at every nearby word //plus// glancing at the first word of every paragraph. Child et al. (2019) introduced this in the original Sparse Transformer, combining local and strided heads to achieve $O(L\sqrt{L})$ complexity.

''Global tokens'' are designated positions that attend to (and are attended by) every token in the sequence. <<link "Longformer" "Longformer">><</link>> (Beltagy et al. 2020) combines a local sliding window with a small number of global tokens -- for example, the [CLS] token in classification tasks. This gives every token an indirect path to every other token through the global positions.

''Random attention'' supplements local and global patterns with a random subset of connections. ''BigBird'' (Zaheer et al. 2020) uses local + global + random and proves this combination is a universal approximator of sequence functions -- it is Turing-complete, matching the theoretical power of full attention.

''Hash-based attention'' takes a different approach entirely. The ''Reformer'' (Kitaev et al. 2020) uses <<link "Locality-Sensitive Hashing" "Locality-Sensitive Hashing">><</link>> (LSH) to group tokens with similar query/key vectors into the same bucket, then restricts attention to within buckets. Tokens only attend to others that would have had high attention weights anyway. Cost: $O(L \log L)$.

''Learned routing'' goes further by training the model to decide which tokens should attend to which. Routing Transformers learn a routing function that clusters tokens dynamically, adapting the sparsity pattern to the input rather than using a fixed pattern.

!! What the Math Is Actually Doing

Standard self-attention computes the full $L \times L$ attention matrix:

$$A = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d}}\right)$$

where $Q, K \in \mathbb{R}^{L \times d}$. The output is $\text{Attention}(Q, K, V) = AV$. Computing $QK^\top$ alone requires $O(L^2 d)$ operations.

Sparse attention defines a ''sparsity pattern'' $\mathcal{S} \subseteq \{1, \ldots, L\} \times \{1, \ldots, L\}$ and computes attention weights only for pairs $(i, j) \in \mathcal{S}$:

$$A_{ij} = \begin{cases} \frac{\exp(q_i^\top k_j / \sqrt{d})}{\sum_{(i,j') \in \mathcal{S}} \exp(q_i^\top k_{j'} / \sqrt{d})} & \text{if } (i,j) \in \mathcal{S} \\ 0 & \text{otherwise} \end{cases}$$

For a ''local window'' of size $W$: $\mathcal{S} = \{(i, j) : |i - j| \leq W/2\}$, giving $|\mathcal{S}| = LW$, so the cost is $O(LW)$.

For ''Longformer'' with window $W$ and $g$ global tokens: $|\mathcal{S}| = L(W + g)$, so cost is $O(L(W + g))$. Since $W$ and $g$ are constants independent of $L$, this is ''linear in sequence length''.

For ''Reformer LSH attention'': tokens are hashed into $b$ buckets of average size $L/b$. Attention within each bucket is $O((L/b)^2)$, and there are $b$ buckets, giving total cost $O(L^2/b)$. With $b = L / \log L$, this yields $O(L \log L)$.

The key mathematical result from ''BigBird'': the combination of local + global + random attention with $O(L)$ total edges can approximate any continuous sequence-to-sequence function to arbitrary precision. This is proven using the theory of random graphs -- the random edges ensure the graph has $O(\log L)$ diameter, meaning information can flow between any two tokens in logarithmically many steps.

!! Sparse Attention in Practice

''Longformer'' (Beltagy et al. 2020) was one of the first practical sparse attention models. It uses a sliding window of 256-512 tokens per layer, with task-specific global tokens (e.g., [CLS] for classification, question tokens for QA). It processes documents up to 4,096 tokens and achieves strong results on long-document tasks like <<link "TriviaQA" "TriviaQA">><</link>> and WikiHop.

''BigBird'' (Zaheer et al. 2020) combines local windowed attention, a small number of random attention connections, and global tokens. It matches or exceeds full-attention <<link "BERT" "BERT">><</link>> on several benchmarks while handling sequences up to 4,096 tokens. Its theoretical contribution -- proving the Turing-completeness of sparse attention -- was equally influential.

''Reformer'' (Kitaev et al. 2020) uses LSH attention to achieve $O(L \log L)$ complexity. It also introduces reversible residual layers to reduce memory from $O(L \cdot n_{\text{layers}})$ to $O(L)$. Reformer can handle sequences up to 64K tokens on a single accelerator.

''Mistral'' and ''Mixtral'' use sliding window attention as a core architectural choice. With a window size of 4,096 tokens and 32 layers, information can theoretically propagate across $4096 \times 32 = 131,072$ tokens through successive layers, even though each individual attention operation is local. This is combined with <<link "KV Cache" "KV Cache">><</link>> rolling to keep memory bounded.

Modern frontier models like GPT-4 and Claude likely employ hybrid strategies -- full attention at shorter contexts transitioning to sparse or approximate attention patterns as context length grows, combined with <<link "Flash Attention" "Flash Attention">><</link>> for hardware efficiency.

!! When Sparse Attention Breaks Down

''1. Information loss from missed dependencies.'' No fixed sparsity pattern is universally optimal. If a critical long-range dependency falls outside the pattern -- say, a pronoun at position 8,000 that refers to a noun at position 200, and those positions aren't connected by local, strided, or global links -- the model simply cannot learn that dependency. The pattern designer must guess which connections matter, and they will sometimes guess wrong. Tasks requiring arbitrary long-range reasoning (e.g., multi-hop QA over very long documents) are especially vulnerable.

''2. Hardware inefficiency from irregular memory access.'' GPUs are optimized for dense, regular computation. Sparse attention patterns create irregular memory access that can severely underutilize hardware. The theoretical FLOP savings from sparsity often don't translate to proportional wall-clock speedups. This is why <<link "Flash Attention" "Flash Attention">><</link>> -- which keeps //dense// attention but optimizes memory access patterns -- can be //faster// than sparse attention for moderate sequence lengths (up to ~8K tokens on modern GPUs). Sparse attention only wins when sequences are genuinely very long.

''3. Training-inference mismatch.'' A model trained with full attention learns to use the full attention pattern. Switching to sparse attention at inference time -- even a theoretically equivalent sparse pattern -- often causes significant quality degradation because the model has learned to rely on connections that sparse attention removes. Models must be trained with the same sparse pattern they'll use at inference, which means committing to the pattern early and accepting its limitations throughout training. Fine-tuning a full-attention model to use sparse attention requires careful adaptation.

!! Technical Details

The original ''Sparse Transformer'' (Child et al. 2019) introduced factorized sparse attention, splitting attention heads into local and strided patterns. This achieved $O(L\sqrt{L})$ complexity and was applied to autoregressive image and text generation with sequences up to 12,288 tokens.

''Complexity comparison'' at $L = 16{,}384$, $d = 64$:
- Full attention: $L^2 = 268M$ pairs
- Local window ($W = 256$): $LW = 4.2M$ pairs (64x reduction)
- Longformer ($W = 256$, $g = 8$): $L(W + g) = 4.3M$ pairs
- Reformer LSH: $L \log L \approx 229K$ bucket operations

''Connection to linear attention'': sparse attention reduces the //number// of pairs computed but still uses softmax attention within the sparse pattern. <<link "Linear Attention" "Linear Attention">><</link>> takes a different approach, replacing the softmax kernel to enable $O(Ld^2)$ computation -- linear in $L$ but quadratic in head dimension $d$.

''Connection to state-space models'': <<link "Mamba" "Mamba">><</link>> and other SSMs avoid the attention matrix entirely, achieving $O(L)$ complexity through recurrent computation. They can be viewed as an extreme form of structured sparsity where the "attention pattern" is constrained to a specific parametric form.
