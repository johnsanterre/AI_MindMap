! Prefix Tuning
<img src="assets/prefix_tuning.svg" alt="Prefix tuning diagram showing trainable prefix vectors prepended to frozen transformer layers" style="max-width:100%;margin:0.5em 0 1em;">

Instead of rewriting an employee's entire job description (full fine-tuning), you give them a short briefing memo that reframes how they approach every task (prefix tuning). The briefing is small and task-specific; the employee's deep knowledge stays completely unchanged.

!! The Core Insight

''Li & Liang (2021)'' introduced prefix tuning as a parameter-efficient alternative to full fine-tuning. The idea: prepend $k$ trainable "virtual token" embeddings — the //prefix// — to every transformer layer's key-value pairs. Only the prefix parameters are updated during training; the rest of the LLM is frozen solid.

The result is striking: prefix tuning achieves performance comparable to full fine-tuning on natural language generation tasks using only ''0.1% of the model's parameters''. Unlike input-only prompt tuning, prefix vectors inject information through the attention mechanism at //every// layer, giving them far more expressive power.

Related approaches: [[Fine-tuning]], [[Instruction Tuning]], [[LoRA]], [[Large Language Models]], [[Transformers]], [[Attention]].

!! Prefix vs Prompt Tuning vs LoRA

Three closely related PEFT methods are often confused:

''Prompt Tuning (Lester 2021)'' — append soft (continuous) tokens only to the input layer. Simplest approach, fewest parameters, least expressive.

''Prefix Tuning (Li & Liang 2021)'' — prepend trainable vectors to the key and value matrices at //all// layers. More expressive because every attention layer can be guided. The prefix lives in the K-V cache, not just the input stream.

''LoRA (Hu 2022)'' — inject trainable low-rank matrices $\Delta W = AB$ into the weight matrices themselves. Most popular today due to simpler optimization and strong empirical results.

''P-Tuning v2 (Liu 2022)'' — essentially prefix tuning reparameterized; confirms that depth of injection (all layers vs input only) is the key driver of quality.

Key conceptual difference: prefix tuning shapes what the model //attends to// (via K and V); LoRA changes how the model //transforms// the input (via weight updates).

!! What the Math Is Actually Doing

Standard self-attention at each layer computes:
$$\text{Attn}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d}}\right)V$$
where $Q$, $K$, $V$ are derived from input tokens.

Prefix tuning prepends $k$ trainable prefix matrices $P_K \in \mathbb{R}^{k \times d}$ and $P_V \in \mathbb{R}^{k \times d}$ to the key and value tensors at every layer:
$$\text{Attn}(Q,\,[P_K;\,K],\,[P_V;\,V])$$
where $[;]$ denotes concatenation along the sequence dimension. The original tokens' queries can now attend to prefix keys and retrieve prefix values — meaning the prefix steers what information gets surfaced, at every depth.

''Reparameterization trick:'' directly optimizing prefix embeddings is numerically unstable. Li & Liang instead train a small MLP: $P = \text{MLP}_\theta(P')$. After training, the MLP is discarded and only the resulting $P$ is stored and swapped at inference.

!! When to Use Prefix vs LoRA

Prefix tuning suits ''generation tasks'' (summarization, translation, dialogue) and scenarios where you want to serve many tasks from a single frozen model by swapping lightweight prefix vectors — no model reload required, just a vector swap.

LoRA tends to win on ''instruction following, classification, and tasks requiring deep behavioral change'', and its training is more stable. LoRA also offers a flexible rank hyperparameter; prefix tuning's expressivity scales with $k$ (longer prefix = more compute at inference).

Practical verdict: LoRA has largely displaced prefix tuning in production pipelines, but prefix tuning's legacy is significant — it demonstrated that PEFT was viable and inspired a wave of soft-prompt research.

!! When Prefix Tuning Breaks Down

''Optimization instability:'' directly optimizing raw prefix embeddings frequently diverges. The reparameterization through an MLP is necessary, adding training complexity.

''Fixed expressivity ceiling:'' the prefix length $k$ is chosen before training. If a task requires more adaptive capacity than $k$ vectors can provide, the only fix is increasing $k$ — which raises inference-time memory and compute proportionally.

''Task interference across layer types:'' some tasks involve structural transformations (e.g., complex reasoning chains) that cannot be naturally expressed as modifications to the attention's key-value lookup. Prefix tuning cannot alter how the model computes its queries or applies its feed-forward sublayers.

!! Technical Details

''Papers:'' Li & Liang 2021 ("Prefix-Tuning: Optimizing Continuous Prompts for Generation"); Lester et al. 2021 ("The Power of Scale for Parameter-Efficient Prompt Tuning"); Liu et al. 2022 ("P-Tuning v2").

''Prefix length:'' $k$ typically ranges from 10 to 100 tokens per layer. Longer prefixes improve quality up to a point, then plateau.

''Memory overhead per task:'' $k \times d \times 2 \times L$ floats (K and V matrices across $L$ layers). For GPT-2 medium ($d=1024$, $L=24$, $k=100$): ~5M parameters — a tiny fraction of 345M total.

''Multi-task serving:'' a single frozen LLM can serve $N$ tasks simultaneously by loading $N$ prefix sets; switching tasks requires only swapping the cached prefix vectors.

''Benchmark result:'' prefix-tuned T5-large matches full fine-tuning on XSUM summarization and other NLG tasks at 0.1% parameter cost.
