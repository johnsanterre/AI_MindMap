! 3D Point Clouds

<img src="assets/point_clouds.svg" alt="Diagram showing PointNet architecture: raw point cloud to per-point MLP with shared weights to symmetric max pooling to classification and segmentation outputs" style="width:100%;max-width:620px;display:block;margin:16px auto 24px auto;">

Imagine you are blindfolded and someone hands you a bag of marbles, each tagged with its position in a room. You can reach in and feel every marble, but they have no fixed order — shake the bag and the marbles rearrange, yet the room they describe stays the same. A ''3D point cloud'' is exactly this: a set of $(x, y, z)$ coordinates captured by LiDAR sensors, depth cameras, or photogrammetry, representing the surface geometry of a scene. The fundamental challenge is that unlike images (which have a fixed pixel grid) or text (which has a fixed word order), point clouds are //unordered sets//. Any architecture that processes them must produce the same output regardless of how the points are shuffled — a property called ''permutation invariance''. PointNet (Qi et al. 2017) cracked this problem with a deceptively simple idea: process each point independently through a shared neural network, then collapse the results with a symmetric function like max pooling.

---

!! The Core Insight

The fundamental barrier to processing point clouds with standard neural networks is the ''ordering problem''. A set of $N$ points has $N!$ possible orderings. If you naively flatten the points into a vector and feed it to a fully connected network, every permutation of the same point set would produce a different output — the network would need to memorize all $N!$ orderings to learn a single shape. For 1,000 points, that is more orderings than atoms in the observable universe.

PointNet solves this with a ''universal approximation theorem for set functions'': any continuous function on a set can be approximated by $f(\{x_1, \ldots, x_N\}) = \gamma \left( \underset{i=1,\ldots,N}{\text{MAX}} \; h(x_i) \right)$, where $h$ is a per-point function (implemented as a shared MLP), MAX is an element-wise maximum across all points (a symmetric function), and $\gamma$ is a downstream network. Because MAX is symmetric — $\max(a, b) = \max(b, a)$ — the entire pipeline is permutation invariant by construction. No sorting, no canonical ordering, no graph construction needed.

This stands in contrast to the two earlier approaches to 3D data: ''voxelization'' (discretizing space into a 3D grid and applying 3D <<link "CNNs" "CNN">><</link>>), which wastes computation on empty space and loses fine detail, and ''mesh-based methods'' (constructing triangulated surfaces and applying <<link "Graph Neural Networks" "Graph Neural Network">><</link>>), which require a topology that raw sensor data does not provide.

---

!! From Points to Hierarchical Features

PointNet treats every point identically — it has no notion of local neighborhoods, so it cannot capture fine-grained geometric patterns the way a <<link "CNN" "CNN">><</link>> captures local textures in images. ''PointNet++'' (Qi et al. 2017b) fixes this with a hierarchical strategy that mirrors the receptive field growth in convolutional networks.

The architecture introduces three key operations. First, ''farthest point sampling'' selects a subset of centroids spread evenly across the point cloud. Second, ''ball query grouping'' gathers the $k$ nearest neighbors within a radius around each centroid, forming local patches. Third, a ''mini-PointNet'' is applied to each local patch to extract a local feature vector. This process repeats — the local features become a new, smaller point cloud, which is again sampled, grouped, and processed — building a hierarchy from fine local geometry to coarse global structure.

This hierarchical design captures both the micro-texture of a surface (the curvature of a fender) and the macro-shape of an object (the overall silhouette of a car), much like how convolutional layers in image networks capture both edges and high-level shapes.

Modern point cloud architectures have pushed further. ''Point Transformer'' (Zhao et al. 2021) applies <<link "Attention" "Attention">><</link>> mechanisms to local point neighborhoods, allowing each point to dynamically weight its neighbors based on learned similarity rather than fixed distance thresholds. ''DGCNN'' constructs a dynamic $k$-nearest-neighbor graph in feature space (not just coordinate space) at every layer, allowing the effective topology to evolve as the network learns.

!! What the Math Is Actually Doing

A point cloud is a set $\mathcal{P} = \{x_1, x_2, \ldots, x_N\}$ where each $x_i \in \mathbb{R}^3$ (or $\mathbb{R}^d$ if additional features like color or normals are included). A ''set function'' $f: 2^{\mathbb{R}^d} \to \mathbb{R}^k$ must satisfy:

$$f(\{x_{\pi(1)}, x_{\pi(2)}, \ldots, x_{\pi(N)}\}) = f(\{x_1, x_2, \ldots, x_N\})$$

for any permutation $\pi$. PointNet's approximation theorem states that any continuous set function on a compact set can be approximated as:

$$f(\mathcal{P}) = \gamma \left( \underset{i=1,\ldots,N}{\text{MAX}} \; h(x_i) \right)$$

where $h: \mathbb{R}^d \to \mathbb{R}^K$ is a continuous function and $\gamma: \mathbb{R}^K \to \mathbb{R}^k$ is a continuous function. The shared MLP implements $h$, lifting each 3D point into a high-dimensional feature space ($K = 1024$ in the original paper). The element-wise MAX collapses $N$ feature vectors into one ''global feature'' vector $g \in \mathbb{R}^K$:

$$g_j = \max_{i=1,\ldots,N} h(x_i)_j \quad \text{for } j = 1, \ldots, K$$

For ''segmentation'' (per-point labeling), the global feature $g$ is concatenated back to each per-point feature $h(x_i)$, giving each point both local and global context:

$$\text{output}_i = \phi\big([h(x_i) \,;\, g]\big)$$

where $\phi$ is another MLP and $[\cdot \,;\, \cdot]$ denotes concatenation.

PointNet also includes a ''T-Net'' — a mini-network that predicts a $3 \times 3$ (or $d \times d$) transformation matrix to align the input points, providing robustness to rigid transformations. A regularization loss encourages this matrix to stay near orthogonal:

$$\mathcal{L}_{\text{reg}} = \|I - AA^T\|_F^2$$

!! Applications in Autonomous Driving and Robotics

''Autonomous driving'' is the largest commercial application of point cloud processing. LiDAR sensors on self-driving cars produce hundreds of thousands of points per frame, and the perception stack must detect vehicles, pedestrians, cyclists, and road boundaries in real time. Architectures like VoxelNet, PointPillars, and CenterPoint combine point cloud processing with 2D detection heads to achieve this. PointPillars, for example, groups points into vertical columns (pillars), applies a PointNet-like encoder to each pillar, scatters the results onto a 2D bird's-eye-view grid, and runs a standard 2D <<link "CNN" "CNN">><</link>> detection head — balancing accuracy with the speed needed for real-time operation.

''Robotics manipulation'' uses point clouds for grasp planning. Given a depth camera view of objects on a table, the robot constructs a point cloud, segments individual objects, estimates their 6-DOF pose, and plans grasp points. PointNet++ and its variants enable end-to-end grasp prediction directly from raw point clouds without hand-crafted geometric features.

''3D reconstruction and mapping'' combines point clouds across multiple viewpoints to build dense scene representations. This intersects with <<link "NeRF" "NeRF">><</link>> and other neural scene representations, where point clouds serve as initialization or supervision for volumetric rendering pipelines.

''Medical imaging'' applies point cloud methods to organ surfaces extracted from CT or MRI scans, enabling shape analysis for surgical planning and disease tracking without the memory overhead of full volumetric processing.

!! When X Breaks Down

''Density variation defeats uniform architectures:'' Real-world LiDAR point clouds are highly non-uniform — dense near the sensor, sparse at distance. A car 5 meters away might have 10,000 points; the same car at 80 meters might have 20. PointNet and its variants struggle with this because the max-pooling aggregation is dominated by dense regions, and ball-query grouping in PointNet++ may find too few neighbors in sparse areas. Architectures must explicitly handle multi-scale density variation, and no current method fully solves this — distant objects remain the primary failure mode in autonomous driving perception.

''Max pooling discards fine structure:'' The global max pool in PointNet collapses $N$ points into a single vector by keeping only the maximum value in each feature dimension. This means that the network's representation of an entire object depends on only a small ''critical set'' of points (roughly $K$ points out of $N$, where $K$ is the feature dimension). Two objects that differ only in regions outside the critical set will produce identical global features. For tasks requiring fine-grained discrimination — distinguishing a coffee mug from a beer stein based on handle curvature — this information bottleneck is severe.

''Noise and outliers corrupt learned features:'' Point clouds from real sensors contain noise (measurement jitter), outliers (multipath reflections, dust, rain), and missing data (occlusions). Unlike images where noise appears as pixel-level perturbation, point cloud noise manifests as //geometric distortion// — points displaced from the true surface. A few outlier points far from the object can dramatically shift the max-pooled feature, and the T-Net alignment can be misled by systematic noise patterns. Robust processing requires explicit outlier filtering or architectures designed to down-weight anomalous points, adding complexity that standard PointNet-family methods do not provide.

!! Technical Details

''Foundational papers:'' Qi et al. 2017 ("PointNet: Deep Learning on Point Sets for 3D Classification and Segmentation") introduced the shared-MLP-plus-max-pool paradigm. Qi et al. 2017b ("PointNet++: Deep Hierarchical Feature Learning on Point Sets in a Metric Space") added hierarchical local feature learning. Wang et al. 2019 ("Dynamic Graph CNN for Learning on Point Clouds") introduced DGCNN with dynamic graph construction. Zhao et al. 2021 ("Point Transformer") applied self-<<link "Attention" "Attention">><</link>> to local point neighborhoods.

''Computational complexity:'' PointNet is $O(NKd)$ where $N$ is the number of points, $K$ is the feature dimension, and $d$ is the MLP depth — linear in $N$, making it highly scalable. PointNet++ adds the cost of farthest point sampling $O(N^2)$ and ball query $O(NK_{\text{neighbors}})$. Point Transformer's local attention is $O(NK^2_{\text{neighbors}})$ per layer, where $K_{\text{neighbors}}$ is the neighborhood size.

''Common datasets:'' ModelNet40 (12,311 CAD models, 40 categories, classification benchmark), ShapeNet (51,300 models with part annotations, segmentation benchmark), S3DIS (6 indoor areas with semantic labels, scene segmentation), KITTI (outdoor LiDAR driving dataset, 3D object detection), nuScenes (1,000 driving scenes, 10 annotation classes, 1.4M LiDAR sweeps), and Waymo Open Dataset (1,150 scenes, the largest public autonomous driving benchmark).

''Relation to other 3D representations:'' Point clouds are one of several 3D representations. Voxel grids discretize space but scale as $O(n^3)$. Meshes encode topology but require connectivity information. <<link "NeRF" "NeRF">><</link>> represents scenes as continuous radiance fields. <<link "Graph Neural Networks" "Graph Neural Network">><</link>> can process point clouds by constructing a $k$-NN graph, with points as nodes and edges weighted by distance. Multi-view methods render 2D images from multiple viewpoints and apply standard <<link "CNN" "CNN">><</link>> backbones, trading 3D reasoning for mature 2D architectures.

''Transformers on point clouds:'' The <<link "Attention" "Attention">><</link>> mechanism from <<link "Transformers" "Transformers">><</link>> has been adapted extensively. Point Cloud Transformer (PCT) uses offset attention with implicit Laplacian smoothing. Stratified Transformer (Lai et al. 2022) partitions 3D space for efficient long-range attention. These methods often achieve state-of-the-art results but at substantially higher computational cost than PointNet-family methods.
