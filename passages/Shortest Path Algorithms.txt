! Shortest Path Algorithms
<img src="assets/shortest_path_algorithms.svg" alt="Weighted graph with shortest path highlighted in red">

Imagine you are planning a road trip across the country. You have a map with cities connected by highways, each labeled with driving time. You want the fastest route from your hometown to the destination. You could try every possible route, but that would take forever. Shortest path algorithms are the clever strategies that let you find the best route without exhaustively checking all possibilities — and they underpin everything from GPS navigation to how data packets traverse the internet.

!! The Core Insight

Every shortest path algorithm relies on one beautifully simple observation called ''relaxation'': if you have found a path to node $u$ with cost $d[u]$, and there is an edge from $u$ to $v$ with weight $w(u,v)$, then you can potentially improve your best known distance to $v$ by checking whether $d[u] + w(u,v) &lt; d[v]$. If so, you update $d[v]$. The different algorithms are essentially different //strategies for deciding which edges to relax and in what order//.

''Dijkstra's algorithm'' is the greedy champion: always relax the closest unvisited node first. ''Bellman-Ford'' is the cautious planner: relax every edge, then do it again, and again — up to $|V|-1$ times — to guarantee correctness even with negative weights. ''Floyd-Warshall'' takes the global view: compute shortest paths between //all pairs// of nodes simultaneously using [[Dynamic Programming|Dynamic Programming]]. ''A* search'' is the informed optimizer: use a heuristic to guide the search toward the goal, exploring fewer nodes than Dijkstra when a good heuristic is available.

!! Algorithms in Detail

''Dijkstra's Algorithm'' maintains a priority queue of nodes sorted by tentative distance. At each step, it extracts the node with the smallest distance, marks it as finalized, and relaxes all its outgoing edges. This [[Greedy Algorithms|Greedy]] strategy works because, with non-negative edge weights, once a node is extracted, its shortest distance is guaranteed. With a binary heap, it runs in $O((|V| + |E|) \log |V|)$ time.

''Bellman-Ford'' iterates $|V| - 1$ times over all edges, relaxing each one. The key insight: in a graph with $|V|$ nodes, any shortest path has at most $|V|-1$ edges, so $|V|-1$ rounds of relaxation suffice. A bonus: if any distance //still// decreases on the $|V|$-th round, the graph contains a ''negative-weight cycle''. It runs in $O(|V| \cdot |E|)$.

''Floyd-Warshall'' fills in an $|V| \times |V|$ distance matrix. The recurrence is:

$$d^{(k)}[i][j] = \min\left(d^{(k-1)}[i][j],\; d^{(k-1)}[i][k] + d^{(k-1)}[k][j]\right)$$

where $d^{(k)}[i][j]$ is the shortest path from $i$ to $j$ using only intermediate nodes from $\{1, \dots, k\}$. This is a textbook application of [[Dynamic Programming|Dynamic Programming]], running in $O(|V|^3)$.

''[[A* Search|A* Search]]'' extends Dijkstra by adding a heuristic estimate $h(v)$ of the remaining cost from $v$ to the goal. It selects nodes to expand by minimizing $f(v) = g(v) + h(v)$, where $g(v)$ is the known cost from source to $v$. When $h$ is //admissible// (never overestimates) and //consistent// (satisfies the triangle inequality), A* finds the optimal path while often exploring far fewer nodes.

!! What the Math Is Actually Doing

The relaxation operation is the atomic building block. Formally, for an edge $(u, v)$ with weight $w$:

$$\text{Relax}(u, v, w): \quad \text{if } d[v] > d[u] + w \text{ then } d[v] \leftarrow d[u] + w, \; \pi[v] \leftarrow u$$

where $\pi[v]$ records the predecessor, letting you reconstruct the path.

Dijkstra's correctness proof relies on the ''optimal substructure'' property: any sub-path of a shortest path is itself a shortest path. Combined with non-negative weights, this ensures the greedy extraction order is safe.

Bellman-Ford leverages a different guarantee: after $k$ relaxation rounds, $d[v]$ is optimal over all paths using at most $k$ edges. After $|V|-1$ rounds:

$$d[v] = \delta(s, v) = \min_{\text{paths } p: s \rightsquigarrow v} w(p)$$

Floyd-Warshall's dynamic programming decomposition considers whether the optimal $i$-to-$j$ path passes through node $k$ or not, yielding the clean recurrence shown above.

For [[Big O Notation|Big O Notation]] reference:
* Dijkstra (binary heap): $O((|V| + |E|) \log |V|)$
* Dijkstra (Fibonacci heap): $O(|V| \log |V| + |E|)$
* Bellman-Ford: $O(|V| \cdot |E|)$
* Floyd-Warshall: $O(|V|^3)$
* A*: depends on heuristic quality; worst case matches Dijkstra

!! Connections to Machine Learning

Shortest path ideas appear throughout ML in surprising ways:

''Graph Neural Networks (GNNs)'' perform //message passing// that closely mirrors relaxation. Each node aggregates information from its neighbors and updates its representation — structurally identical to one round of Bellman-Ford. Stacking $k$ GNN layers lets information propagate along paths of length $k$, echoing the $|V|-1$ iteration bound.

''Reinforcement learning'' planning algorithms use shortest-path logic. Value iteration in an MDP is essentially Bellman-Ford on a stochastic graph, where the "relaxation" becomes the Bellman update: $V(s) \leftarrow \min_a \left[ c(s,a) + \gamma \sum_{s'} P(s'|s,a) V(s') \right]$.

''Routing and network optimization'' in distributed systems relies on variants of Dijkstra and Bellman-Ford (the internet's BGP protocol is based on distributed Bellman-Ford). Understanding these algorithms illuminates how data flows through computational graphs during training.

''Viterbi decoding'' in hidden Markov models finds the most likely state sequence — which is a shortest path problem on a trellis graph, typically solved with dynamic programming analogous to Bellman-Ford.

!! When Shortest Path Algorithms Break Down

''Negative-weight cycles:'' If a graph contains a cycle whose total weight is negative, the concept of "shortest path" becomes undefined — you can loop forever, driving the cost to $-\infty$. Dijkstra fails silently on negative edges (it may return wrong answers). Bellman-Ford at least //detects// negative cycles but cannot produce a finite shortest path through one.

''Inadmissible heuristics in A*:'' If the heuristic $h(v)$ overestimates the true cost to the goal, A* may return a suboptimal path. The algorithm becomes fast but wrong. In ML planning applications, learned heuristics are often inadmissible, requiring weighted-A* or anytime variants that trade optimality for speed.

''Scale and density:'' Floyd-Warshall's $O(|V|^3)$ time and $O(|V|^2)$ space become prohibitive for large graphs (millions of nodes). Even Dijkstra struggles on massive graphs without careful engineering — hierarchical decomposition, contraction hierarchies, or approximate methods become necessary. In GNN contexts, this is the "over-squashing" problem: information from distant nodes gets compressed through bottleneck paths.

!! Technical Details

''Single-source vs. all-pairs:'' Dijkstra and Bellman-Ford solve the ''single-source shortest path'' (SSSP) problem: shortest paths from one node to all others. Floyd-Warshall solves the ''all-pairs shortest path'' (APSP) problem. Running Dijkstra $|V|$ times also solves APSP in $O(|V|(|V|+|E|)\log|V|)$, which beats Floyd-Warshall on sparse graphs.

''Johnson's algorithm'' combines Bellman-Ford with Dijkstra for APSP on sparse graphs with negative weights: use Bellman-Ford once to reweight edges (eliminating negatives), then run Dijkstra from every source. Total: $O(|V|^2 \log|V| + |V| \cdot |E|)$.

''DAG shortest paths:'' If the graph is a directed acyclic graph, topological sort plus a single relaxation pass yields $O(|V| + |E|)$ — faster than all general methods. This is directly relevant to computation graphs in neural network forward passes.

''Contraction hierarchies'' preprocess a graph by iteratively contracting less important nodes and adding shortcut edges. After preprocessing, bidirectional Dijkstra on the augmented graph answers point-to-point queries in milliseconds even on continental road networks.

''Shortest path as linear programming:'' The SSSP problem can be formulated as: minimize $d[t]$ subject to $d[v] \leq d[u] + w(u,v)$ for all edges and $d[s] = 0$. The dual of this LP is a maximum flow problem, revealing deep connections between shortest paths and network flows in [[Graph Theory|Graph Theory]].
