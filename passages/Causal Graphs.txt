! Causal Graphs
<img src="assets/causal_graphs.svg" alt="Causal Graphs" width="800">

Imagine you notice that cities with more fire stations tend to have more fires. Should you close fire stations to reduce fires? Of course not — both are caused by a third factor: city size. ''Causal graphs'' are the mathematical tool that lets you draw out these relationships explicitly, distinguishing genuine causes from misleading correlations. They are directed acyclic graphs (DAGs) where each arrow represents a direct causal mechanism, and they give you a precise language for asking: //what would happen if we actually intervened//, rather than merely //what do we observe//?

!! The Core Insight

The fundamental idea behind causal graphs is that the world has a //direction//. Rain causes wet streets, not the other way around. A causal graph encodes this directionality as arrows between variables. Once you have the graph, you can read off exactly which statistical associations are causal and which are ''spurious'' — artifacts of shared causes, selection effects, or mediating pathways. The graph becomes a map that tells you which variables to adjust for, which to leave alone, and which you //must not// condition on, because doing so would actually //create// a false association where none existed.

!! From Correlation to Causation

A ''directed acyclic graph'' (DAG) is a set of nodes (variables) connected by directed edges (arrows) with no cycles — you can never follow the arrows and return to where you started. In a causal DAG, each arrow $X \rightarrow Y$ means that $X$ is a direct cause of $Y$ relative to the other variables in the graph.

Three fundamental structures appear in every causal graph:

//Chains//: $X \rightarrow M \rightarrow Y$. Here $M$ is a ''mediator''. The causal effect of $X$ on $Y$ flows //through// $M$. If you condition on the mediator, you block the path and can no longer see the effect of $X$ on $Y$ through that route.

//Forks//: $X \leftarrow C \rightarrow Y$. Here $C$ is a ''confounder''. It causes both $X$ and $Y$, creating a [[Spurious Correlations|Spurious Correlations]] between them. Conditioning on $C$ blocks the spurious path and reveals the true relationship (or lack thereof) between $X$ and $Y$.

//Colliders//: $X \rightarrow C \leftarrow Y$. Here $C$ is a ''collider''. The path between $X$ and $Y$ through $C$ is //naturally blocked//. But if you condition on $C$ (or any of its descendants), you //open// the path and create a spurious association. This is sometimes called ''collider bias'' or ''Berkson's paradox''.

''D-separation'' is the graphical criterion that formalizes all of this. Two variables $X$ and $Y$ are d-separated given a set $Z$ if every path between them is blocked — either by a non-collider in $Z$ or by a collider not in $Z$. D-separation in the graph implies conditional independence in any distribution compatible with the graph: $X \perp\!\!\!\perp Y \mid Z$.

These structures connect directly to [[Bayesian Networks|Bayesian Networks]] and [[Probabilistic Graphical Models|Probabilistic Graphical Models]], which share the same graphical language but add a quantitative probability layer.

!! What the Math Is Actually Doing

The central mathematical distinction in causal inference is between //observing// and //intervening//. Observing $X = x$ means learning that $X$ happened to take value $x$. Intervening means //forcing// $X$ to take value $x$, regardless of its usual causes. Judea Pearl's ''do-operator'' captures this:

$$P(Y \mid do(X = x)) \neq P(Y \mid X = x)$$

The left side asks: what is the distribution of $Y$ if we //set// $X$ to $x$? The right side asks: what is $Y$ among cases where $X$ //happened// to be $x$? These differ whenever confounders exist.

The ''backdoor criterion'' provides a sufficient condition for computing causal effects from observational data. A set of variables $Z$ satisfies the backdoor criterion relative to $X \rightarrow Y$ if: (1) no variable in $Z$ is a descendant of $X$, and (2) $Z$ blocks every path from $X$ to $Y$ that has an arrow //into// $X$ (a "backdoor path"). When satisfied, the ''adjustment formula'' gives:

$$P(Y \mid do(X = x)) = \sum_{z} P(Y \mid X = x, Z = z) \, P(Z = z)$$

This is the bridge from causal questions to statistical estimation — you adjust for (stratify on) the right confounders, and observational data gives you the causal answer.

The ''frontdoor criterion'' handles cases where you //cannot// observe the confounder directly. If a mediator $M$ lies on every directed path from $X$ to $Y$, and no unblocked backdoor paths exist from $X$ to $M$ or from $M$ to $Y$ after adjustment, then:

$$P(Y \mid do(X = x)) = \sum_{m} P(M = m \mid X = x) \sum_{x'} P(Y \mid X = x', M = m) \, P(X = x')$$

Pearl's ''do-calculus'' consists of three inference rules that, together, are //complete//: any causal quantity identifiable from the graph can be derived by applying these rules. The rules govern when you can insert or remove observations, interventions, and actions in probability expressions, based on d-separation criteria in modified versions of the graph.

These ideas build on the foundations of [[Conditional Probability|Conditional Probability]] and extend [[Causal Inference|Causal Inference]] with a rigorous graphical language.

!! Causal Reasoning in Practice

A ''structural causal model'' (SCM) is the full formal framework underlying causal graphs. It consists of: (1) a set of endogenous variables, (2) a set of exogenous (background) variables, (3) a set of structural equations $X_i = f_i(\text{parents}(X_i), U_i)$ defining each variable as a deterministic function of its parents and noise. The causal graph is a visual summary of the SCM, and interventions correspond to //replacing// one or more structural equations.

''A/B testing'' (randomized controlled trials) is the gold standard of causal inference because randomization physically implements the do-operator: by randomly assigning treatment, you sever all incoming edges to the treatment node, exactly as the right-hand panel of a mutilated graph shows. When experiments are impossible, we turn to observational methods.

''Instrumental variables'' provide another path. An instrument $Z$ affects the outcome $Y$ //only through// the treatment $X$, and shares no common causes with $Y$. Graphically: $Z \rightarrow X \rightarrow Y$ with no other paths from $Z$ to $Y$. This structure lets you estimate causal effects even with unmeasured confounders.

''Causal discovery algorithms'' attempt to learn the graph structure from data. The ''PC algorithm'' starts with a complete undirected graph and removes edges by testing conditional independences, then orients edges using collider patterns. The ''GES'' (Greedy Equivalence Search) algorithm searches over equivalence classes of DAGs by greedily optimizing a scoring function like BIC. Both algorithms return a ''Markov equivalence class'' — the set of DAGs that encode exactly the same conditional independences — since data alone typically cannot distinguish all causal directions.

!! When Causal Reasoning Breaks Down

* ''Unmeasured confounders''. If a common cause of treatment and outcome is missing from the graph, the backdoor criterion fails, and no adjustment set can remove the bias. The frontdoor criterion or instrumental variables may help, but only if the graph structure supports them. In practice, you can never be certain that all confounders have been measured, which is why observational causal claims always carry assumptions that cannot be tested from data alone.

* ''Faithfulness violations''. The standard framework assumes that every d-connection in the graph corresponds to a real statistical association (''faithfulness''). But exact cancellations — where two causal paths produce effects that perfectly cancel — violate faithfulness and cause discovery algorithms to infer wrong structures. These cancellations are measure-zero in theory but can arise in finely tuned systems, such as feedback control loops or symmetric architectures.

* ''Cycles and feedback''. Standard causal DAGs require acyclicity: no variable can cause itself, even indirectly. Many real systems — gene regulatory networks, economic markets, climate models — involve feedback loops. Extending causal reasoning to cyclic settings requires ''structural equation models with equilibrium'' or ''dynamic causal models'' that unfold time, and the neat guarantees of do-calculus no longer apply straightforwardly.

!! Technical Details

The ''causal Markov condition'' states that each variable is conditionally independent of its non-descendants given its parents in the causal DAG. This is the foundational assumption linking graph structure to probability distributions, and it is the same factorization property used in [[Bayesian Networks|Bayesian Networks]]:

$$P(X_1, \ldots, X_n) = \prod_{i=1}^{n} P(X_i \mid \text{Parents}(X_i))$$

''Faithfulness'' is the converse assumption: the //only// conditional independences in the data are those implied by d-separation. Together, the Markov condition and faithfulness make the graph identifiable (up to Markov equivalence) from the independence structure of the distribution.

A ''Markov equivalence class'' is the set of all DAGs that share the same skeleton (undirected edges) and the same v-structures (collider patterns $X \rightarrow C \leftarrow Y$ where $X$ and $Y$ are not adjacent). These classes are represented compactly as ''completed partially directed acyclic graphs'' (CPDAGs). Additional assumptions — such as additive noise models or non-Gaussianity — can break equivalences and allow identification of the unique causal DAG.

Pearl's three rules of do-calculus are:
* ''Rule 1 (Insertion/deletion of observations)'': $P(Y \mid do(X), Z, W) = P(Y \mid do(X), W)$ if $Y \perp\!\!\!\perp Z \mid X, W$ in the graph where all arrows into $X$ are removed.
* ''Rule 2 (Action/observation exchange)'': $P(Y \mid do(X), do(Z), W) = P(Y \mid do(X), Z, W)$ if $Y \perp\!\!\!\perp Z \mid X, W$ in the graph where arrows into $X$ are removed and arrows out of $Z$ are removed.
* ''Rule 3 (Insertion/deletion of actions)'': $P(Y \mid do(X), do(Z), W) = P(Y \mid do(X), W)$ if $Y \perp\!\!\!\perp Z \mid X, W$ in the graph where arrows into $X$ are removed and $Z$-ancestors removed in a specific way.

These rules are provably //complete//: if a causal effect is identifiable from the graph and observational data, do-calculus can derive it.

''Transportability'' extends causal reasoning across domains. Given experiments in a source population and observational data in a target population, transportability theory uses ''selection diagrams'' — causal graphs annotated with nodes indicating domain differences — to determine whether causal conclusions can be transferred, and how.
