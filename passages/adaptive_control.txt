! Adaptive Control

<img src="assets/adaptive_control.svg" alt="Adaptive Control block diagram showing controller, plant, adaptation law, and reference model in a feedback loop" style="max-width:700px; width:100%; margin:1em auto; display:block;">

Imagine you are learning to ride a bicycle for the first time. You do not know exactly how much to lean or how hard to pedal, so you keep adjusting—overcorrecting, then correcting less, until you ride smoothly. Now imagine the bicycle keeps changing: the tires deflate, the road tilts, wind gusts arrive. You never stop adjusting. That is ''adaptive control''—a controller that continuously tunes its own parameters as the system it governs shifts underneath it. Where classical <<link "Control Theory" "Control Theory">><</link>> assumes you know the plant, adaptive control assumes you //do not//, and learns on the fly.

!! The Core Insight

A fixed controller works perfectly only when the mathematical model of the plant is exact. In reality, models are never exact: aircraft lose fuel and change mass, chemical reactors age, robot joints wear down. Adaptive control separates the problem into two loops running simultaneously. The //inner loop// is a conventional controller that drives the plant. The //outer loop// watches the tracking error—the gap between what the system actually does and what a ''reference model'' says it //should// do—and adjusts the controller parameters to shrink that gap. The key insight is that you do not need to identify every unknown parameter of the plant; you only need to update the controller parameters so the closed-loop system behaves like the reference model.

!! Model-Reference vs Self-Tuning

There are two major families of adaptive control:

''Model-Reference Adaptive Control (MRAC)'' defines a reference model that embodies the desired closed-loop behavior. The adaptation law directly adjusts controller gains so the plant output tracks the reference model output. No explicit identification of plant parameters is required. This is the //direct// approach.

''Self-Tuning Regulators (STR)'' first estimate the plant parameters online using techniques like recursive least squares, then redesign the controller at each time step as though the estimates were the true values. This is the //indirect// approach—identify first, then control.

Both families share the same goal: make the closed-loop system perform well despite parametric uncertainty. MRAC is elegant and has strong stability guarantees via <<link "Lyapunov Functions" "Lyapunov Functions">><</link>>. STR is more flexible and can handle structural changes, but proving stability is harder.

A hybrid approach, ''dual control'', explicitly balances //exploitation// (controlling the plant with current knowledge) and //exploration// (probing the plant to improve parameter estimates)—an idea deeply connected to <<link "Reinforcement Learning" "Reinforcement Learning">><</link>>.

!! What the Math Is Actually Doing

Consider a first-order plant with unknown parameter $a$:

$$\dot{y}(t) = a \, y(t) + b \, u(t)$$

We want $y(t)$ to track a reference model $\dot{y}_m(t) = a_m \, y_m(t) + b_m \, r(t)$, where $a_m < 0$ ensures stability.

The controller uses adjustable parameters $\hat{\theta}(t) = [\hat{\theta}_1(t), \hat{\theta}_2(t)]$:

$$u(t) = \hat{\theta}_1(t) \, y(t) + \hat{\theta}_2(t) \, r(t)$$

Define the tracking error $e(t) = y(t) - y_m(t)$. The ''MIT rule'' (the earliest adaptation law) updates parameters proportional to the gradient of $e^2$:

$$\frac{d\hat{\theta}}{dt} = -\gamma \, e(t) \, \frac{\partial e}{\partial \hat{\theta}}$$

where $\gamma > 0$ is the adaptation gain. This is essentially online gradient descent on the squared error—a direct bridge to machine learning.

The MIT rule has no formal stability guarantee. The ''Lyapunov approach'' fixes this by choosing a candidate function:

$$V(\hat{\theta}, e) = \frac{1}{2} e^2 + \frac{1}{2\gamma} \|\tilde{\theta}\|^2$$

where $\tilde{\theta} = \hat{\theta} - \theta^*$ is the parameter error. Differentiating and choosing the adaptation law so that $\dot{V} \leq 0$ guarantees that $e(t) \to 0$ and parameters remain bounded. This is the foundation of ''Lyapunov-based MRAC''.

For indirect adaptive control, the plant parameters are estimated online:

$$\hat{a}(t+1) = \hat{a}(t) + P(t) \, y(t) \, \bigl(y(t+1) - \hat{a}(t) y(t) - \hat{b}(t) u(t)\bigr)$$

using recursive least squares, where $P(t)$ is the covariance matrix. The controller is then redesigned using the current estimates $\hat{a}(t)$, $\hat{b}(t)$ via pole placement or <<link "Optimal Control" "Optimal Control">><</link>> methods.

!! Practical Deployment and Robustness

Pure adaptive control can be fragile in practice. Several modifications are standard in deployed systems:

''Dead-zone modification'' — stop adapting when the tracking error is below a threshold, preventing parameter drift from noise.

''Projection'' — constrain parameters to a known feasible set, preventing them from wandering to dangerous values.

''Sigma modification'' ($\sigma$-modification) — add a damping term $-\sigma \hat{\theta}$ to the adaptation law, which gently pulls parameters back toward zero and improves robustness to unmodeled dynamics.

''Persistent excitation'' — the reference signal must be "rich enough" to ensure all parameters are identifiable. A constant setpoint does not excite all modes, and parameters can drift even if tracking error is zero.

Real-world applications include flight control (the F-16 uses gain-scheduled adaptive elements), process control in chemical plants, adaptive optics in telescopes, and increasingly in robotics where payloads and environments change unpredictably.

The <<link "Kalman Filter" "Kalman Filter">><</link>> often appears alongside adaptive controllers: the filter estimates states, while the adaptation law estimates parameters.

!! When X Breaks Down

''Unmodeled high-frequency dynamics'' — Adaptive controllers assume a model structure (e.g., first-order) and adapt parameters within it. If the real plant has fast parasitic modes not captured in the model, the adaptation can excite those modes and cause instability. The Rohrs counterexample (1985) demonstrated that MRAC with the MIT rule could go unstable when a small unmodeled lag was added—a cautionary tale that reshaped the field.

''Lack of persistent excitation'' — When the reference input is not sufficiently rich, the adaptation law cannot distinguish between different parameter combinations that produce the same output. Parameters drift aimlessly (the ''bursting phenomenon''), and a sudden disturbance can trigger large transients. Without excitation, convergence of parameter estimates is not guaranteed even if tracking error is small.

''Rapid time-varying parameters'' — Adaptive control assumes parameters change slowly relative to the adaptation speed. If the plant undergoes abrupt jumps (e.g., structural failure, sudden load change), the adaptation law lags behind, producing large transient errors. Switching or multiple-model adaptive control can help, but adds complexity and its own failure modes.

!! Technical Details

''Certainty equivalence principle:'' Indirect adaptive control replaces unknown parameters with their estimates and proceeds as if those estimates were correct. This works under persistent excitation and slow variation, but fails when uncertainty is large.

''Barbalat's lemma:'' Used alongside Lyapunov analysis—if $V$ is bounded below, $\dot{V} \leq 0$, and $\dot{V}$ is uniformly continuous, then $\dot{V} \to 0$, which implies $e(t) \to 0$. This is the standard tool for proving asymptotic tracking in MRAC.

''Parameter convergence vs. tracking convergence:'' Lyapunov MRAC guarantees $e(t) \to 0$, but $\hat{\theta}(t) \to \theta^*$ requires persistent excitation. Without it, parameters may not converge to true values even though tracking is perfect.

''Gain scheduling:'' A practical precursor to full adaptation—controller gains are pre-computed for a grid of operating points and interpolated in real time. Not truly adaptive (no online learning), but widely used because of predictability. Many "adaptive" flight controllers are actually gain-scheduled.

''$\mathcal{L}_1$ adaptive control:'' A modern architecture (Hovakimyan & Cao, 2010) that decouples adaptation from robustness by inserting a low-pass filter between the adaptive law and the control input. This allows arbitrarily fast adaptation without sacrificing robustness to unmodeled dynamics—addressing the classical tradeoff that plagued earlier designs.

''Multiple model adaptive control (MMAC):'' Maintains a bank of candidate models and switches or blends among them. Provides faster transient response than single-model schemes when the plant jumps between known operating regimes.

''Connection to reinforcement learning:'' Adaptive control's dual control problem—balancing identification and control—is formally equivalent to the exploration-exploitation tradeoff in <<link "Reinforcement Learning" "Reinforcement Learning">><</link>>. Modern ''adaptive dynamic programming'' merges Bellman optimality with online parameter estimation.
