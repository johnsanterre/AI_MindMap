! Combinatorics

<img src="assets/combinatorics.svg" alt="Hub-and-spoke diagram showing major branches of combinatorics: counting, graph theory, design theory, coding theory, and extremal combinatorics" style="width:100%;max-width:620px;display:block;margin:16px auto 24px auto;">

Combinatorics is the mathematics of choosing, arranging, and counting. Imagine a restaurant menu with 5 appetizers, 8 entrees, and 4 desserts — how many distinct three-course meals can you order? The answer ($5 \times 8 \times 4 = 160$) follows from the ''product rule'', the most basic counting principle. From this humble starting point, combinatorics scales up to handle questions like "how many distinct neural network architectures can be built from a menu of layer types?" or "how many subsets of features should we consider in model selection?" Whenever a problem asks //how many//, combinatorics provides the language — and often the answer.

!! The Core Insight

Every finite counting problem reduces to establishing a ''bijection'' — a one-to-one correspondence — between the objects you want to count and a set whose size you already know. The ''sum rule'' says: if choices $A$ and $B$ are mutually exclusive, the total count is $|A| + |B|$. The ''product rule'' says: if you make choice $A$ followed by independent choice $B$, the total is $|A| \times |B|$. ''Permutations'' count ordered arrangements ($n!$ ways to arrange $n$ distinct objects), while ''combinations'' count unordered selections ($\binom{n}{k}$ ways to choose $k$ objects from $n$). The deep insight is that //symmetry reduces counting//: if every arrangement has exactly $k!$ equivalent orderings, you divide by $k!$ to get unordered counts.

!! Permutations, Combinations, and Beyond

''Permutations'' of $n$ objects taken $k$ at a time: $P(n,k) = \frac{n!}{(n-k)!}$. Order matters — seating 3 people in 5 chairs gives $5 \times 4 \times 3 = 60$ arrangements.

''Combinations'': $\binom{n}{k} = \frac{n!}{k!(n-k)!}$. Order does not matter — choosing a 3-person committee from 5 people gives $\binom{5}{3} = 10$.

''Binomial theorem'': $(1 + x)^n = \sum_{k=0}^{n} \binom{n}{k} x^k$. This connects algebra to counting: the coefficient of $x^k$ //is// the number of ways to choose $k$ items from $n$.

''Multinomial coefficients'': $\binom{n}{k_1, k_2, \ldots, k_r} = \frac{n!}{k_1! k_2! \cdots k_r!}$ counts ways to partition $n$ objects into groups of specified sizes. These appear in the multinomial distribution used in text classification.

''Stars and bars'': The number of ways to distribute $n$ identical objects into $k$ distinct bins is $\binom{n+k-1}{k-1}$. This counts multisets and arises in polynomial feature counting.

''Burnside's lemma'': When a group $G$ acts on a set $X$, the number of distinct objects (up to symmetry) is $\frac{1}{|G|} \sum_{g \in G} |X^g|$, where $X^g$ is the set of elements fixed by $g$. This is how you count distinct necklaces, molecular configurations, or equivalence classes of model architectures under permutation symmetries.

!! What the Math Is Actually Doing

''The binomial coefficient identity:''

$$\binom{n}{k} = \binom{n-1}{k-1} + \binom{n-1}{k}$$

This is ''Pascal's rule'' — each entry in Pascal's triangle is the sum of the two entries above it. It encodes a recursive decomposition: either the $n$-th element is in the chosen subset (reducing to choosing $k-1$ from $n-1$) or it is not (choosing $k$ from $n-1$).

''Generating function encoding:'' The ordinary generating function for the sequence $\binom{n}{0}, \binom{n}{1}, \ldots, \binom{n}{n}$ is simply $(1+x)^n$. For the number of partitions of $n$ into parts of size $\leq k$, the generating function is $\prod_{i=1}^{k} \frac{1}{1-x^i}$.

''Inclusion-exclusion principle:''

$$\left|\bigcup_{i=1}^{n} A_i\right| = \sum_{i} |A_i| - \sum_{i<j} |A_i \cap A_j| + \sum_{i<j<k} |A_i \cap A_j \cap A_k| - \cdots$$

This alternating sum corrects for overcounting when sets overlap — essential for derangement counting, Euler's totient function, and computing the number of surjections.

''Stirling's approximation:'' $n! \approx \sqrt{2\pi n}\left(\frac{n}{e}\right)^n$, which gives $\log \binom{n}{k} \approx n \cdot H(k/n)$ where $H$ is the binary entropy function. This connects combinatorics directly to [[Binomial Distribution|Binomial Distribution]] and information theory.

!! Combinatorics in Machine Learning

''Feature subset selection:'' With $d$ features, there are $2^d$ possible subsets. This exponential blowup is why exhaustive feature search is infeasible for even moderate $d$, motivating greedy methods, L1 regularization, and random search.

''Architecture enumeration:'' Neural architecture search (NAS) operates over a combinatorial space of layer types, widths, connections, and activation functions. The search space for a cell with $B$ blocks, each choosing from $O$ operations, grows as $O(O^{2B} \cdot B!)$ — motivating differentiable architecture search (DARTS) and other relaxations.

''Combinatorial optimization:'' Problems like the traveling salesman ($n!/2$ tours), graph partitioning, and maximum clique are NP-hard combinatorial problems that arise in circuit design, scheduling, and cluster assignment. ML approaches (e.g., pointer networks, graph neural networks for TSP) attempt to learn heuristics for these.

''Hashing and sketching:'' Locality-sensitive hashing counts collisions in combinatorial bucket assignments to approximate nearest neighbors in high dimensions. The analysis relies on combinatorial probability.

''Data augmentation counting:'' If you apply $k$ independent binary augmentations, you generate $2^k$ augmented versions of each example — a direct application of the product rule.

!! When Combinatorics Breaks Down

''Exponential explosion.'' The most common failure mode: the number of objects grows so fast that exact enumeration is impossible. With 100 features, $2^{100} \approx 10^{30}$ subsets cannot be searched exhaustively. The practical response is to use //approximation algorithms//, randomized sampling, or structural constraints (sparsity, modularity) that prune the search space. This is the fundamental reason L1 regularization, which implicitly selects sparse subsets, is preferred over exhaustive search.

''Symmetry blindness.'' Naive counting often overcounts by ignoring symmetries. A graph with $n$ labeled vertices has $2^{\binom{n}{2}}$ possible edge sets, but the number of //non-isomorphic// graphs is vastly smaller. Burnside's lemma and Polya enumeration correct for this, but computing the number of orbits under a group action can itself be #P-hard for complex symmetry groups. In ML, permutation symmetries in neural networks (neuron relabeling) mean that the true space of functionally distinct networks is much smaller than the parameter space suggests.

''Continuous relaxation mismatch.'' Many combinatorial problems are solved in practice by relaxing discrete variables to continuous ones (e.g., replacing binary selection with soft attention weights). But the relaxed solution may not round cleanly back to a valid combinatorial object. The //integrality gap// — the difference between the relaxed optimum and the true discrete optimum — can be arbitrarily large for some problems. This affects NAS, sparse subset selection, and any method that treats a fundamentally discrete problem as continuous optimization.

!! Technical Details

''Catalan numbers:'' $C_n = \frac{1}{n+1}\binom{2n}{n}$ count the number of full binary trees with $n$ internal nodes, the number of valid parenthesizations of $n+1$ factors, and the number of monotone lattice paths that stay below the diagonal. Their generating function satisfies $C(x) = \frac{1 - \sqrt{1-4x}}{2x}$.

''Ramsey theory:'' For any $r, s \geq 2$, there exists a minimum $R(r,s)$ such that any 2-coloring of edges of $K_{R(r,s)}$ contains a monochromatic $K_r$ or $K_s$. Known exact values are sparse: $R(3,3) = 6$, $R(4,4) = 18$, while $R(5,5)$ is unknown (bounded $43 \leq R(5,5) \leq 48$).

''The twelvefold way:'' A systematic classification of all problems of distributing $n$ balls into $k$ boxes under combinations of: balls distinct/identical, boxes distinct/identical, any/injective/surjective mappings. This gives 12 fundamental counting problems whose solutions involve permutations, combinations, Stirling numbers, partitions, and compositions.

''Polya enumeration theorem:'' Extends Burnside's lemma by incorporating weights. The cycle index polynomial $Z_G(x_1, \ldots, x_n)$ of a permutation group $G$ encodes all the combinatorial information needed to count colored objects up to symmetry.
