! Multiple Hypothesis Testing
<img src="assets/multiple_hypothesis_testing.svg" alt="Multiple hypothesis testing correction methods and the multiple comparisons problem" style="max-width:100%;margin:1em 0;">

Imagine you flip a coin 20 times and declare it unfair if any single flip lands heads. You would "reject fairness" almost every time — not because the coin is rigged, but because you gave yourself too many chances to be wrong. Multiple hypothesis testing is the same problem: when you run many statistical tests at once, the probability of at least one false alarm skyrockets. Correction methods like Bonferroni, Holm, and Benjamini-Hochberg exist to keep your error rates honest.

!! The Core Insight

A single test at significance level $\alpha = 0.05$ has a 5% chance of a false positive. But running $m$ independent tests, the probability of ''at least one'' false positive is:

$$P(\text{any false positive}) = 1 - (1 - \alpha)^m$$

For $m = 20$: $1 - 0.95^{20} \approx 0.64$ — a 64% chance of at least one spurious "discovery." This is the ''multiple comparisons problem.'' It appears whenever you test multiple genes, multiple A/B variants, multiple subgroups, or multiple features simultaneously.

The solution is to adjust the significance threshold downward so the overall error rate stays controlled. Different methods control different error metrics — the ''family-wise error rate (FWER)'' or the ''false discovery rate (FDR)'' — with different trade-offs between strictness and <<link "Statistical Power" "Statistical Power">><</link>>.

!! Three Correction Strategies

''Bonferroni correction'' is the simplest: reject hypothesis $i$ only if $p_i \le \alpha / m$. For 20 tests at $\alpha = 0.05$, the per-test threshold drops to $0.0025$. This guarantees $\text{FWER} \le \alpha$ regardless of dependence structure, but it is extremely conservative — many real effects are missed.

''Holm's step-down procedure (1979)'' improves on Bonferroni without sacrificing FWER control. Sort p-values: $p_{(1)} \le p_{(2)} \le \cdots \le p_{(m)}$. Compare each $p_{(k)}$ against $\alpha / (m - k + 1)$. Reject from the smallest upward until the first failure. Holm is ''uniformly more powerful'' than Bonferroni — it always rejects at least as many hypotheses — so there is no reason to prefer Bonferroni over Holm.

''Benjamini-Hochberg (BH, 1995)'' controls the ''false discovery rate (FDR)'' instead of FWER. FDR is the expected proportion of rejected hypotheses that are false positives: $\text{FDR} = E[V/R]$, where $V$ is the number of false rejections and $R$ is the total rejections. By tolerating a small fraction of false discoveries rather than demanding zero, BH achieves substantially more <<link "Statistical Power" "Statistical Power">><</link>>.

!! What the Math Is Actually Doing

''Bonferroni'' applies the union bound:

$$P\left(\bigcup_{i=1}^{m} A_i\right) \le \sum_{i=1}^{m} P(A_i) = m \cdot \frac{\alpha}{m} = \alpha$$

This holds for any dependence structure between tests, making it universally valid but loose when tests are correlated.

''Holm's procedure'' uses ordered p-values $p_{(1)} \le \cdots \le p_{(m)}$. At step $k$, test:

$$p_{(k)} \le \frac{\alpha}{m - k + 1}$$

Reject $H_{(k)}$ if all previous hypotheses were also rejected. The denominators decrease ($m, m{-}1, \ldots$), giving progressively less strict thresholds. This is a ''closed testing'' procedure and controls FWER at level $\alpha$.

''Benjamini-Hochberg'' finds the largest $k$ such that:

$$p_{(k)} \le \frac{k}{m} \cdot \alpha$$

and rejects all $H_{(1)}, \ldots, H_{(k)}$. Under independence (or positive regression dependence), this guarantees $\text{FDR} \le \alpha$. The key insight: FDR control allows the threshold to ''increase'' with rank, making it fundamentally more lenient than FWER methods.

''Adjusted p-values'' (q-values): Rather than adjusting the threshold, one can compute adjusted p-values. For Bonferroni: $\tilde{p}_i = \min(m \cdot p_i, 1)$. For BH: the adjusted p-value (''q-value'') for rank $k$ is $\tilde{p}_{(k)} = \min_{j \ge k} \frac{m}{j} p_{(j)}$. A q-value of 0.1 means that among all hypotheses with q-values $\le 0.1$, at most 10% are expected to be false discoveries.

!! Practical Applications

''Genomics:'' A genome-wide association study (GWAS) tests millions of genetic variants. At $\alpha = 0.05$ without correction, $\sim$50,000 false positives are expected. The genome-wide significance threshold of $5 \times 10^{-8}$ is effectively a Bonferroni correction for $\sim$1 million independent tests.

''A/B testing:'' Running multiple <<link "A/B Testing" "A/B Testing">><</link>> variants or peeking at results repeatedly inflates false positive rates. Companies like Netflix and Microsoft use sequential testing frameworks with FDR control to manage thousands of simultaneous experiments.

''Neuroimaging:'' An fMRI scan tests activity at $\sim$100,000 voxels simultaneously. Without correction, random noise produces convincing-looking "brain activation" maps. Methods range from Bonferroni (too strict) to cluster-based FDR approaches.

''Machine learning feature selection:'' Testing which of $p$ features significantly predict the outcome requires multiple comparison correction, especially when $p \gg n$.

!! When Multiple Hypothesis Testing Breaks Down

''Extreme conservatism under dependence:'' Bonferroni and even Holm assume worst-case dependence. When tests are highly correlated (e.g., nearby genomic loci in linkage disequilibrium), these methods are far too strict, discarding real effects. Permutation-based methods or the Benjamini-Yekutieli procedure handle arbitrary dependence but at a power cost.

''FDR is misleading with few tests:'' FDR controls the ''expected proportion'' of false discoveries. With only 5 tests and 2 rejections, "FDR $\le$ 0.1" means you expect $\le$0.2 false discoveries — but you either have 0 or 1. The FDR guarantee is most meaningful when the number of rejections $R$ is large, as in genomics.

''Pre-registration vs. post-hoc correction:'' No correction method fixes the fundamental problem of ''data dredging'' — running unplanned tests until something is significant. If you test 100 subgroups but only report the 3 significant ones, correction on those 3 is meaningless. The full set of tests performed must be declared. Pre-registration of the analysis plan is the real solution.

!! Technical Details

- ''Bonferroni (1936):'' FWER $\le \alpha$, valid under any dependence; per-test threshold $\alpha/m$
- ''Sidak (1967):'' per-test threshold $1 - (1-\alpha)^{1/m}$, slightly less conservative than Bonferroni; assumes independence
- ''Holm (1979):'' step-down procedure; uniformly more powerful than Bonferroni; controls FWER under any dependence
- ''Hochberg (1988):'' step-up FWER procedure; more powerful than Holm but requires non-negative dependence (Simes' inequality)
- ''Benjamini-Hochberg (1995):'' controls FDR at level $\alpha$ under independence or PRDS (positive regression dependence on subsets)
- ''Benjamini-Yekutieli (2001):'' FDR control under arbitrary dependence; uses threshold $\frac{k}{m \cdot \sum_{i=1}^{m} 1/i} \cdot \alpha$
- ''Storey (2002):'' estimates the proportion of true nulls $\pi_0$ to sharpen BH; implemented as q-value
- FWER methods are standard when even one false positive is unacceptable (drug trials, confirmatory studies)
- FDR methods dominate in exploratory high-dimensional settings (genomics, proteomics, neuroimaging)
- ''Effective number of tests:'' when tests are correlated, the effective number $m_{\text{eff}} < m$, and using $m_{\text{eff}}$ in Bonferroni is less conservative
