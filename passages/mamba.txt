! Mamba

<img src="assets/mamba.svg" alt="Diagram illustrating Mamba and State Space Models" style="width:100%;max-width:620px;display:block;margin:16px auto 24px auto;">

Imagine reading a very long novel and trying to answer questions about it. A speed-reader with a photographic memory would glance at every page equally, building a complete but expensive mental snapshot — that is roughly how [[Transformers]] handle text. An experienced reader, by contrast, pays close attention when something important appears and skims the filler, updating a compact running summary as they go. Mamba is the experienced reader: a sequence model that //selectively// decides, for each incoming word, how much to update its memory and what to forget. The result is a model that handles sequences of any length using memory that grows linearly, not quadratically, with sequence length — a genuine architectural alternative to [[Attention]].

---

!! The Core Insight

The central idea behind Mamba is ''selectivity'': the parameters that govern how the hidden state is updated are not fixed constants but are computed //from the input itself//. Every token gets to decide how much of the current memory to keep and how much of itself to write in. This sounds simple, but it is a fundamental departure from earlier state space models and from [[RNN]] variants like [[LSTM]], where gating was learned but operated on fixed recurrent weight matrices rather than dynamically reshaping the state-transition equations.

Mamba belongs to a family called ''State Space Models (SSMs)'', which are classical tools from control theory repurposed for sequence modeling. SSMs define a hidden state that evolves continuously over time and is sampled at discrete steps to produce outputs. The appeal is that they can be computed //either// as a recurrence (fast at inference, O(1) memory per step) //or// as a convolution (fast at training, fully parallel). This duality gives SSMs the best of both worlds that [[RNN]] and [[Transformers]] each lack separately.

The broader context matters: [[Large Language Models]] built on [[Transformers]] pay an $O(L^2)$ memory and compute cost in sequence length $L$ due to the full attention matrix. For short sequences this is manageable; for documents of tens of thousands of tokens it becomes prohibitive. Mamba achieves $O(L)$ memory during inference, making it theoretically suited to arbitrarily long contexts — a property with large implications for [[Large Language Models]], genomics, and long audio processing. The [[Hardware Lottery]] is also relevant here: Mamba's advantage only materializes with a carefully engineered kernel that avoids the slow memory-bandwidth bottleneck on modern GPUs.

---

!! Selective State Spaces

Early SSMs like ''S4'' (Gu et al., 2022) showed that a carefully parameterized state space model with a special diagonal-plus-low-rank matrix $A$ could match or beat [[LSTM]] and even some [[Transformers]] on the Long Range Arena benchmark. But S4 used //fixed// parameters $A$, $B$, $C$ — the same transition matrix applied regardless of what the current token actually says. This works well for continuous signals like audio, where every time step carries roughly equal information. It works poorly for language, where most tokens ("the", "a", "of") carry very little information and a few tokens ("not", a proper noun, a negation) carry everything.

''Mamba'' (Gu and Dao, 2023) solves this by making the step size $\Delta$, the input projection $B$, and the output projection $C$ all functions of the current input $x$. When the model sees an unimportant token, it can set a tiny $\Delta$, barely nudging the hidden state — equivalent to fast-forwarding. When it sees a critical token, it sets a large $\Delta$, allowing the new information to overwrite or substantially update the state. This is the ''selective scan'': a scan algorithm whose coefficients vary per step based on the input. The analogy to [[LSTM]] gating is real but imprecise — Mamba's selectivity operates at the level of the state-space equations themselves, not just a gating vector multiplied onto a fixed recurrent output.

The evolution from S4 to Mamba illustrates a recurring theme in deep learning: a mathematically elegant but inflexible model (S4) becomes dramatically more powerful when given input-dependent flexibility, at the cost of losing one computational shortcut (S4 could be computed as a //global convolution// with a fixed kernel; Mamba cannot, because the kernel changes at every step). Mamba recovers efficiency through a different route: the hardware-aware parallel scan.

---

!! What the Math Is Actually Doing

A ''continuous-time SSM'' describes a hidden state $h(t) \in \mathbb{R}^N$ that evolves according to a linear ODE driven by input $x(t) \in \mathbb{R}$:

$$h'(t) = A\,h(t) + B\,x(t), \qquad y(t) = C\,h(t)$$

Here $A \in \mathbb{R}^{N \times N}$ is the state-transition matrix, $B \in \mathbb{R}^{N \times 1}$ projects the scalar input into state space, and $C \in \mathbb{R}^{1 \times N}$ projects the state back to a scalar output. For neural network use the continuous system is ''discretized'' using the Zero-Order Hold (ZOH) rule with a learned step size $\Delta > 0$:

$$\bar{A} = e^{\Delta A}, \qquad \bar{B} = (\Delta A)^{-1}\!\left(e^{\Delta A} - I\right)\Delta B$$

The discrete recurrence is then $h_t = \bar{A}\,h_{t-1} + \bar{B}\,x_t$, $y_t = C\,h_t$. In S4 the matrices $A$, $B$, $C$ and the step $\Delta$ are fixed after training. In Mamba the crucial change is:

$$\Delta_t = \text{softplus}(W_\Delta\, x_t + b_\Delta), \quad B_t = W_B\, x_t, \quad C_t = W_C\, x_t$$

Each of $\Delta_t$, $B_t$, $C_t$ is now a //per-token// linear projection of the input. This makes $\bar{A}_t$ and $\bar{B}_t$ also input-dependent, so the recurrence coefficients change at every step. The large-$\Delta$ limit makes $\bar{A}_t \to 0$ and $\bar{B}_t \approx B_t$ — the model forgets its past and copies the current input directly into state. The small-$\Delta$ limit makes $\bar{A}_t \to I$ and $\bar{B}_t \to 0$ — the model ignores the current input and passes the old state through unchanged. The model therefore learns to //focus// on relevant tokens and //skip// irrelevant ones, all within a linear recurrence.

---

!! Hardware-Aware Implementation

The selective scan cannot be written as a global convolution with a fixed kernel (unlike S4), so naïve sequential computation would be slow. Mamba uses a ''parallel scan algorithm'' — a classical parallel prefix-sum technique applied to the time-varying linear recurrence — to compute all $h_t$ in $O(\log L)$ parallel steps rather than $O(L)$ sequential steps. This makes training-time parallelism comparable to a Transformer.

The more subtle engineering challenge is memory bandwidth. A naïve implementation would write every intermediate hidden state $h_t \in \mathbb{R}^{N}$ to GPU High Bandwidth Memory (HBM) and read it back, causing massive memory traffic for long sequences. Mamba borrows the ''hardware-aware'' philosophy of Flash Attention (Dao et al., 2022): the entire scan is fused into a single GPU kernel that keeps intermediate states in fast on-chip SRAM, never materializing them in HBM. Only the final outputs $y_t$ and the last hidden state need to be written back to HBM.

The memory consequence is dramatic. A [[Transformers|Transformer]] must store the full $L \times L$ attention matrix — or at minimum the $L \times d$ key-value cache at inference — giving $O(L^2)$ or $O(L \cdot d)$ memory. A Mamba model keeps only the current hidden state $h \in \mathbb{R}^{N}$ between tokens at inference time: $O(N)$ regardless of $L$. In practice $N$ is a small constant (e.g., 16 per channel), making inference memory nearly independent of sequence length. This is a fundamental advantage for very long context tasks where Transformer KV-cache sizes become infeasible.

---

!! When Mamba Breaks Down

''In-context retrieval on associative recall tasks:'' Mamba struggles on tasks that require exactly retrieving a specific key-value pair seen earlier in a long context — for example, remembering that "the capital of X is Y" after thousands of intervening tokens. The fixed-size hidden state is a lossy compression of the past; unlike [[Attention]], which can look back at any specific earlier token via the KV cache, Mamba has no mechanism for precise random access to historical information. Hybrid architectures (alternating Mamba and attention layers) have been proposed specifically to recover this capability.

''Multi-head and multi-query generalization:'' The original Mamba architecture uses a single SSM channel structure without the multi-head parallelism that gives [[Transformers|Transformer]] attention much of its representational power to track multiple independent information streams simultaneously. Mamba-2 (2024) introduced a structured state space duality (SSD) framework that reformulates the selective SSM as a form of linear attention, enabling multi-head variants — but the simpler Mamba-1 design underperforms on tasks that naturally decompose into parallel reasoning streams.

''Vision and non-sequential modalities:'' Language is naturally sequential; images, graphs, and point clouds are not. Early Vision Mamba (Vim) experiments required inventing scan orders (row-major, column-major, diagonal zigzag) to linearize 2D image patches, and results on dense prediction tasks like segmentation remained below comparable [[CNN]] or vision Transformer baselines. The fundamental issue is that SSMs assume a meaningful sequential order; imposing an arbitrary scan order on inherently 2D or graph-structured data discards spatial locality in ways that attention's permutation-equivariant structure does not.

!! Technical Details

''Foundational papers:'' S4 — Gu et al., "Efficiently Modeling Long Sequences with Structured State Spaces," ICLR 2022. Mamba — Gu and Dao, "Mamba: Linear-Time Sequence Modeling with Selective State Spaces," arXiv 2312.00752, 2023. Mamba-2 — Dao and Gu, "Transformers are SSMs: Generalized Models and Efficient Algorithms Through Structured State Space Duality," ICML 2024.

''Complexity:'' Mamba training is $O(L \cdot N \cdot D)$ time and $O(L \cdot D)$ memory (with the fused kernel), where $L$ is sequence length, $N$ is SSM state dimension, and $D$ is model width. [[Transformers|Transformer]] attention is $O(L^2 \cdot D)$ time and $O(L^2)$ memory (or $O(L \cdot D)$ with Flash Attention but still $O(L^2)$ compute). Mamba inference is $O(N \cdot D)$ memory per step — the KV-cache equivalent is a single fixed-size state, not a growing cache.

''State dimension $N$:'' In Mamba-1, $N = 16$ is typical. The hidden state per channel is small; the expressiveness comes from the large number of channels $D$ (e.g., 4096 for a 3B-parameter model). The SSM is applied independently per channel (diagonal $A$), keeping the per-step computation $O(N \cdot D)$ rather than $O(N^2 \cdot D)$.

''Long Range Arena (LRA):'' The standard benchmark suite for long-sequence models. S4 substantially outperformed [[LSTM]] and vanilla [[Transformers]] on LRA tasks such as Path-X (16k-step sequences) and ListOps. Mamba matches or exceeds S4 on most LRA tasks while adding language modeling capability.

''Structured State Space Duality (Mamba-2):'' Dao and Gu showed that the selective SSM with a particular (semiseparable) matrix structure is mathematically equivalent to a form of ''linear attention'' with a specific masking pattern. This duality allows Mamba-2 to use tensor parallelism and multi-head structure from Transformer tooling, and to be trained with optimized BLAS routines, closing the hardware-efficiency gap with Transformers on modern accelerators.

''Hybrid models:'' Jamba (AI21 Labs, 2024) and Zamba interleave Mamba and [[Attention|attention]] layers, achieving Transformer-level associative recall while retaining Mamba's $O(L)$ inference cost for most layers. This hybrid approach is currently the most practical path to deploying SSMs in production [[Large Language Models]].

''Vision Mamba (Vim, 2024):'' applied bidirectional Mamba blocks to image patches, achieving competitive results on ImageNet classification. PlainMamba, VMamba, and LocalMamba subsequently explored different 2D scan strategies, with VMamba's cross-scan achieving strong results on dense prediction tasks by scanning in four directions and fusing the results.
