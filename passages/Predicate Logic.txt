! Predicate Logic
<img src="assets/predicate_logic.svg" alt="Predicate Logic diagram showing universal and existential quantifiers with domain examples and the transition from propositional to first-order logic" style="max-width:100%;margin:1em 0;">

Imagine you run a factory and need to state a rule: "Every widget must pass inspection." Propositional logic can only say "Widget #37 passed" or "Widget #38 passed" — one sentence per widget, forever. Predicate logic gives you variables and quantifiers so you can say it once: $\forall x\;(\text{Widget}(x) \rightarrow \text{Passed}(x))$. It upgrades logic from talking about specific facts to talking about //patterns// across entire domains — exactly the leap a machine learning model makes when it generalizes from training examples to unseen data.

!! The Core Insight

Where [[Propositional Logic|Propositional Logic]] treats statements as indivisible atoms (true or false), predicate logic cracks them open. A ''predicate'' $P(x)$ is a function from objects to truth values: $\text{Dog}(x)$ is true when $x$ is a dog, false otherwise. ''Quantifiers'' sweep over a domain of objects: $\forall x$ ("for all $x$") and $\exists x$ ("there exists an $x$"). Together they let you express relationships ($\text{Likes}(x, y)$), nested dependencies ($\forall x\;\exists y\;\text{Teaches}(y, x)$ — everyone has a teacher), and structural constraints impossible in propositional logic. First-order logic (FOL) is the most common form: quantifiers range over objects but not over predicates or functions themselves.

!! Predicates, Terms, and Quantifiers

A ''term'' is either a variable ($x$), a constant ($\text{Alice}$), or a function applied to terms ($\text{mother}(x)$). An ''atomic formula'' applies a predicate to terms: $\text{Enrolled}(\text{Alice}, \text{CS101})$. Complex formulas are built with connectives ($\land, \lor, \rightarrow, \neg$) and quantifiers ($\forall, \exists$).

''Scope'' matters. In $\forall x\;(\exists y\;\text{Likes}(x, y))$, $y$'s quantifier is //inside// $x$'s scope — for each person there is //some// liked thing (possibly different per person). Swap the quantifier order to $\exists y\;(\forall x\;\text{Likes}(x, y))$ and you claim a single thing is liked by everyone — a much stronger statement. Beginners confuse these constantly; the difference is the heart of predicate logic.

''Free variables'' are unbound by any quantifier, making a formula an open sentence (neither true nor false until you assign values). A ''sentence'' (closed formula) has every variable bound.

!! What the Math Is Actually Doing

A ''first-order structure'' (or interpretation) $\mathcal{M} = (D, \mathcal{I})$ consists of a domain $D$ (nonempty set of objects) and an interpretation $\mathcal{I}$ that maps each constant to an element of $D$, each $n$-ary function symbol to a function $D^n \to D$, and each $n$-ary predicate to a relation $\subseteq D^n$.

''Satisfaction'' is defined recursively:

$$\mathcal{M} \models \forall x\;\phi(x) \iff \text{for every } d \in D,\;\mathcal{M} \models \phi(d)$$

$$\mathcal{M} \models \exists x\;\phi(x) \iff \text{for some } d \in D,\;\mathcal{M} \models \phi(d)$$

A formula is ''valid'' (a tautology) if it holds in every structure; ''satisfiable'' if it holds in at least one. Unlike propositional logic, first-order validity is ''undecidable'' (Church–Turing theorem, 1936) — no algorithm can determine, for every FOL sentence, whether it is valid.

''Prenex normal form'' moves all quantifiers to the front: any FOL formula $\phi$ is logically equivalent to $Q_1 x_1\; Q_2 x_2\;\ldots\;Q_n x_n\;\psi$ where each $Q_i$ is $\forall$ or $\exists$ and $\psi$ is quantifier-free. This is useful for automated theorem provers and resolution.

''Skolemization'' eliminates existential quantifiers by replacing $\exists y$ with a ''Skolem function'' that depends on previously universally-quantified variables. For $\forall x\;\exists y\;P(x, y)$, introduce a function $f$ and rewrite as $\forall x\;P(x, f(x))$. This preserves satisfiability (not logical equivalence) and is essential for converting FOL formulas to clausal form for resolution-based provers.

''Herbrand's theorem'' says a universally quantified formula is unsatisfiable iff some finite set of its ground instances (substituting constant terms for variables) is propositionally unsatisfiable. This reduces first-order unsatisfiability checking to a propositional search — the basis of many automated theorem provers.

!! Predicate Logic in Machine Learning

''Knowledge representation:'' [[Knowledge Graphs|Knowledge Graphs]] encode facts as triples like $\text{LocatedIn}(\text{Paris}, \text{France})$ — essentially ground atomic formulas. Ontologies (OWL, RDF) use fragments of FOL to define class hierarchies and constraints. Link prediction in knowledge graphs can be viewed as predicting the truth value of an atomic formula given partial evidence.

''Logic programming:'' Prolog and Datalog implement Horn clause fragments of FOL (clauses with at most one positive literal). Inductive Logic Programming (ILP) learns first-order rules from data — e.g., learning $\forall x\;(\text{Grandparent}(x,z) \leftarrow \exists y\;(\text{Parent}(x,y) \land \text{Parent}(y,z)))$ from family trees. This gives interpretable, compositional hypotheses that neural networks cannot naturally express.

''Neuro-symbolic AI:'' systems like DeepProbLog, Logic Tensor Networks, and Neural Theorem Provers embed FOL reasoning into differentiable computation. Predicates become neural networks (soft truth values in $[0,1]$), quantifiers become aggregations (product for $\forall$, max for $\exists$), and the whole system is trained end-to-end with gradient descent. This bridges the gap between statistical learning and logical reasoning.

''Theorem provers:'' FOL provers (Vampire, E, Prover9) use resolution and superposition calculi grounded in Skolemization and Herbrand's theorem. Modern provers are guided by ML models that choose which clauses to resolve next, dramatically improving search efficiency.

!! When Predicate Logic Breaks Down

''Undecidability:'' first-order validity is undecidable — a theorem prover may run forever without determining if a formula is valid. Practical systems use time limits and heuristics, but there is no guarantee of termination for arbitrary inputs.

''Brittleness to uncertainty:'' classical FOL is all-or-nothing — $P(x)$ is true or false. Real-world knowledge is probabilistic ("smoking //usually// causes cancer"). Markov Logic Networks (MLNs) address this by attaching weights to FOL formulas, but inference becomes #P-hard and approximations are needed. [[Bayesian Networks|Bayesian Networks]] handle uncertainty more naturally but lack FOL's relational expressiveness.

''Grounding explosion:'' when the domain is large, grounding a first-order formula (enumerating all variable assignments) produces an exponential number of propositional clauses. Lifted inference techniques try to reason about groups of symmetric objects without full grounding, but scalability remains a major challenge.

!! Technical Details

''Completeness:'' Gödel's completeness theorem (1930) guarantees that every valid FOL sentence has a finite proof. This is distinct from decidability — proofs exist, but finding them may take unbounded time.

''Compactness:'' if every finite subset of a set of FOL sentences is satisfiable, then the whole set is satisfiable. This has deep consequences for model theory but also means FOL cannot express "there are finitely many $x$ such that $P(x)$."

''Expressiveness hierarchy:'' propositional logic $\subset$ monadic FOL (decidable) $\subset$ full FOL (undecidable) $\subset$ second-order logic (quantifies over predicates). Description logics (used in OWL) are carefully chosen decidable fragments of FOL.

''Unification:'' the core algorithm for resolution. Given two atomic formulas $P(f(x), a)$ and $P(y, a)$, the most general unifier is $\{y \mapsto f(x)\}$. Robinson's unification algorithm (1965) runs in near-linear time and underpins Prolog execution, type inference in programming languages, and FOL theorem proving.
