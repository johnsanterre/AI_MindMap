! Coordinate Descent
<img src="assets/coordinate_descent.svg" alt="Coordinate descent staircase path on contour plot with algorithm variants" style="max-width:100%;margin:1em 0;">

Imagine you are lost in a hilly landscape at night and can only walk due north/south or due east/west — never diagonally. Each time you pick one direction axis, walk downhill as far as you can along it, then switch to the other axis. You will zigzag your way to the valley floor in a distinctive staircase pattern. That is coordinate descent: instead of computing a gradient over all variables simultaneously, you optimize one variable (or one block of variables) at a time while holding everything else fixed.

!! The Core Insight

Many optimization problems become dramatically simpler when all but one variable are held constant. A problem that is hard in $p$ dimensions may reduce to $p$ one-dimensional problems, each with a closed-form solution. This is especially powerful for ''separable penalties'' — penalties that decompose as $\sum_j g(x_j)$ — because the per-coordinate subproblem inherits only one penalty term. The <<link "Lasso" "Lasso">><</link>> ($L_1$ regularization) is the textbook example: the full problem has no closed-form solution, but each coordinate update is just a soft-thresholding operation.

See also: <<link "Gradient Descent" "Gradient Descent">><</link>>, <<link "Convex Optimization" "Convex Optimization">><</link>>, <<link "Proximal Operators" "Proximal Operators">><</link>>, <<link "Elastic Net" "Elastic Net">><</link>>.

!! Cyclic, Randomized, and Block Variants

''Cyclic coordinate descent'' updates variables in a fixed order $j = 1, 2, \ldots, p, 1, 2, \ldots$ and is the most common implementation. It converges for convex differentiable objectives and for convex objectives with separable non-smooth terms (Tseng 2001).

''Randomized coordinate descent'' (Nesterov 2012) picks the coordinate $j$ uniformly at random at each step. This enables sharper convergence rate analysis: for a smooth convex function with coordinate-wise Lipschitz constants $L_j$, the expected suboptimality after $t$ iterations satisfies:

$$\mathbb{E}[f(x^{(t)}) - f(x^*)] \leq \frac{2p \bar{L} \|x^{(0)} - x^*\|^2}{t}$$

where $\bar{L} = \frac{1}{p}\sum_j L_j$. This scales with the //average// Lipschitz constant rather than the maximum, which can be much smaller.

''Block coordinate descent'' groups variables into blocks $B_1, \ldots, B_K$ and updates an entire block at a time. This is natural when variables cluster (e.g., group <<link "Lasso" "Lasso">><</link>>, multi-task learning) and can exploit structure in the block subproblem.

!! What the Math Is Actually Doing

Consider minimizing $f(x) = g(x) + \sum_{j=1}^{p} h_j(x_j)$ where $g$ is smooth convex and $h_j$ are convex but possibly non-smooth (like $|x_j|$ in the Lasso). The coordinate update for variable $j$ is:

$$x_j^{(t+1)} = \arg\min_{z} \left[ g(x_1^{(t+1)}, \ldots, x_{j-1}^{(t+1)}, z, x_{j+1}^{(t)}, \ldots, x_p^{(t)}) + h_j(z) \right]$$

For the <<link "Lasso" "Lasso">><</link>> with $f(x) = \frac{1}{2}\|Ax - b\|^2 + \lambda\|x\|_1$, define the partial residual $r_j = b - \sum_{k \neq j} A_k x_k$ where $A_k$ is column $k$. The coordinate update becomes the ''soft-thresholding operator'':

$$x_j \leftarrow S_{\lambda / \|A_j\|^2}\!\left(\frac{A_j^\top r_j}{\|A_j\|^2}\right), \quad S_\tau(u) = \text{sign}(u)\max(|u| - \tau, 0)$$

Each step requires only $O(n)$ operations (one inner product with column $A_j$), compared to $O(np)$ for a full <<link "Gradient Descent" "Gradient Descent">><</link>> step. When $p$ is large and the solution is sparse, this per-coordinate cost is transformative.

For the <<link "Elastic Net" "Elastic Net">><</link>> ($L_1 + L_2$ penalty), the update is similarly closed-form: scale the soft-threshold input by $1/(1 + \lambda_2)$ to account for the $L_2$ term.

!! Practical Implementation: The Pathwise Warm-Start Strategy

The ''pathwise'' (or regularization path) approach computes Lasso solutions for a decreasing sequence of $\lambda$ values $\lambda_{\max} > \lambda_1 > \cdots > \lambda_{\min}$. Starting from $\lambda_{\max}$ (where the solution is all zeros), each subsequent problem is warm-started from the previous solution. Since neighboring $\lambda$ values yield similar solutions, each warm-started solve requires very few coordinate passes — often 1 to 3 full cycles through all $p$ variables.

''Active set strategies'' further accelerate convergence: after a few full passes, identify the set of non-zero coordinates (the active set) and iterate only over those. For highly sparse solutions ($s \ll p$ non-zeros), this reduces the per-pass cost from $O(np)$ to $O(ns)$.

The ''glmnet'' library (Friedman, Hastie, Tibshirani 2010) implements this combination — pathwise warm starts, active set cycling, and coordinate descent — and can solve Lasso problems with $p > 10^6$ features in seconds. It remains the default solver for <<link "Lasso" "Lasso">><</link>> and <<link "Elastic Net" "Elastic Net">><</link>> across R and Python (via scikit-learn).

!! When Coordinate Descent Breaks Down

''Non-separable constraints:'' when constraints couple variables (e.g., $x_1 + x_2 \leq 1$), fixing all but one variable and optimizing may cycle without converging. The classic counterexample is minimizing $f(x_1, x_2) = -x_1 x_2$ on the unit square — coordinate descent gets stuck at a saddle point. Non-separable smooth penalties can also impede convergence if the interaction between variables is strong relative to the diagonal dominance.

''High correlation among features:'' when columns of $A$ are highly correlated, the coordinate-wise Lipschitz constants become similar to the global one, erasing the computational advantage of randomized coordinate descent. The staircase path also becomes extremely elongated — the method zigzags through a narrow valley, requiring many more passes to converge. <<link "Gradient Descent" "Gradient Descent">><</link>> or accelerated methods may be preferable.

''Non-convex objectives:'' coordinate descent can converge to non-stationary points for general non-convex problems. Even for non-convex objectives where each coordinate subproblem is well-defined, the method may cycle between multiple local optima of different coordinates without reaching any overall local minimum.

!! Technical Details

- ''Tseng 2001'' — convergence of cyclic coordinate descent for convex $f = g + h$ with $g$ smooth and $h$ separable
- ''Nesterov 2012'' — efficiency of randomized coordinate descent; $O(p \bar{L}/\epsilon)$ iteration complexity
- ''Friedman, Hastie, Tibshirani 2007'' — pathwise coordinate descent for the Lasso (glmnet)
- ''Wright 2015'' — Coordinate Descent Algorithms (survey); unifies cyclic, randomized, greedy, and block variants
- ''Richtarik and Takac 2014'' — parallel coordinate descent; near-linear speedup when coordinates are partially separable
- Per-coordinate Lasso update: $O(n)$ flops; full pass: $O(np)$; compare full gradient: $O(np)$
- Convergence rate (smooth convex, randomized): $O(p/t)$ in expectation
- Active set cycling reduces effective $p$ to the support size $s$ after a few initial passes
- For $L_1$-regularized problems, coordinate descent achieves exact zeros — no thresholding heuristic needed, unlike <<link "Proximal Operators" "Proximal Operators">><</link>> applied to the full gradient
