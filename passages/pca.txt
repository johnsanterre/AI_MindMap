! Principal Component Analysis
[img[assets/pca.svg]]

Imagine you are photographing a three-dimensional sculpture. From most angles, the photo looks like a confusing blob. But there is one particular angle where the sculpture's shape is most clearly visible — where the silhouette captures the most detail. ''Principal Component Analysis'' (PCA) does exactly this for high-dimensional data: it finds the "camera angle" that reveals the most structure, then the next-best angle perpendicular to the first, and so on. Each of these angles is a //principal component//, and together they let you compress a sprawling dataset into its most informative lower-dimensional shadow.

!! The Core Insight

Most real-world datasets contain features that are correlated — height and weight tend to move together, for example. PCA exploits this redundancy. It rotates the coordinate system so the new axes (principal components) align with the directions of greatest ''variance'' in the data. The first principal component captures the single direction along which the data is most spread out. The second captures the most spread in a direction //perpendicular// to the first, and so on. Because correlated features share information, a handful of principal components often capture the vast majority of the total variance, letting you discard the rest with minimal information loss.

This is fundamentally a change of basis: PCA does not throw away data points — it re-expresses every point in a new coordinate system where the axes are ranked by importance. Dimensionality reduction happens when you choose to keep only the top $k$ components.

!! Eigenvectors and Variance Maximization

The mathematical heart of PCA lies in the <<link "eigenvalue decomposition" "Eigenvalues and Eigenvectors">><</link>> of the data's <<link "covariance matrix" "Covariance Matrix">><</link>>. Given a data matrix $X$ (centered so each feature has zero mean), the covariance matrix is:

$$C = \frac{1}{n-1} X^\top X$$

Each eigenvector of $C$ defines a principal component direction, and its corresponding eigenvalue tells you how much variance that component captures. The eigenvector with the //largest// eigenvalue is PC1, the next-largest is PC2, and so on.

Why eigenvectors? Consider the problem: find a unit vector $\mathbf{w}$ that maximizes the variance of the projected data, $\text{Var}(\mathbf{w}^\top X) = \mathbf{w}^\top C \mathbf{w}$. Using a Lagrange multiplier to enforce $\|\mathbf{w}\| = 1$, the optimality condition becomes $C\mathbf{w} = \lambda \mathbf{w}$ — the definition of an eigenvector. The maximum variance equals the eigenvalue $\lambda$.

PCA can also be computed via the <<link "Singular Value Decomposition" "SVD">><</link>> of $X$ directly, which is numerically more stable: $X = U \Sigma V^\top$, where the columns of $V$ are the principal component directions and $\sigma_i^2 / (n-1)$ gives the variance along each component.

!! What the Math Is Actually Doing

''Step 1: Center the data.'' Subtract the mean of each feature so the cloud is centered at the origin:

$$\tilde{X} = X - \mathbf{1}\boldsymbol{\mu}^\top$$

''Step 2: Compute the covariance matrix'' (or go straight to SVD):

$$C = \frac{1}{n-1}\tilde{X}^\top \tilde{X}$$

''Step 3: Eigendecompose.'' Solve $C = V \Lambda V^\top$ where $\Lambda = \text{diag}(\lambda_1, \lambda_2, \ldots, \lambda_d)$ with $\lambda_1 \geq \lambda_2 \geq \cdots \geq \lambda_d$.

''Step 4: Project.'' Keep the top $k$ eigenvectors $V_k$ and project:

$$Z = \tilde{X} V_k \quad \in \mathbb{R}^{n \times k}$$

The ''explained variance ratio'' for component $i$ is $\lambda_i / \sum_j \lambda_j$. A common heuristic is to keep enough components to explain 95% of total variance.

''Reconstruction'' from $k$ components introduces an approximation error:

$$\hat{X} = Z V_k^\top + \mathbf{1}\boldsymbol{\mu}^\top, \quad \text{error} = \sum_{i=k+1}^{d} \lambda_i$$

This makes PCA the optimal linear <<link "dimensionality reduction" "Dimensionality Reduction">><</link>> in the least-squares sense — no other $k$-dimensional linear subspace yields a smaller reconstruction error.

!! PCA in Practice

''Standardize first.'' If features have different units (e.g., meters vs. kilograms), PCA on the raw covariance matrix will be dominated by the feature with the largest scale. Standardizing to unit variance (using the correlation matrix) is usually essential.

''Choosing $k$.'' Three common approaches: (1) the //scree plot// — graph eigenvalues and look for an "elbow"; (2) cumulative explained variance — pick $k$ where you cross a threshold like 90–95%; (3) cross-validation on a downstream task.

''Speed.'' Full eigendecomposition costs $O(d^3)$. For large $d$, randomized SVD or iterative methods (Lanczos, power iteration) compute only the top $k$ components in $O(ndk)$ time.

''Common applications:'' noise reduction (discard low-variance components), visualization (project to 2–3 dims), decorrelation as a preprocessing step for models that assume uncorrelated inputs, and compression (faces, spectra, gene expression).

!! When PCA Breaks Down

''1. Nonlinear structure.'' PCA finds linear axes. If data lies on a curved manifold (e.g., a Swiss roll), PCA will fail to unroll it. <<link "Kernel PCA" "Kernel PCA">><</link>> addresses this by first mapping data into a higher-dimensional feature space via a kernel function, then performing PCA there.

''2. Outlier sensitivity.'' Because PCA maximizes variance, a single outlier can dramatically rotate the principal components to accommodate it. Robust PCA variants use the median or trimmed covariance, and robust methods like PCP (Principal Component Pursuit) decompose the data into a low-rank component plus a sparse outlier matrix.

''3. Interpretability loss.'' Each principal component is a linear combination of //all// original features. When you need to know which individual features matter, PCA's mixed components can be opaque. Sparse PCA adds an $\ell_1$ penalty to force most loadings to zero, yielding components that depend on only a few features.

!! Technical Details

* ''Computational complexity:'' Full decomposition is $O(\min(nd^2, n^2d))$; randomized SVD for top $k$ components is $O(ndk)$.
* ''PCA is equivalent to'' minimizing reconstruction error $\|X - X V_k V_k^\top\|_F^2$.
* ''Probabilistic PCA (PPCA):'' Models data as $\mathbf{x} = W\mathbf{z} + \boldsymbol{\mu} + \boldsymbol{\epsilon}$, where $\mathbf{z} \sim \mathcal{N}(0, I)$ and $\boldsymbol{\epsilon} \sim \mathcal{N}(0, \sigma^2 I)$. Maximum likelihood recovers PCA; also handles missing data via EM.
* ''Incremental PCA:'' Processes data in mini-batches for datasets that do not fit in memory.
* ''Whitening:'' After PCA projection, dividing each component by $\sqrt{\lambda_i}$ yields uncorrelated, unit-variance features — useful as preprocessing for neural networks and ICA.
* ''Relationship to SVD:'' The right singular vectors of the centered data matrix are the principal component directions; the singular values relate to eigenvalues via $\lambda_i = \sigma_i^2 / (n-1)$.
* ''PCA on the correlation matrix'' is equivalent to PCA on standardized (z-scored) data and should be preferred when features have different units or vastly different scales.
* ''Kernel PCA'' uses the kernel trick to compute principal components in an implicit high-dimensional feature space, capturing nonlinear structure at the cost of an $n \times n$ kernel matrix.
