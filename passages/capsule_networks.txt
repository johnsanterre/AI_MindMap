! Capsule Networks
<img src="assets/capsule_networks.svg" style="width:100%;max-width:800px;display:block;margin:0.5em auto 1em;">

A standard neuron tells you ''there is an eye here'' — a single number expressing presence. A capsule tells you ''there is an eye here, tilted 15 degrees, at position (0.3, 0.7), with scale 0.8'' — a whole vector of properties. Because capsules carry pose information, a network built from them can recognize a face even when it is rotated or viewed from an unfamiliar angle, something that routinely trips up standard CNNs whose pooling layers discard exactly this information.

!! The Core Insight

Introduced by Sabour, Frosst, and Hinton (2017), capsule networks replace scalar neuron activations with ''activity vectors''. The magnitude of the vector encodes the probability that an entity is present; the direction encodes its instantiation parameters — pose, position, scale, orientation. A higher-level capsule is active only when its constituent lower-level capsules reach ''agreement'' about the pose of a shared parent entity. This encodes part-whole relationships explicitly rather than learning them implicitly through millions of training examples. Connections: [[CNN]], [[Deep Learning]], [[Vision Transformer]], [[Object Detection]], [[Semantic Segmentation]].

!! Dynamic Routing by Agreement

Each lower-level capsule $i$ predicts the pose of every higher-level capsule $j$ via a learned transformation matrix:

$$\hat{\mathbf{u}}_{j|i} = \mathbf{W}_{ij} \mathbf{u}_i$$

Routing then proceeds iteratively. Initialize log prior coupling scores $b_{ij} = 0$. For a fixed number of iterations (typically 3):

1. Compute coupling coefficients via softmax: $c_{ij} = \exp(b_{ij}) / \sum_k \exp(b_{ik})$
2. Compute weighted prediction sum: $\mathbf{s}_j = \sum_i c_{ij} \hat{\mathbf{u}}_{j|i}$
3. Apply squash non-linearity to get output vector $\mathbf{v}_j$
4. Update agreement score: $b_{ij} \leftarrow b_{ij} + \hat{\mathbf{u}}_{j|i} \cdot \mathbf{v}_j$

Lower-level capsules whose predictions align with the final output vote are given higher coupling weight; disagreeing capsules are down-weighted. The routing is performed at inference time — it is not gradient descent.

!! What the Math Is Actually Doing

The ''squash'' function acts as the non-linearity, ensuring the output vector magnitude stays in $[0, 1)$ so it can be interpreted as a probability:

$$\mathbf{v}_j = \frac{\|\mathbf{s}_j\|^2}{1+\|\mathbf{s}_j\|^2} \cdot \frac{\mathbf{s}_j}{\|\mathbf{s}_j\|}$$

Short input vectors are squashed toward zero; long input vectors are squashed toward unit length. Training uses a ''margin loss'' per output class $k$, with $T_k = 1$ if class $k$ is present:

$$\mathcal{L}_k = T_k \max(0,\, m^+ - \|\mathbf{v}_k\|)^2 + \lambda(1-T_k)\max(0,\, \|\mathbf{v}_k\| - m^-)^2$$

where $m^+ = 0.9$, $m^- = 0.1$, $\lambda = 0.5$. The key distinction from invariance is ''equivariance'': if the input rotates by 30 degrees, the capsule vector direction rotates predictably. The network does not ignore the transformation — it tracks it.

!! EM Routing and Later Developments

Hinton, Sabour, and Frosst (2018) extended capsules to use 4×4 ''pose matrices'' (representing full affine transforms) rather than simple vectors, and replaced routing-by-agreement with an ''expectation-maximization'' algorithm to cluster votes. EM routing achieved better viewpoint generalization on the SmallNorb benchmark. Kosiorek et al. (2019) introduced stacked capsule autoencoders, learning part-whole hierarchies in an unsupervised setting. A conceptual parallel: [[Vision Transformer]] patch tokens are philosophically similar — each patch token carries spatial and appearance information — but the routing is replaced by learned attention, which is more scalable. Capsule ideas have influenced thinking about ''compositional generalization'', the ability to recognize novel combinations of known parts.

!! When Capsule Networks Break Down

''Scalability:'' routing-by-agreement is run at inference time and is $O(L^2)$ in the number of capsule pairs per layer, making deep capsule networks slow. In practice, capsule models have been applied mainly to small images (MNIST, SmallNorb, CIFAR-10). ''Benchmark performance:'' despite strong theoretical motivations, capsule networks have not matched CNNs or Vision Transformers on large-scale benchmarks like ImageNet. The routing algorithm adds significant compute without delivering proportional gains at scale. ''Crowded scenes:'' the original routing algorithm assumes at most one instance of each entity type per region, so it degrades when many instances of the same class overlap — the ''crowding problem''. Stacked capsule autoencoders partially address this, but it remains an active research challenge.

!! Technical Details

''Key papers:'' Sabour, Frosst, Hinton (NeurIPS 2017) — dynamic routing between capsules; Hinton, Sabour, Frosst (ICLR 2018) — matrix capsules with EM routing; Kosiorek, Sabour, Teh, Hinton (NeurIPS 2019) — stacked capsule autoencoders. ''Results:'' 0.25% test error on MNIST (competitive with best CNNs at the time), modest gains on SmallNorb viewpoint generalization, limited results on CIFAR-10, no competitive ImageNet results. ''Routing iterations:'' typically 3; more iterations give diminishing returns and increase cost. ''Vector dimension:'' 8D for lower capsules, 16D for higher capsules in the original paper. ''Reconstruction regularizer:'' the original paper added a decoder that reconstructs the input from the winning capsule vector; this regularizer encourages the vector to encode genuine pose information. ''Relation to attention:'' dot-product self-attention can be viewed as a differentiable, gradient-trained form of soft routing, which may explain why Transformers capture some of the same representational benefits with better scalability.
