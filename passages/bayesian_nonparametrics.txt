! Bayesian Nonparametrics
<img src="assets/bayesian_nonparametrics.svg" alt="Diagram showing Dirichlet Process stick-breaking, Chinese Restaurant Process tables, Indian Buffet Process binary matrix, and Gaussian Process posterior with confidence bands" style="max-width:100%;margin:1em 0;">

Imagine sorting a pile of unlabeled photographs into groups. You don't know ahead of time how many categories there are — maybe five, maybe fifty. A fixed-$K$ model like $K$-means forces you to commit to a number before seeing the data, and getting it wrong warps everything downstream. ''Bayesian nonparametric'' models dodge this problem entirely: they place a prior over structures whose complexity is //unbounded//, then let the data itself reveal how much complexity is warranted. More data can always create new clusters, new features, or new function wiggles. The model never runs out of room, yet the <<link "Bayes Theorem" "Bayes Theorem">><</link>> machinery automatically penalizes unwarranted complexity through the prior.

!! The Core Insight

In a ''parametric'' model, the number of parameters is fixed before training — pick $K = 5$ clusters, fit 5 means. In a ''nonparametric'' model, the effective number of parameters //grows with the data//. "Nonparametric" does not mean "no parameters" — it means "not a //fixed// number of parameters." A histogram is nonparametric: add more data, add more bins. Bayesian nonparametrics combine this flexibility with principled uncertainty quantification from <<link "Bayes Theorem" "Bayes Theorem">><</link>>.

The three foundational building blocks are:
* The ''Dirichlet Process'' (DP) — a distribution over distributions, used for <<link "Clustering" "Clustering">><</link>> with an unknown number of clusters.
* The ''Indian Buffet Process'' (IBP) — a distribution over binary feature matrices with an unbounded number of features.
* The ''Gaussian Process'' (GP) — a distribution over //functions//, used for regression and classification with full uncertainty. See <<link "Gaussian Processes" "Gaussian Processes">><</link>>.

!! Metaphors That Do the Heavy Lifting

The ''Chinese Restaurant Process'' (CRP) is the most famous metaphor in Bayesian nonparametrics, and it makes the Dirichlet Process concrete. Imagine an infinite restaurant. The first customer sits at table 1. Each subsequent customer either joins an existing table with probability proportional to how many people already sit there, or starts a //new// table with probability proportional to a concentration parameter $\alpha$. Formally, customer $n+1$ joins table $k$ with probability $\frac{n_k}{n + \alpha}$, or starts a new table with probability $\frac{\alpha}{n + \alpha}$.

This "rich get richer" dynamic naturally produces a power-law distribution of cluster sizes — a few large clusters and a long tail of small ones — which matches many real-world phenomena. Crucially, the number of tables (clusters) grows as $O(\alpha \log n)$, so the model always has room for new structure without exploding.

The ''Indian Buffet Process'' extends this idea to //features// instead of clusters. Customers walk along an infinite buffet. Each customer samples previously chosen dishes with probability proportional to their popularity, then samples $\text{Poisson}(\alpha / n)$ new dishes. The result is a binary matrix $Z$ where rows are objects and columns are latent features — and the number of active columns grows logarithmically with the number of objects.

!! What the Math Is Actually Doing

The ''Dirichlet Process'' $\text{DP}(\alpha, G_0)$ is a distribution over probability distributions. A draw $G \sim \text{DP}(\alpha, G_0)$ is a discrete distribution (with probability 1), even if the base distribution $G_0$ is continuous. The ''stick-breaking construction'' makes this explicit:

$$G = \sum_{k=1}^{\infty} \pi_k \,\delta_{\theta_k}, \quad \theta_k \sim G_0$$

where the weights $\pi_k$ are constructed by repeatedly breaking a unit-length stick:

$$v_k \sim \text{Beta}(1, \alpha), \quad \pi_k = v_k \prod_{j=1}^{k-1}(1 - v_j)$$

Large $\alpha$ yields many small pieces (many clusters); small $\alpha$ yields a few dominant pieces. The <<link "Dirichlet Distribution" "Dirichlet Distribution">><</link>> over finite partitions is the finite-dimensional marginal of this process.

For a ''DP mixture model'', each data point $x_i$ is generated by first drawing a cluster assignment from $G$, then sampling from a cluster-specific likelihood:

$$G \sim \text{DP}(\alpha, G_0), \quad \theta_i \sim G, \quad x_i \sim F(\theta_i)$$

Because $G$ is discrete, multiple $\theta_i$ values will coincide — those data points share a cluster. Inference via <<link "MCMC" "MCMC">><</link>> or variational methods discovers both the number of clusters and their parameters simultaneously.

The ''Gaussian Process'' takes a different nonparametric route. Instead of a distribution over discrete structures, it defines a distribution over //functions//: any finite collection of function values $(f(x_1), \ldots, f(x_n))$ is jointly Gaussian. The kernel function $k(x, x')$ encodes prior beliefs about smoothness. The posterior mean interpolates observed data exactly (in the noise-free case), and the posterior variance grows where data is sparse — a natural uncertainty estimate. See <<link "Gaussian Processes" "Gaussian Processes">><</link>> for the full treatment.

!! Practical Applications

''Topic modeling:'' The Hierarchical Dirichlet Process (HDP) extends LDA to learn the number of topics from data rather than fixing $K$. Each document draws from a document-specific mixture of topics, and the global set of topics grows as needed.

''Object recognition and scene understanding:'' The IBP models images as combinations of latent binary features (parts, textures). The model discovers how many features exist without a fixed budget.

''Time series and changepoint detection:'' The Dependent Dirichlet Process models evolving cluster structure, detecting when new regimes appear in financial data or sensor streams.

''Spatial statistics and emulation:'' Gaussian Processes serve as surrogate models for expensive simulations (climate models, CFD), providing predictions with calibrated uncertainty bands.

''Bayesian optimization:'' GPs model an unknown objective function, and an acquisition function (Expected Improvement, UCB) uses the GP posterior to decide where to query next — balancing exploration and exploitation.

!! When Bayesian Nonparametrics Break Down

''Computational cost:'' Naive <<link "MCMC" "MCMC">><</link>> for DP mixtures scales poorly beyond a few thousand data points. The Chinese Restaurant Franchise sampler for HDPs is notoriously slow. Variational truncated approximations (truncate the stick-breaking at $K$ terms) help but sacrifice the infinite-dimensional guarantee. GPs scale as $O(n^3)$ for $n$ data points — prohibitive for large datasets without sparse approximations or inducing points.

''Sensitivity to the base distribution and kernel:'' The DP's base distribution $G_0$ acts as a prior over cluster parameters, and a poor choice biases the posterior severely. Similarly, a GP with the wrong kernel (e.g., too smooth for a rough function) produces overconfident and misleading uncertainty bands. Model misspecification in the nonparametric prior can be harder to diagnose than in parametric models because the model space is so vast.

''Cluster fragmentation:'' The "rich get richer" property of the CRP can also pathologically //fragment// clusters. With noisy data, the DP may split a single true cluster into many tiny ones rather than discovering the correct structure. Pitman-Yor processes (a generalization of the DP with a discount parameter) can mitigate this but add another hyperparameter to tune.

!! Technical Details

''Historical foundations:'' Ferguson 1973 introduced the Dirichlet Process. Sethuraman 1994 gave the stick-breaking construction. Escobar and West 1995 developed practical MCMC for DP mixtures. Teh et al. 2006 introduced the Hierarchical Dirichlet Process. Griffiths and Ghahramani 2011 formalized the Indian Buffet Process.

''Stick-breaking truncation:'' In practice, the infinite sum is truncated at $K$ terms. For $K \gg \alpha \log n$, the approximation error is negligible. This is the basis of variational inference for DP mixtures (Blei and Jordan 2006).

''Pitman-Yor Process:'' Generalization with parameters $(\alpha, d)$ where $0 \leq d < 1$. The CRP becomes: join table $k$ with probability $\frac{n_k - d}{n + \alpha}$, new table with probability $\frac{\alpha + Kd}{n + \alpha}$. When $d = 0$, this recovers the DP. The PYP produces power-law tail behavior more naturally than the DP, making it popular for language modeling (word frequency distributions follow Zipf's law).

''Connections to other frameworks:'' The DP is the infinite limit of finite Dirichlet-multinomial models (as $K \to \infty$ with appropriate scaling). The IBP is the infinite limit of a Beta-Bernoulli model. <<link "Gaussian Processes" "Gaussian Processes">><</link>> are the infinite-width limit of Bayesian neural networks (Neal 1996), connecting nonparametric methods to deep learning theory.

''Key inference algorithms:'' Gibbs sampling with CRP representation; slice sampling (Walker 2007); variational inference with truncated stick-breaking; spectral methods for large-scale GP approximation; stochastic variational inference for streaming DP mixtures.
