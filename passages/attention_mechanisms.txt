! Attention Mechanisms

[img[assets/attention_mechanisms.svg]]

Imagine reading a long legal contract to answer one specific question. You do not read every word with equal care -- your eyes jump to the clauses that matter, skim the boilerplate, and linger on the paragraph that actually contains the answer. Attention mechanisms give neural networks this same ability. Instead of compressing an entire input into a single fixed-size summary (the way early sequence models did), attention lets a model //look back// at every part of the input and decide, on the fly, how much weight each part deserves for the task at hand.

---

!! The Core Insight

The central idea is ''dynamic, content-dependent weighting''. In a traditional encoder-decoder architecture, the decoder receives a single context vector -- a lossy summary of the entire input sequence. Bahdanau et al. (2014) showed that this bottleneck could be removed by letting the decoder, at each time step, compute a weighted combination of //all// encoder hidden states. The weights are not fixed; they are learned functions of the current decoder state and each encoder state, so different decoder steps attend to different input positions.

This simple change solved the long-sequence degradation problem in machine translation almost overnight. But the real revolution came when Vaswani et al. (2017) asked: what if attention is //all// you need? The [[Transformers]] architecture discarded recurrence entirely, using ''self-attention'' to let every token in a sequence attend to every other token. That architectural bet -- replacing sequential processing with parallel, attention-based processing -- underpins virtually every modern large language model, from BERT to GPT-4 and beyond.

The attention mechanism has three learned projections: ''Query'' (what am I looking for?), ''Key'' (what do I contain?), and ''Value'' (what information do I carry?). Matching queries against keys produces relevance scores; those scores, after [[Softmax]] normalisation, determine how much of each value contributes to the output. This Q-K-V framework is the universal language of modern attention, from [[Self-Attention]] within a single sequence to cross-attention between an encoder and decoder.

---

!! From Additive to Scaled Dot-Product

The original attention mechanism from Bahdanau et al. (2014) used ''additive attention'' (also called //concatenation-based// attention). Given a decoder hidden state $s_t$ and an encoder hidden state $h_j$, the alignment score was:

$$e_{t,j} = v^T \tanh(W_1 s_t + W_2 h_j)$$

This is a small feedforward network with learnable weight matrices $W_1$, $W_2$ and a weight vector $v$. Additive attention is flexible -- the tanh nonlinearity lets it model complex alignment patterns -- but it is slow because each score requires a full matrix multiply and a nonlinearity.

Luong et al. (2015) introduced ''multiplicative attention'' (dot-product attention), which computes scores as:

$$e_{t,j} = s_t^T h_j$$

This is much faster because it can be parallelised as a single matrix multiplication across all positions. However, raw dot products grow with the dimensionality of the vectors: when $d_k$ is large, the dot products can become very large in magnitude, pushing the softmax into regions where its gradients are vanishingly small.

Vaswani et al. (2017) fixed this with ''scaled dot-product attention'', dividing by $\sqrt{d_k}$:

$$\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

The $\sqrt{d_k}$ scaling keeps the variance of the dot products roughly constant regardless of dimension, preventing gradient saturation. This is the form used in virtually all modern [[Transformers]], and it is the foundation for [[Multi-Head Attention]], where multiple independent attention heads each project Q, K, V into smaller subspaces and attend in parallel.

---

!! What the Math Is Actually Doing

Given input matrices $Q \in \mathbb{R}^{n \times d_k}$, $K \in \mathbb{R}^{m \times d_k}$, and $V \in \mathbb{R}^{m \times d_v}$, scaled dot-product attention computes:

$$A = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right) \in \mathbb{R}^{n \times m}$$

$$\text{Output} = A \cdot V \in \mathbb{R}^{n \times d_v}$$

Each row of $A$ is a probability distribution over the $m$ key positions. The entry $A_{i,j}$ tells you how much query $i$ attends to key $j$. Multiplying $A$ by $V$ produces, for each query position, a weighted average of all value vectors -- a ''soft lookup'' into a differentiable memory.

The softmax ensures two properties: (1) all weights are non-negative, and (2) they sum to one, forming a valid probability distribution. The $\sqrt{d_k}$ scaling factor corrects for the expected magnitude of the dot products. If the entries of $Q$ and $K$ are i.i.d. with mean 0 and variance 1, their dot product has mean 0 and variance $d_k$. Dividing by $\sqrt{d_k}$ restores unit variance:

$$\text{Var}\!\left(\frac{q \cdot k}{\sqrt{d_k}}\right) = \frac{d_k}{d_k} = 1$$

Without this scaling, large $d_k$ would cause some softmax outputs to approach 0 or 1, creating near-one-hot distributions that block gradient flow. The scaling keeps the softmax operating in its sensitive, gradient-rich regime.

In ''multi-head attention'', the model runs $h$ independent attention functions in parallel, each with its own learned projections $W_i^Q$, $W_i^K$, $W_i^V$, then concatenates and linearly projects the results:

$$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h) W^O$$

where $\text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V)$. Each head can specialise: one might track syntactic dependencies, another semantic similarity, another positional proximity.

---

!! Attention Variants

''Self-attention'' (intra-attention) is when queries, keys, and values all come from the same sequence. Every token attends to every other token (including itself), building context-aware representations. [[Self-Attention]] is the core operation in both encoder-only models (BERT) and decoder-only models (GPT). In autoregressive (decoder) models, a ''causal mask'' sets all entries above the diagonal to $-\infty$ before the softmax, preventing tokens from attending to future positions.

''Cross-attention'' occurs between two different sequences -- for example, a decoder attending to encoder outputs in machine translation. The queries come from the decoder; the keys and values come from the encoder. This is the mechanism that lets a translation model selectively look back at the source sentence while generating each target word.

''Multi-head attention'' runs several attention operations in parallel with independent projections, as described above. [[Multi-Head Attention]] lets the model jointly attend to information from different representation subspaces. A typical transformer uses 8 to 128 heads depending on model size.

''Sparse attention'' restricts each token to attend only to a subset of positions -- local windows, dilated patterns, or learned sparse masks. This reduces the quadratic cost to $O(N \sqrt{N})$ or even $O(N \log N)$. Longformer, BigBird, and related architectures use structured sparsity to handle documents of 4K to 16K tokens.

''Linear attention'' replaces the softmax with a kernel function $\phi$, allowing the computation to be rewritten as $\phi(Q)(\phi(K)^T V)$, which can be computed in $O(N)$ time by exploiting associativity. Performers and Random Feature Attention use this trick, though they typically sacrifice some quality compared to full softmax attention.

''[[Flash Attention]]'' is not a different //type// of attention but an IO-aware //implementation// of exact scaled dot-product attention that avoids materialising the full $N \times N$ score matrix in GPU memory. It delivers identical outputs with dramatically reduced memory and wall-clock time.

---

!! When Attention Breaks Down

''Quadratic cost at long sequences:'' The $N \times N$ score matrix means that computation and memory scale as $O(N^2)$ in sequence length. At $N = 32{,}768$ with 32-bit floats, the score matrix alone is 4 GB. This makes standard attention impractical for very long documents, genomic sequences, or high-resolution images without sparse approximations or efficient implementations like [[Flash Attention]]. The quadratic wall is the single biggest limitation of the transformer architecture.

''Attention collapse and entropy degradation:'' In deep transformers, attention distributions can become excessively peaked (near one-hot) or excessively uniform across layers. Peaked attention means the model ignores most of the context; uniform attention means it treats every token as equally relevant, degrading to a bag-of-words. Both failure modes reduce the model's effective context window. Techniques like attention dropout, layer normalisation, and careful initialisation mitigate this, but it remains an active area of research.

''Positional insensitivity without explicit encoding:'' Raw attention is ''permutation-equivariant'' -- swapping two tokens in the input swaps their outputs but does not change the attention scores. Without positional encodings or positional biases, attention has no concept of word order, which is obviously catastrophic for language. Sinusoidal encodings (Vaswani et al., 2017), learned position embeddings, ALiBi, and RoPE each address this differently, but every solution introduces its own tradeoffs for generalisation to unseen sequence lengths.

---

!! Technical Details

''Original papers:'' Bahdanau, Cho, Bengio (2014) "Neural Machine Translation by Jointly Learning to Align and Translate" introduced additive attention. Luong, Pham, Manning (2015) "Effective Approaches to Attention-based Neural Machine Translation" introduced multiplicative/dot-product attention. Vaswani et al. (2017) "Attention Is All You Need" introduced scaled dot-product attention, multi-head attention, and the transformer architecture.

''Complexity:'' Standard self-attention is $O(N^2 d)$ in time and $O(N^2 + Nd)$ in memory, where $N$ is sequence length and $d$ is model dimension. The $N^2$ term dominates at long sequences. Multi-head attention with $h$ heads and head dimension $d_k = d/h$ has the same total cost as single-head attention with dimension $d$.

''Head dimension:'' In practice, $d_k = d_v = d_{\text{model}} / h$. For GPT-3 ($d = 12288$, $h = 96$), each head operates in $d_k = 128$ dimensions. Smaller head dimensions allow more heads (more diverse attention patterns) but reduce per-head capacity.

''Attention masking:'' Causal (autoregressive) masking adds $-\infty$ to upper-triangular entries of $QK^T / \sqrt{d_k}$ before softmax, zeroing those attention weights. Padding masks similarly zero out attention to padding tokens. Both are applied additively before softmax.

''KV caching:'' During autoregressive generation, keys and values from previous tokens are cached so that each new token only computes its own Q and attends to the growing cache. This reduces per-token generation from $O(N^2 d)$ to $O(Nd)$ but requires $O(N \cdot h \cdot d_k \cdot 2)$ memory for the cache (factor of 2 for K and V).

''Grouped Query Attention (GQA):'' Introduced by Ainslie et al. (2023), GQA shares key-value heads across multiple query heads, reducing KV cache size by a factor equal to the grouping ratio. Llama 2 70B uses GQA with 8 KV heads shared across 64 query heads (8:1 ratio). This dramatically reduces memory during inference with minimal quality loss.

''Temperature scaling:'' Some implementations replace $\sqrt{d_k}$ with a learned or tuned temperature parameter $\tau$, computing $\text{softmax}(QK^T / \tau)$. Lower $\tau$ sharpens attention (more peaked distributions); higher $\tau$ flattens it (more uniform). This generalises the fixed $\sqrt{d_k}$ scaling.
