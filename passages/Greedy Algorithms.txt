! Greedy Algorithms

<img src="assets/greedy_algorithms.svg" alt="Diagram showing greedy choice property, classic greedy problems, and failure cases" style="width:100%;max-width:620px;display:block;margin:16px auto 24px auto;">

Imagine you are at a buffet with limited plate space. A ''greedy algorithm'' is the strategy of always picking whatever dish looks most appealing right now, without worrying about what comes next. You never put food back, never rearrange, never plan ahead — you just grab the best thing available at each step. Surprisingly, this myopic strategy produces optimal results for a wide class of problems, from building cheapest networks to compressing data. The catch: for other problems, the locally best choice leads to a globally terrible outcome. Knowing which problems yield to greed — and proving it — is the core of greedy algorithm theory.

!! The Core Insight

The power of greedy algorithms rests on two properties. The ''greedy choice property'' says that a locally optimal choice can always be extended to a globally optimal solution — you never need to undo a decision. ''Optimal substructure'' says that after making the greedy choice, the remaining problem is a smaller instance of the same type. Together, these properties mean you can build the solution incrementally, one irrevocable decision at a time, and end up at the global optimum.

The standard proof technique is the ''exchange argument'': assume an optimal solution that does //not// include the greedy choice, then show you can swap elements to include it without worsening the objective. If the exchange always works, greedy is optimal. This is fundamentally different from [[Dynamic Programming|Dynamic Programming]], which explores all subproblems. Greedy commits to one path and never looks back — when it works, it is dramatically faster.

!! The Greedy Paradigm in Action

''Activity selection'' is the textbook example: given $n$ activities with start and finish times, find the maximum number of non-overlapping activities. The greedy strategy — always pick the activity that finishes earliest — produces an optimal schedule. Why? Finishing early leaves the most room for future activities. The exchange argument: any optimal solution using a later-finishing activity can swap it for the earlier one without losing any other activities.

''Huffman coding'' builds an optimal prefix-free binary code by repeatedly merging the two least-frequent symbols. This greedy merge produces a tree where frequent symbols get short codes and rare symbols get long codes, minimizing expected code length. The proof uses the exchange argument on tree structure: swapping a deep frequent symbol with a shallow rare one always reduces cost.

''Kruskal's algorithm'' for [[Minimum Spanning Trees|Minimum Spanning Trees]] sorts all edges by weight and greedily adds the lightest edge that does not create a cycle. ''Prim's algorithm'' grows a tree from a starting vertex, always attaching the nearest unvisited vertex. Both are greedy, both are optimal — because spanning trees have matroid structure.

''Dijkstra's algorithm'' for [[Shortest Path Algorithms|Shortest Path Algorithms]] greedily expands the unvisited vertex with the smallest known distance. With non-negative edge weights, this produces shortest paths because relaxing closer vertices first can never be improved by a longer detour.

!! What the Math Is Actually Doing

A ''matroid'' $(E, \mathcal{I})$ is a set $E$ with a collection of "independent" subsets $\mathcal{I}$ satisfying three axioms: (1) $\emptyset \in \mathcal{I}$, (2) if $A \subseteq B$ and $B \in \mathcal{I}$ then $A \in \mathcal{I}$ (hereditary), and (3) if $|A| < |B|$ and both are in $\mathcal{I}$, then there exists $e \in B \setminus A$ such that $A \cup \{e\} \in \mathcal{I}$ (exchange axiom). The foundational theorem:

$$\text{Greedy is optimal for maximizing a linear objective over a matroid.}$$

More precisely, if you want to find a maximum-weight independent set in a matroid, the greedy algorithm — repeatedly add the highest-weight element that keeps the set independent — always succeeds. This is the ''Rado-Edmonds theorem'', and it completely characterizes when greedy works for independence systems: greedy is optimal for //all// weight functions if and only if the independence system is a matroid.

For the ''fractional knapsack'', the greedy criterion is value-to-weight ratio. Sort items by $v_i / w_i$ in decreasing order and fill the knapsack:

$$\text{Greedy picks items by } \frac{v_i}{w_i} \geq \frac{v_{i+1}}{w_{i+1}}$$

This is optimal because you can take fractions. The ''0/1 knapsack'' — where items are indivisible — breaks the greedy choice property and requires [[Dynamic Programming|Dynamic Programming]] or [[Backtracking|Backtracking]].

For Huffman coding, the objective is minimizing expected code length $\sum_i f_i \cdot d_i$, where $f_i$ is frequency and $d_i$ is code depth. The greedy merge minimizes this because the two least-frequent symbols should be deepest in any optimal tree — proved by contradiction via the exchange argument.

!! Greedy Algorithms in ML and AI

Greedy strategies appear throughout machine learning, often as practical heuristics rather than provably optimal algorithms.

''Decision tree construction'' (ID3, C4.5, CART) is greedy: at each node, pick the feature split that maximizes information gain (or minimizes Gini impurity). This is greedy because you never revisit a split. The resulting tree is not globally optimal — finding the optimal decision tree is NP-hard — but greedy splitting works remarkably well in practice.

''Forward feature selection'' greedily adds the feature that most improves model performance. ''Backward elimination'' greedily removes the least useful feature. Neither guarantees the optimal feature subset, but both are widely used because exhaustive search over $2^p$ subsets is intractable.

''Beam search'' in sequence models (machine translation, speech recognition) is a bounded-width greedy search. Pure greedy decoding picks the single highest-probability token at each step; beam search keeps the top $k$ candidates, a controlled relaxation of pure greed.

''Matching pursuit'' and ''orthogonal matching pursuit'' greedily select dictionary atoms to approximate a signal — used in compressed sensing and sparse coding. Each step picks the atom most correlated with the current residual.

!! When Greedy Algorithms Break Down

''The 0/1 knapsack trap.'' Greedy by value-to-weight ratio fails for indivisible items. Consider items with values $(60, 100, 120)$ and weights $(10, 20, 30)$ with capacity $50$. Greedy picks by ratio: item 1 (ratio 6), item 2 (ratio 5), leaving weight $20$ and total value $160$. But the optimal solution takes items 2 and 3 for value $220$. The problem lacks the greedy choice property because you cannot take fractions — a high-ratio item may block a better combination. This is exactly where [[Dynamic Programming|Dynamic Programming]] or [[Divide and Conquer|Divide and Conquer]] strategies become necessary.

''Coin change with non-standard denominations.'' With US coins $(1, 5, 10, 25)$, greedy (largest coin first) is optimal. But with coins $\{1, 3, 4\}$ and target $6$, greedy picks $4 + 1 + 1 = 3$ coins, while optimal is $3 + 3 = 2$ coins. The greedy choice property fails because the denomination system lacks the matroid-like structure that US coins happen to have. There is no general way to know if a coin system is "greedy-safe" without checking — another case requiring dynamic programming.

''Graph coloring and TSP.'' Greedy graph coloring (assign each vertex the smallest available color) can use arbitrarily more colors than necessary — the approximation ratio is $\Theta(n / \log n)$ in the worst case. For the Travelling Salesman Problem without the triangle inequality, no polynomial-time algorithm achieves any bounded approximation ratio (unless P=NP). These problems lack both the greedy choice property and the matroid structure that would guarantee greedy optimality. The lesson: greedy gives you speed but no quality guarantee unless you can //prove// the structural properties hold.

!! Technical Details

''Complexity:'' Most greedy algorithms run in $O(n \log n)$ time, dominated by an initial sort. Kruskal's MST: $O(E \log E)$ with union-find. Prim's MST: $O(E \log V)$ with a binary heap, $O(E + V \log V)$ with Fibonacci heap. Dijkstra: same as Prim. Huffman: $O(n \log n)$ with a priority queue. Activity selection: $O(n \log n)$ for sorting, then $O(n)$ scan.

''Key references:'' Edmonds (1971) — matroid theory and greedy optimality. Huffman (1952) — optimal prefix codes. Kruskal (1956) and Prim (1957) — minimum spanning trees. Dijkstra (1959) — shortest paths. Cormen, Leiserson, Rivest, Stein (CLRS), Chapter 16 — canonical textbook treatment.

''Relationship to other paradigms:'' Greedy is strictly simpler than [[Dynamic Programming|Dynamic Programming]] (which considers all subproblems), [[Divide and Conquer|Divide and Conquer]] (which splits and recombines), and [[Backtracking|Backtracking]] (which explores and undoes). When a problem has matroid structure or provable greedy choice property, always prefer greedy — it is faster and simpler. When it does not, greedy becomes a heuristic with no guarantees.

''Connections:'' [[Minimum Spanning Trees|Minimum Spanning Trees]] — Kruskal and Prim are canonical greedy algorithms. [[Shortest Path Algorithms|Shortest Path Algorithms]] — Dijkstra is greedy over vertex distances. [[Divide and Conquer|Divide and Conquer]] — alternative paradigm for problems without greedy structure. [[Backtracking|Backtracking]] — explores solution space that greedy ignores.
