! Graph Coloring

<img src="assets/graph_coloring.svg" alt="Graph with five vertices colored using three colors so no adjacent vertices share a color" style="width:100%;max-width:620px;display:block;margin:16px auto 24px auto;">

Think about seating rivals at a dinner party. You have a table, a guest list, and one ironclad rule: no two people who dislike each other may sit in the same section. You want to use as few sections as possible. ''Graph coloring'' is exactly this problem dressed in mathematical clothing. Each guest is a vertex, each rivalry is an edge, and each section is a color. The question becomes: what is the //fewest// number of colors you can assign to vertices so that no edge connects two vertices of the same color? This deceptively simple question turns out to be one of the deepest problems in [[Graph Theory|Graph Theory]] and [[Combinatorics|Combinatorics]], with direct consequences for compiler design, wireless networks, and scheduling.

!! The Core Insight

A ''proper coloring'' of a graph $G$ is an assignment of colors to vertices such that no two adjacent vertices receive the same color. The ''chromatic number'' $\chi(G)$ is the smallest number of colors needed for a proper coloring. This single number captures a fundamental structural property of the graph — it measures how "tangled" the adjacency relationships are. A graph with $\chi(G) = 2$ is ''bipartite'': its vertices split cleanly into two independent groups. A graph requiring many colors has dense, interleaved conflicts that cannot be separated easily. Determining $\chi(G)$ for an arbitrary graph is one of the canonical problems in [[NP-Completeness|NP-Completeness]].

!! Coloring Without Conflicts

''The Chromatic Number.'' For complete graphs, $\chi(K_n) = n$ — every vertex is adjacent to every other, so each needs its own color. For cycles, $\chi(C_n) = 2$ if $n$ is even and $3$ if $n$ is odd. For trees, $\chi(T) = 2$ always — trees are bipartite. These special cases are easy, but finding $\chi(G)$ for general graphs is NP-hard.

''Greedy Coloring.'' The simplest coloring algorithm processes vertices one at a time, assigning each the smallest color not already used by its neighbors. This [[Greedy Algorithms|greedy approach]] always produces a valid coloring, but not necessarily an optimal one. The number of colors used depends on the vertex ordering. In the worst case, greedy coloring on a graph with chromatic number 2 can use up to $n/2$ colors with a bad ordering. However, there always //exists// an ordering for which greedy uses exactly $\chi(G)$ colors.

''The Four Color Theorem.'' Every ''planar graph'' — any graph that can be drawn in the plane without edge crossings — can be properly colored with at most four colors. Conjectured in 1852 by Francis Guthrie, it was finally proved in 1976 by Appel and Haken using a computer-assisted proof that checked 1,936 configurations. This remains one of the most famous results about [[Planar Graphs|Planar Graphs]], and the first major theorem proved with essential computer assistance.

''Edge Coloring.'' While vertex coloring assigns colors to nodes, ''edge coloring'' assigns colors to edges so that no two edges sharing an endpoint have the same color. Vizing's theorem states that the number of edge colors needed (the ''chromatic index'' $\chi'(G)$) is either $\Delta(G)$ or $\Delta(G) + 1$, where $\Delta(G)$ is the maximum vertex degree. Deciding which case applies is itself NP-complete.

!! What the Math Is Actually Doing

The ''chromatic polynomial'' $P(G, k)$ counts the number of proper colorings of $G$ using exactly $k$ colors. For a complete graph $K_n$:

$$P(K_n, k) = k(k-1)(k-2) \cdots (k - n + 1)$$

For a tree $T$ on $n$ vertices:

$$P(T, k) = k(k-1)^{n-1}$$

The chromatic polynomial satisfies a ''deletion-contraction'' recurrence. For any edge $e = (u,v)$:

$$P(G, k) = P(G - e, k) - P(G / e, k)$$

where $G - e$ deletes the edge and $G / e$ contracts it (merges $u$ and $v$ into one vertex). This recurrence is the engine behind many exact coloring algorithms, though it runs in exponential time.

''Bounds on $\chi(G)$:'' The chromatic number is squeezed between the clique number $\omega(G)$ (the largest complete subgraph) and the number of vertices: $\omega(G) \leq \chi(G) \leq n$. A tighter upper bound comes from ''Brook's theorem'': for any connected graph that is neither a complete graph nor an odd cycle, $\chi(G) \leq \Delta(G)$, where $\Delta(G)$ is the maximum degree. The ''fractional chromatic number'' and ''Lov&#225;sz theta function'' provide further bounds computable in polynomial time.

!! Graph Coloring in CS and ML

''Register allocation.'' Compilers must assign variables to CPU registers. Two variables that are "live" at the same time cannot share a register. Build an ''interference graph'' where variables are vertices and edges connect simultaneously live variables, then color it. The number of colors equals the number of registers needed. This is the single most important application of graph coloring in computer science.

''Scheduling.'' Exam timetabling is graph coloring in disguise: exams are vertices, two exams share an edge if a student takes both, and colors are time slots. Minimum coloring minimizes the number of time slots. Job-shop scheduling, meeting room assignment, and sports tournament scheduling all reduce to coloring variants.

''Frequency assignment.'' In wireless networks, nearby transmitters that broadcast on the same frequency interfere. Model transmitters as vertices, proximity as edges, and frequencies as colors. Minimizing the number of frequencies is a graph coloring problem. This application extends to channel assignment in cellular networks and Wi-Fi access points.

''Map coloring and visualization.'' The original motivation: coloring regions of a map so that neighboring countries get different colors. Geographic information systems, data visualization, and cartography all rely on graph coloring to produce legible, conflict-free displays.

''Constraint satisfaction in ML.'' Many constraint satisfaction problems in ML pipelines — feature group assignments, parallel task scheduling in distributed training, and conflict-free hashing — can be formulated as graph coloring. Graph neural networks have recently been applied //to// the coloring problem itself, learning heuristics that outperform classical greedy approaches on certain graph families.

!! When Coloring Approaches Break Down

''1. NP-hardness bites for large, dense graphs.'' Determining whether a graph is 3-colorable is NP-complete. For sparse or structured graphs (planar, interval, chordal), polynomial algorithms exist, but general instances with hundreds of thousands of vertices and high density remain intractable for exact methods. Greedy and heuristic approaches give valid colorings but may use far more colors than necessary.

''2. Greedy coloring is order-dependent.'' The greedy algorithm's quality is entirely determined by vertex ordering. Random orderings give no worst-case guarantees. Sophisticated orderings (largest-first, smallest-last, DSatur) improve practical performance but cannot overcome the fundamental gap between greedy and optimal. A graph with $\chi(G) = 3$ can fool greedy into using $O(\sqrt{n})$ colors.

''3. The chromatic number ignores weighted and list variants.'' Real-world problems often involve //list coloring// (each vertex has a restricted set of available colors) or //weighted coloring// (colors have costs). The chromatic number $\chi(G)$ does not capture these constraints. The ''list chromatic number'' $\chi_\ell(G)$ can be strictly larger than $\chi(G)$, and weighted variants introduce additional NP-hard layers. Algorithms tuned for standard coloring may perform poorly on these generalizations.

!! Technical Details

''Exact algorithms:'' The fastest known exact algorithm for $\chi(G)$ runs in $O(2.2461^n)$ time via inclusion-exclusion over independent sets. Dynamic programming over subsets achieves $O^*(2^n)$ with exponential space.

''Approximation hardness:'' Unless P = NP, the chromatic number cannot be approximated within a factor of $n^{1-\epsilon}$ for any $\epsilon > 0$. This is one of the strongest inapproximability results known. In practice, heuristics (tabu search, simulated annealing, evolutionary algorithms) are the workhorses.

''Special graph classes:'' ''Chordal graphs'' (every cycle of length $\geq 4$ has a chord) can be optimally colored in linear time using a perfect elimination ordering. ''Interval graphs'' (intersection graphs of intervals on a line) are chordal and thus also easy to color. ''Perfect graphs'' satisfy $\chi(H) = \omega(H)$ for every induced subgraph $H$ — the Strong Perfect Graph Theorem (Chudnovsky et al., 2006) characterizes them as graphs with no odd holes or odd antiholes.
