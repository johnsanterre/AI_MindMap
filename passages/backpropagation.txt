! Backpropagation
[img[assets/backpropagation.svg]]

Imagine you are an archery coach watching a student miss the bullseye. You do not just say "you missed" — you trace the error backward through the chain of causes: the arrow drifted left because the release was off, the release was off because the elbow dropped, the elbow dropped because of a weak shoulder position. Each correction is proportional to how much that link contributed to the miss. ''Backpropagation'' does exactly this for neural networks: it takes the final prediction error and traces it backward through every layer, computing how much each weight contributed to the mistake so that each weight can be nudged in the right direction.

!! The Core Insight

A neural network is a chain of simple, differentiable functions composed together. The output error (the ''loss'') depends on every weight in the network, but computing each weight's contribution directly would be impossibly expensive. Backpropagation exploits the ''chain rule of calculus'' to decompose this enormous derivative computation into a sequence of small, local multiplications — one per layer — that flow backward from the output to the input. Each layer only needs to know its own local derivative and the gradient arriving from the layer above.

The result is that computing gradients for //all// weights costs roughly the same as a single forward pass — an insight that transformed neural networks from a curiosity into a practical learning algorithm. Related: <<link "Gradient Descent" "Gradient Descent">><</link>>, <<link "Chain Rule" "Chain Rule">><</link>>, <<link "Loss Function" "Loss Function">><</link>>.

!! The Chain Rule in Action

Consider a three-layer network: input $x$ passes through layers with weights $W_1$, $W_2$, $W_3$ and activation functions $\sigma$ to produce output $\hat{y}$:

$$\hat{y} = \sigma(W_3 \cdot \sigma(W_2 \cdot \sigma(W_1 \cdot x)))$$

The ''forward pass'' computes this left-to-right, storing each intermediate activation $h_i$. The ''backward pass'' then applies the chain rule in reverse. To find how the loss $L$ changes with respect to $W_1$ (buried deep in the network), we decompose:

$$\frac{\partial L}{\partial W_1} = \frac{\partial L}{\partial \hat{y}} \cdot \frac{\partial \hat{y}}{\partial h_3} \cdot \frac{\partial h_3}{\partial h_2} \cdot \frac{\partial h_2}{\partial h_1} \cdot \frac{\partial h_1}{\partial W_1}$$

Each factor is a ''local derivative'' — it depends only on the values at that layer. The backward pass computes these factors from right to left, reusing intermediate results so that no derivative is computed twice. This reuse is what makes backpropagation efficient: instead of $O(W \cdot N)$ cost (recomputing the full chain for each weight), we get $O(W + N)$.

!! What the Math Is Actually Doing

Let $L$ be the loss, $h_l$ the activation at layer $l$, and $W_l$ the weight matrix at layer $l$. Define the ''error signal'' (or ''delta'') at layer $l$:

$$\delta_l = \frac{\partial L}{\partial h_l}$$

The backward pass computes $\delta_l$ recursively from the output layer back to the input:

$$\delta_L = \frac{\partial L}{\partial \hat{y}} \cdot \sigma'(z_L)$$

$$\delta_l = (W_{l+1}^T \cdot \delta_{l+1}) \odot \sigma'(z_l)$$

where $z_l = W_l \cdot h_{l-1}$ is the pre-activation and $\odot$ denotes element-wise multiplication. The gradient for each weight matrix is then:

$$\frac{\partial L}{\partial W_l} = \delta_l \cdot h_{l-1}^T$$

This is an outer product of the error signal at layer $l$ with the activation from the layer below — geometrically, it says: "adjust the weight connecting input $h_{l-1}$ to output $h_l$ in proportion to both the input's magnitude and the error's magnitude."

The weight update rule is then: $W_l \leftarrow W_l - \eta \cdot \frac{\partial L}{\partial W_l}$ where $\eta$ is the learning rate.

!! Computational Graphs and Autodiff

Modern deep learning frameworks do not implement backpropagation by hand. Instead, they build a ''computational graph'' — a directed acyclic graph where each node is an operation (add, multiply, apply ReLU) and edges carry tensors. The forward pass evaluates the graph from inputs to loss. The backward pass traverses the graph in reverse topological order, applying the chain rule at each node.

This is ''reverse-mode automatic differentiation'' (autodiff). Each primitive operation (addition, multiplication, exp, log) registers a small "backward function" that knows how to compute its local derivative. The framework chains these together automatically. This means you never write gradient code — you define the forward computation, and the framework derives the backward pass.

Key insight: reverse-mode autodiff computes the gradient of a scalar output (the loss) with respect to //all// parameters in a single backward pass — regardless of how many parameters there are. This is why it is preferred over forward-mode autodiff for training neural networks, where we have one scalar loss but millions of parameters. Related: <<link "Automatic Differentiation" "Automatic Differentiation">><</link>>.

!! When Backpropagation Breaks Down

''Vanishing gradients:'' When activation functions like sigmoid or tanh saturate, their derivatives approach zero. The chain rule multiplies these near-zero factors across layers, causing gradients in early layers to become negligibly small. The network's first layers stop learning entirely. Fixes include ReLU activations, residual connections, and careful initialization (He or Xavier). See <<link "Vanishing and Exploding Gradients" "Vanishing and Exploding Gradients">><</link>>.

''Exploding gradients:'' The opposite problem — if weight matrices have spectral norms greater than 1, repeated multiplication amplifies gradients exponentially. Loss diverges to infinity. The standard fix is ''gradient clipping'': cap the gradient norm to a maximum value before the weight update. LSTMs and GRUs were designed specifically to manage gradient flow through long sequences.

''Non-differentiable operations:'' Backpropagation requires every operation in the computational graph to have a well-defined (sub-)gradient. Operations like hard thresholding, argmax, or discrete sampling break the chain. Workarounds include the straight-through estimator (pretend the gradient of a step function is 1), the Gumbel-softmax trick (replace discrete sampling with a differentiable approximation), and REINFORCE-style policy gradient estimators.

!! Technical Details

''History:'' The chain rule itself dates to Leibniz (1676). Automatic differentiation was formalized by Wengert (1964). Backpropagation for neural networks was independently discovered multiple times — Bryson and Ho (1969), Werbos (1974), and Parker (1985) — but the landmark paper by Rumelhart, Hinton, and Williams (1986) demonstrated it convincingly and ignited the neural network renaissance.

''Computational cost:'' One backward pass costs approximately 2x the forward pass (each operation computes one forward value and one gradient, but the gradient computation often involves an extra multiplication). Total training iteration = 1 forward + 1 backward = ~3x the cost of forward inference.

''Memory cost:'' The backward pass needs all intermediate activations from the forward pass. For a network with N layers and batch size B, this requires O(N * B * D) memory where D is the hidden dimension. Gradient checkpointing trades compute for memory by recomputing activations instead of storing them.

''Second-order methods:'' Backpropagation computes first derivatives (the Jacobian-vector product). Computing second derivatives (the Hessian) costs O(P) additional backward passes for P parameters, making exact second-order optimization infeasible for large networks. Approximations like K-FAC, Adam's diagonal Hessian estimate, and Hessian-free optimization make second-order information tractable.

''Backpropagation through time (BPTT):'' For recurrent networks (RNNs, LSTMs), the computational graph is "unrolled" across time steps, and backpropagation proceeds through this unrolled graph. Truncated BPTT limits the number of time steps to bound memory and computation, at the cost of ignoring long-range temporal dependencies.
