! Offline RL

<img src="assets/offline_rl.svg" alt="Diagram illustrating Offline RL learning from fixed datasets with conservative methods" style="width:100%;max-width:620px;display:block;margin:16px auto 24px auto;">

Imagine you inherit a filing cabinet stuffed with medical records — thousands of patients, treatments tried, outcomes observed — but you can never run a new clinical trial. You must figure out the best treatment protocol purely from what previous doctors already did. ''Offline RL'' (also called ''Batch RL'') faces exactly this challenge: learning the best possible policy from a fixed dataset of past interactions, with no ability to collect new experience by interacting with the environment. The agent must learn from what happened, not from what //could// happen if it tried something new.

---

!! The Core Insight

Standard <<link "Reinforcement Learning" "Reinforcement Learning">><</link>> algorithms improve by acting in the world, observing outcomes, and adjusting. They depend on a tight feedback loop: try action, see result, update policy, repeat. Offline RL breaks this loop entirely. The agent receives a static dataset $\mathcal{D} = \{(s_i, a_i, r_i, s'_i)\}$ collected by some unknown ''behavior policy'' $\pi_\beta$ and must extract the best policy it can without ever querying the environment again.

This matters enormously in practice. In healthcare, autonomous driving, industrial control, and dialogue systems, letting an untrained agent explore freely is dangerous, expensive, or impossible. Offline RL promises to unlock the value trapped in the massive logs of historical data that already exist in these domains.

The fundamental obstacle is ''distribution shift'': the learned policy $\pi$ will want to take actions that the behavior policy $\pi_\beta$ never tried. Standard <<link "Q-Learning" "Q Learning">><</link>> happily assigns high Q-values to these unseen actions because there is no data to contradict an optimistic estimate. Without the corrective feedback of actually trying those actions and observing failure, the Q-function becomes an unreliable fantasist — confidently recommending actions it knows nothing about.

---

!! Conservative and Pessimistic Approaches

The field converged on a core principle: ''be pessimistic about what you haven't seen''. If the data doesn't cover an action, assume it's bad. Three landmark methods embody this idea in different ways.

''Conservative Q-Learning (CQL)'' (Kumar et al., 2020) directly penalises the Q-function for assigning high values to out-of-distribution actions. It adds a regulariser that pushes down Q-values for actions sampled from the current policy while pushing up Q-values for actions actually observed in the dataset. The result is a Q-function that provably lower-bounds the true Q-function on the data distribution — a conservative estimate that prevents the policy from chasing phantom value.

''Implicit Q-Learning (IQL)'' (Kostrikov et al., 2022) takes a subtler approach: it avoids ever querying the Q-function on actions outside the dataset. Instead of computing $\max_a Q(s, a)$ (which requires evaluating unseen actions), IQL uses ''expectile regression'' to approximate the maximum Q-value using only in-dataset actions. By fitting the value function to the upper quantiles of the Q-distribution, IQL implicitly extracts a good policy without explicit out-of-distribution queries.

''Decision Transformer'' (Chen et al., 2021) abandons the <<link "Bellman Equation" "Bellman Equation">><</link>> framework altogether. It reframes RL as a sequence modeling problem: given a desired return, the model autoregressively generates the sequence of actions that achieves it. Built on the Transformer architecture, it conditions on (return-to-go, state, action) tuples and simply predicts the next action. No Q-functions, no temporal difference learning — just supervised prediction on trajectories, conditioned on the desired outcome.

---

!! What the Math Is Actually Doing

Standard Q-learning updates the Q-function using the <<link "Bellman Equation" "Bellman Equation">><</link>>:

$$Q(s, a) \leftarrow r + \gamma \max_{a'} Q(s', a')$$

The $\max_{a'}$ operator is the source of trouble in offline RL: it queries the Q-function at actions $a'$ that may never appear in $\mathcal{D}$, producing unchecked overestimation.

''CQL'' adds a conservative penalty to the standard Bellman backup. The CQL objective minimises:

$$\mathcal{L}_{\text{CQL}} = \alpha \left(\mathbb{E}_{s \sim \mathcal{D}, a \sim \mu}[Q(s, a)] - \mathbb{E}_{s, a \sim \mathcal{D}}[Q(s, a)]\right) + \frac{1}{2}\mathbb{E}_{(s,a,r,s') \sim \mathcal{D}}\!\bigl[(Q(s,a) - \hat{\mathcal{B}}^\pi Q(s,a))^2\bigr]$$

where $\mu$ is a sampling distribution (often the current policy) and $\hat{\mathcal{B}}^\pi$ is the empirical Bellman operator. The first term pushes Q down on $\mu$-sampled actions and up on dataset actions; the second is the standard Bellman error. The hyperparameter $\alpha$ controls conservatism.

''IQL'' replaces the problematic $\max$ with expectile regression. Define the expectile loss:

$$L_\tau(u) = |\tau - \mathbb{1}(u < 0)| \cdot u^2$$

IQL fits a value function $V(s)$ by minimising $\mathbb{E}_{(s,a) \sim \mathcal{D}}[L_\tau(Q(s,a) - V(s))]$ with $\tau \in (0.5, 1)$. As $\tau \to 1$, $V(s)$ converges to $\max_{a \in \mathcal{D}} Q(s, a)$ — the maximum over //observed// actions only, never venturing out of distribution.

''Decision Transformer'' optimises a standard cross-entropy or MSE loss on action prediction:

$$\mathcal{L}_{\text{DT}} = \mathbb{E}_{\tau \sim \mathcal{D}}\!\left[\sum_t \|a_t - f_\theta(\hat{R}_t, s_t, \hat{R}_{t-1}, s_{t-1}, a_{t-1}, \ldots)\|^2\right]$$

where $\hat{R}_t = \sum_{t'=t}^T r_{t'}$ is the return-to-go. At inference, the practitioner sets a high target return and the model generates actions conditioned on achieving it.

!! Practical Applications and Deployment

Offline RL has found traction in domains where online exploration is costly or dangerous.

''Healthcare:'' Treatment policy optimisation from electronic health records. The dataset contains years of clinician decisions and patient outcomes. An offline RL agent can learn treatment strategies that outperform average clinician behavior without experimenting on patients.

''Robotics:'' Large-scale robot learning from demonstration datasets. Projects like RT-2 and RoboMimic leverage offline RL to train manipulation policies from pre-collected trajectories, then fine-tune with limited online interaction.

''Recommender systems:'' Platforms log billions of user interactions daily. Offline RL can optimise long-term user engagement from these logs without the A/B testing cost of deploying untested policies to live users.

''Dialogue and language:'' The connection to language models is deep — RLHF pipelines for LLMs use offline preference data, and Decision Transformer-style approaches blur the line between RL and supervised sequence modeling. <<link "Policy Gradient" "Policy Gradient">><</link>> methods like PPO used in RLHF can also be viewed through an offline RL lens when the policy is constrained to stay near the reference model.

---

!! When X Breaks Down

''Insufficient data coverage:'' If the behavior policy $\pi_\beta$ never visited important regions of state-action space, no amount of pessimism can recover a good policy for those regions. Conservative methods will correctly avoid those actions — but "correctly cautious" may still mean "mediocre." The quality of the learned policy is fundamentally bounded by the quality and diversity of the dataset. A dataset of only bad driving cannot teach good driving; it can only teach "the least bad option among bad options."

''Extrapolation error compounding:'' Even with conservative methods, small errors in Q-value estimation compound over long horizons through the Bellman backup. Each bootstrapped step propagates and amplifies approximation errors. In long-horizon tasks with hundreds of timesteps, this compounding can overwhelm the conservative penalty, causing the learned policy to degrade far below the behavior policy. <<link "Distribution Shift" "Distribution Shift">><</link>> becomes especially severe when the dataset trajectories are short but the evaluation horizon is long.

''Conservatism-performance tradeoff:'' Too much pessimism and the agent refuses to deviate from the behavior policy at all — it just imitates $\pi_\beta$, gaining nothing over behavioral cloning. Too little pessimism and the agent hallucinates value for unseen actions and fails catastrophically at deployment. The hyperparameter $\alpha$ in CQL or $\tau$ in IQL controls this tradeoff, but tuning it requires either domain knowledge or a held-out evaluation mechanism, which itself requires data coverage assumptions.

---

!! Technical Details

''Theoretical foundations:'' Offline RL has connections to robust MDPs and distributionally robust optimization. CQL provides PAC-style guarantees: under certain data coverage assumptions, the learned policy's value is a lower bound on the true value with high probability. IQL provides implicit pessimism without explicit penalty terms, relying instead on expectile regression properties.

''Benchmarks:'' D4RL (Fu et al., 2020) is the standard benchmark suite, providing datasets of varying quality (random, medium, expert, medium-expert, medium-replay) across MuJoCo locomotion and Atari tasks. Performance is typically reported as normalised score relative to the behavior policy and an expert.

''Comparison of approaches:'' CQL and IQL are value-based methods that maintain Q-functions and derive policies via advantage-weighted regression or similar. Decision Transformer is model-free in a different sense — it requires no value function at all, but it cannot stitch together sub-optimal trajectories (it can only reproduce behavior quality present in the data). Value-based methods can //stitch//: combine the best parts of different trajectories to synthesize a policy better than any single trajectory in the dataset.

''Online fine-tuning:'' Modern offline-to-online pipelines initialise with offline RL, then fine-tune with limited online interaction. IQL is particularly well-suited to this because its learned Q-function transfers smoothly. Cal-QL extends CQL for the online fine-tuning setting. This hybrid approach often achieves the best final performance.

''Relationship to imitation learning:'' Behavioral cloning is the simplest offline approach: just do supervised learning on actions. Offline RL goes beyond this by using reward information to improve upon the behavior policy. When the dataset is expert-quality, behavioral cloning suffices; when the dataset is mixed-quality, offline RL methods that can //stitch// trajectories have a decisive advantage.

''Key papers:'' Levine et al., "Offline Reinforcement Learning: Tutorial, Review, and Perspectives on Open Problems" (2020); Kumar et al., "Conservative Q-Learning for Offline Reinforcement Learning" (NeurIPS 2020); Kostrikov et al., "Offline Reinforcement Learning with Implicit Q-Learning" (ICLR 2022); Chen et al., "Decision Transformer" (NeurIPS 2021).
