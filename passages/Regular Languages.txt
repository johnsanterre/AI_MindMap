! Regular Languages

<img src="assets/regular_languages.svg" alt="Regular Languages" width="800">

Imagine a security guard at a building entrance who can only remember //one thing// at a time: which room they are currently standing in. They follow simple rules — "if I see badge type A, move to the lobby; if I see badge type B, move to the checkpoint" — and at the end, they either let you in or turn you away based on which room they ended up in. That guard, with no notepad, no memory of what came before, just a current location and a rulebook, is a ''finite automaton''. The set of all badge sequences that get you through the door is a ''regular language''. These are the simplest interesting class of formal languages — powerful enough to describe patterns in text, network protocols, and lexical analysis, yet simple enough that every question about them can be answered by a machine with finite memory and no stack, no tape, no recursion.

!! The Core Insight

Regular languages sit at the bottom of the ''Chomsky hierarchy'' — the four-level classification of formal languages by the computational power needed to recognize them. What makes regular languages special is a remarkable three-way equivalence proven by Stephen Kleene in 1956: a language is regular if and only if it can be described by a ''regular expression'', if and only if it can be recognized by a ''deterministic finite automaton'' (DFA), if and only if it can be recognized by a ''nondeterministic finite automaton'' (NFA). Three completely different formalisms — algebraic (regex), machine-based (DFA), and nondeterministic (NFA) — describe exactly the same class of languages.

This equivalence is not obvious. A regular expression like $(a|b)^*abb$ is a compact algebraic description. A DFA is a concrete machine with states and transitions. An NFA can be in //multiple states simultaneously//. Yet they all capture the same computational boundary: patterns recognizable with ''finite memory'' — a fixed number of states, independent of input length. No counting, no nesting, no unbounded storage. This limitation is both their weakness and their strength: because they are so constrained, almost every question about regular languages is ''decidable'' — membership, emptiness, equivalence, containment — and decidable //efficiently//.

!! The Chomsky Hierarchy and Where Regular Languages Fit

The ''Chomsky hierarchy'' (Noam Chomsky, 1956) classifies formal languages into four types based on the grammar rules that generate them and the machines that recognize them:

''Type 3 — Regular languages.'' Generated by ''right-linear grammars'' (rules of the form $A \to aB$ or $A \to a$). Recognized by finite automata. No memory beyond the current state. Examples: identifiers in programming languages, email address patterns, the set of binary strings divisible by 3.

''Type 2 — Context-free languages.'' Generated by ''context-free grammars'' (rules of the form $A \to \gamma$ where $\gamma$ is any string of terminals and nonterminals). Recognized by pushdown automata (finite automata plus a stack). Can handle nested structures like balanced parentheses, arithmetic expressions, and programming language syntax. Strictly more powerful than regular languages.

''Type 1 — Context-sensitive languages.'' Generated by grammars where the left side can include context ($\alpha A \beta \to \alpha \gamma \beta$). Recognized by linear-bounded automata. Can handle cross-serial dependencies like $\{a^n b^n c^n \mid n \geq 1\}$.

''Type 0 — Recursively enumerable languages.'' Generated by unrestricted grammars. Recognized by Turing machines. The broadest class — includes undecidable languages where membership cannot always be determined.

Each level strictly contains the one below: every regular language is context-free, every context-free language is context-sensitive, and every context-sensitive language is recursively enumerable. The strict containment means there exist context-free languages that are //not// regular — the classic example being $\{a^n b^n \mid n \geq 1\}$, which requires counting and therefore unbounded memory.

!! What the Math Is Actually Doing

A ''deterministic finite automaton'' (DFA) is a 5-tuple $M = (Q, \Sigma, \delta, q_0, F)$ where:

$$Q = \text{finite set of states}$$
$$\Sigma = \text{finite input alphabet}$$
$$\delta: Q \times \Sigma \to Q = \text{transition function}$$
$$q_0 \in Q = \text{start state}$$
$$F \subseteq Q = \text{set of accepting states}$$

The machine reads an input string $w = w_1 w_2 \ldots w_n$ one symbol at a time, transitioning from state to state according to $\delta$. It accepts $w$ if the final state $\delta(q_0, w) \in F$. The language of $M$ is $L(M) = \{w \in \Sigma^* \mid \delta(q_0, w) \in F\}$.

A ''nondeterministic finite automaton'' (NFA) generalizes this: $\delta: Q \times (\Sigma \cup \{\varepsilon\}) \to \mathcal{P}(Q)$ maps to //sets// of states, and the NFA accepts if //any// computation path ends in an accepting state. Despite this seeming extra power, the ''subset construction'' converts any NFA with $n$ states into an equivalent DFA with at most $2^n$ states — proving NFA and DFA recognize exactly the same languages.

''Regular expressions'' build languages from the empty string $\varepsilon$, individual symbols, and three operations:

$$\text{Union: } L_1 \cup L_2 \quad \text{Concatenation: } L_1 \cdot L_2 \quad \text{Kleene star: } L^* = \bigcup_{i=0}^{\infty} L^i$$

''Kleene's theorem'' states: a language is accepted by a DFA if and only if it is described by a regular expression. The proof constructs a regex from a DFA by systematic state elimination, and a DFA from a regex by composing small NFAs via Thompson's construction.

The ''pumping lemma'' is the primary tool for proving a language is //not// regular. For any regular language $L$, there exists a pumping length $p$ such that any string $s \in L$ with $|s| \geq p$ can be written as $s = xyz$ where: (1) $|y| > 0$, (2) $|xy| \leq p$, and (3) for all $i \geq 0$, $xy^iz \in L$. If you can find a string that //cannot// be pumped, the language is not regular.

The ''Myhill-Nerode theorem'' gives an exact characterization: a language $L$ is regular if and only if the equivalence relation $\equiv_L$ (where $x \equiv_L y$ iff for all $z$, $xz \in L \Leftrightarrow yz \in L$) has finitely many equivalence classes. The number of classes equals the number of states in the ''minimal DFA'' — the unique smallest DFA recognizing $L$.

!! Regular Languages in Practice

Regular languages are everywhere in computing, even if they are rarely called by that name:

''Lexical analysis.'' Every compiler's first phase — the lexer or scanner — uses DFAs to tokenize source code. Identifiers, keywords, numeric literals, and operators are all regular languages. Tools like //lex// and //flex// compile regular expressions into DFA-based scanners.

''Text search and pattern matching.'' The Unix tools ''grep'', ''sed'', and ''awk'' are built on regular expression engines. Modern regex implementations in Python, JavaScript, and Perl extend classical regular expressions with backreferences and lookaheads — features that actually push beyond regular languages into more complex territory, sometimes causing exponential worst-case behavior.

''Network protocols and hardware.'' Packet filters, protocol parsers, and network intrusion detection systems often use DFA-based pattern matching for wire-speed classification. Hardware implementations of DFAs in FPGAs and ASICs process inputs at one symbol per clock cycle.

''NLP and tokenization.'' Finite-state transducers (FSTs) — extensions of finite automata that produce output — are used in morphological analysis, phonological rule application, and text normalization. The OpenFst library provides weighted FST operations used in speech recognition pipelines.

''Model checking and verification.'' Temporal logic properties of finite-state systems are checked by constructing automata (often Buchi automata for infinite words) and testing language emptiness of intersections. This is how hardware and protocol designs are formally verified.

''Relation to ML.'' Recurrent neural networks with finite precision are theoretically equivalent to finite automata. Research on ''neural network extraction'' attempts to extract interpretable DFAs from trained RNNs. Understanding what patterns //cannot// be captured by regular languages (e.g., long-range dependencies, nested structures) clarifies why simple sequence models fail on certain tasks and why architectures with memory (LSTMs, Transformers with attention) are needed.

!! When Regular Languages Break Down

''1. No counting or matching.'' Regular languages cannot count. The language $\{a^n b^n \mid n \geq 1\}$ — strings with equal numbers of $a$'s followed by equal numbers of $b$'s — is not regular because a DFA has only finitely many states and cannot track an unbounded count. This is proved by the pumping lemma. Any task requiring //matching// of unbounded quantities — balanced parentheses, nested HTML tags, palindromes — lies beyond regular languages in the context-free or context-sensitive levels. This is why regex-based HTML parsers are fundamentally broken.

''2. Exponential blowup in determinization.'' While every NFA can be converted to a DFA, the subset construction can produce exponentially many states — an NFA with $n$ states may yield a DFA with $\Theta(2^n)$ states. This is not merely a theoretical curiosity: certain practical patterns (e.g., "match any string where the $k$th-from-last symbol is $a$") provably require exponentially many DFA states. In practice, this limits the scalability of DFA-based pattern matching when many complex patterns must be matched simultaneously, as in deep packet inspection.

''3. Extended regex features break regularity.'' Modern "regular expressions" in programming languages include features like backreferences ($\texttt{\textbackslash 1}$, $\texttt{\textbackslash 2}$) that match previously captured groups. The language $\{ww \mid w \in \{a,b\}^*\}$ — strings that are the concatenation of a word with itself — is not even context-free, yet backreferences can express it. These extended features make matching NP-hard in general and can cause catastrophic backtracking — exponential runtime on adversarial inputs (ReDoS attacks). The simple elegance of true regular languages, with their guaranteed linear-time matching, is lost.

!! Technical Details

''Closure properties:'' Regular languages are closed under union, intersection, complement, concatenation, Kleene star, reversal, homomorphism, inverse homomorphism, and quotient. This makes them the most well-behaved class in the Chomsky hierarchy. Context-free languages, by contrast, are //not// closed under intersection or complement.

''Decision problems (all decidable):''
- ''Membership:'' Given DFA $M$ and string $w$, is $w \in L(M)$? Solvable in $O(|w|)$ time by simulation.
- ''Emptiness:'' Is $L(M) = \emptyset$? Solvable in $O(|Q| + |\delta|)$ by reachability analysis.
- ''Equivalence:'' Given DFAs $M_1, M_2$, is $L(M_1) = L(M_2)$? Solvable in $O(n \log n)$ via Hopcroft's minimization algorithm.
- ''Universality:'' Is $L(M) = \Sigma^*$? Decidable via complement and emptiness check.
- ''Containment:'' Is $L(M_1) \subseteq L(M_2)$? Decidable via $L(M_1) \cap \overline{L(M_2)} = \emptyset$.

''Minimization:'' The Myhill-Nerode theorem guarantees a unique minimal DFA for every regular language. Hopcroft's algorithm (1971) computes it in $O(n \log n)$ time by iteratively refining state partitions.

''Star height:'' The ''star height'' of a regular expression is its maximum nesting depth of Kleene stars. Determining the minimum star height of a regular language is decidable but computationally expensive (Hashiguchi, 1988).

''Weighted and probabilistic automata:'' Generalizing DFAs to assign weights (from a semiring) to transitions yields weighted finite automata, which underpin hidden Markov models, CTC decoding in speech recognition, and the forward-backward algorithm.

''Key results:''
- Kleene (1956): Equivalence of DFA, NFA, and regex
- Myhill (1957), Nerode (1958): Characterization theorem and minimal DFA uniqueness
- Rabin and Scott (1959): Nondeterministic finite automata and subset construction (Turing Award, 1976)
- Hopcroft (1971): $O(n \log n)$ DFA minimization
- Pumping lemma: Folklore, formalized in Bar-Hillel, Perles, Shamir (1961)
