! Turing Machines
<img src="assets/turing_machines.svg" alt="Turing Machine diagram showing tape, read/write head, and state transitions">

Imagine a clerk sitting at an infinitely long desk covered with a strip of paper divided into squares. The clerk can read the symbol on the current square, erase it and write a new one, slide left or right to the next square, and consult a small instruction booklet that says what to do next based on the current symbol and the clerk's current "state of mind." That is, in essence, a ''Turing machine'' — the most influential abstract model of computation ever devised, introduced by Alan Turing in 1936 to formalize what it means for a problem to be //computable//.

Despite its almost comical simplicity — just a tape, a head, and a finite set of rules — a Turing machine can simulate any algorithm that any modern computer can execute. It is the theoretical bedrock on which computer science, programming language theory, and even parts of modern machine learning are built.

!! The Core Insight

A Turing machine shows that ''computation is mechanical symbol manipulation''. You do not need arithmetic circuits, floating-point units, or gigabytes of RAM. All you need is:

* An ''infinite tape'' divided into cells, each holding a symbol from a finite alphabet (including a blank symbol $B$).
* A ''read/write head'' that points at one cell at a time.
* A ''finite state control'' — a finite set of states $Q$ with a designated start state $q_0$, plus accept and reject states.
* A ''transition function'' $\delta: Q \times \Gamma \to Q \times \Gamma \times \{L, R\}$ that, given the current state and the symbol under the head, specifies the new state, the symbol to write, and the direction to move.

At each step the machine reads, writes, moves, and changes state. It halts when it enters the accept or reject state (or it may loop forever — which is itself a deep issue; see [[Halting Problem|Halting Problem]]).

!! Tape, Head, and the Dance of States

A computation proceeds as a sequence of ''configurations''. A configuration is a snapshot: the tape contents, the head position, and the current state. Formally a configuration is written $u\, q\, v$ where $u$ is the tape to the left of the head, $q$ is the current state, and $v$ is the tape from the head onward.

The machine starts in configuration $q_0\, w$ where $w$ is the input string. It then follows $\delta$ step by step. If it reaches $q_{\text{accept}}$, we say the machine ''accepts'' $w$. If it reaches $q_{\text{reject}}$, it ''rejects''. If it never halts, we say the machine ''loops'' on $w$.

The set of all strings a TM accepts is called its ''language'':

$$L(M) = \{w \in \Sigma^* \mid M \text{ accepts } w\}$$

This simple framework lets us precisely define what problems are solvable by //any// algorithmic process.

!! What the Math Is Actually Doing

A Turing machine $M$ is formally a 7-tuple:

$$M = (Q,\; \Sigma,\; \Gamma,\; \delta,\; q_0,\; q_{\text{accept}},\; q_{\text{reject}})$$

where $Q$ is the finite set of states, $\Sigma$ is the input alphabet (not containing blank $B$), $\Gamma \supseteq \Sigma \cup \{B\}$ is the tape alphabet, and $\delta$ is the transition function.

The ''Church–Turing thesis'' asserts that this model captures //everything// that is effectively computable. Alonzo Church's [[Lambda Calculus|Lambda Calculus]], Emil Post's production systems, and Turing's machines all define the same class of computable functions — a remarkable convergence that gives the thesis its force.

A ''universal Turing machine'' $U$ takes as input the encoding $\langle M \rangle$ of any TM $M$ and an input $w$, and simulates $M$ on $w$:

$$U(\langle M \rangle, w) = M(w)$$

This is the theoretical ancestor of the stored-program computer. The existence of $U$ means a single fixed machine can run //any// program — the idea that makes general-purpose computers possible.

!! Decidability and Recognizability

Not all problems are created equal:

* A language is ''decidable'' (recursive) if some TM always halts and correctly accepts or rejects every input.
* A language is ''recognizable'' (recursively enumerable) if some TM accepts every string in the language — but may loop on strings outside it.
* A language is ''co-recognizable'' if its complement is recognizable.

A language is decidable if and only if it is //both// recognizable and co-recognizable. The [[Halting Problem|Halting Problem]] is the canonical example of a language that is recognizable but not decidable.

These distinctions connect directly to [[Complexity Classes|Complexity Classes]], where we further constrain how //efficiently// a TM can decide a language (e.g., polynomial time vs. exponential time).

!! Variants and Equivalences

Several TM variants look more powerful but turn out to compute exactly the same class of functions:

* ''Multi-tape TMs'' — $k$ tapes with independent heads. Can be simulated by a single-tape TM with at most a quadratic slowdown: if the multi-tape machine runs in $T(n)$ steps, the single-tape simulation runs in $O(T(n)^2)$.
* ''Nondeterministic TMs (NTMs)'' — $\delta$ maps to a //set// of possible transitions. The machine accepts if //any// branch accepts. Every NTM can be simulated by a deterministic TM, though potentially with an exponential blowup. Whether this blowup is necessary is the $P$ vs. $NP$ question.
* ''Enumerators'' — TMs that print strings. A language is recognizable iff some enumerator lists all its members.

These equivalences reinforce the Church–Turing thesis: the details of the model do not matter; the computational power is the same.

The [[Finite State Machine|Finite State Machine]] is what you get when you remove the tape entirely — a strictly weaker model that can only recognize regular languages. [[Automata Theory|Automata Theory]] studies the full hierarchy from finite automata up through Turing machines.

!! Connection to Machine Learning

Turing machines are not just a historical curiosity for ML practitioners:

* ''Neural Turing Machines (NTMs)'' — Graves et al. (2014) augmented neural networks with an external differentiable memory (analogous to the tape) and attention-based read/write heads. The network learns //programs// by gradient descent, enabling tasks like copying, sorting, and priority-queue manipulation that standard RNNs struggle with.
* ''Computational universality of RNNs'' — Siegelmann and Sontag (1995) proved that recurrent neural networks with rational weights are Turing-complete. In theory, an RNN can compute anything a TM can, given sufficient precision and unbounded time. In practice, finite precision and vanishing gradients limit this power, motivating architectures like LSTMs and Transformers.
* ''Kolmogorov complexity and learning'' — the length of the shortest TM that outputs a dataset is a measure of its intrinsic complexity. While uncomputable in general, approximations (e.g., Minimum Description Length) are used in model selection and regularization.

!! When Turing Machines Break Down

# ''The uncomputability wall'' — some problems (the [[Halting Problem|Halting Problem]], Rice's theorem) are //provably// unsolvable by any TM. No amount of hardware improvement changes this; these are fundamental limits on algorithmic reasoning.
# ''Ignoring resource constraints'' — a TM can use unlimited time and tape, but real machines have finite memory and time budgets. Complexity theory patches this by measuring time and space usage, but raw TM theory says nothing about efficiency.
# ''Poor model of physical computation'' — TMs assume perfect reliability, discrete steps, and infinite tape. Quantum computation, analog computation, and biological neural networks all challenge different assumptions of the model, though none have been proven to exceed its computational power (per the extended Church–Turing thesis).

!! Technical Details

A TM $M = (Q, \Sigma, \Gamma, \delta, q_0, q_{\text{accept}}, q_{\text{reject}})$ processes input $w = w_1 w_2 \ldots w_n \in \Sigma^*$. The initial configuration is $q_0\, w_1 w_2 \ldots w_n$. At each step, if the machine is in state $q$ reading symbol $a$, and $\delta(q, a) = (r, b, D)$, it writes $b$, transitions to state $r$, and moves the head in direction $D \in \{L, R\}$.

The ''running time'' of $M$ on input $w$ is the number of steps before halting (if it halts). The ''time complexity'' of $M$ is $T(n) = \max\{t \mid M \text{ halts on } w \text{ in } t \text{ steps, } |w| = n\}$.

The ''space complexity'' is $S(n)$, the maximum number of tape cells visited on any input of length $n$.

Key theorems:
* ''Time hierarchy theorem'': for constructible $f$, $\text{DTIME}(f(n)) \subsetneq \text{DTIME}(f(n)^2)$ — more time strictly buys more power.
* ''Space hierarchy theorem'': analogous result for space.
* ''Simulation theorem'': a $k$-tape TM running in time $T(n)$ can be simulated by a 1-tape TM in $O(T(n)^2)$.
* ''Nondeterministic simulation'': an NTM running in time $T(n)$ can be simulated deterministically in $2^{O(T(n))}$.
