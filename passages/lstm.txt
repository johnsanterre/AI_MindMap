! Long Short-Term Memory

[img[assets/lstm.svg]]

Imagine you are reading a novel. A standard neural network trying to process the story is like a reader with a five-second memory — by the time they reach chapter ten, they have forgotten chapter one entirely. A ''Long Short-Term Memory'' (LSTM) network is like a reader who keeps a notebook beside them. At every page, they decide what new facts to jot down, which old notes to cross out, and which notes are relevant to understanding the current sentence. The notebook is the ''cell state'' — a dedicated highway that can carry critical information across hundreds or thousands of time steps without it degrading. The decisions about what to write, erase, and consult are made by learned ''gates'', and the genius of the design is that the network learns //which// information matters, all on its own.

---

!! The Core Insight

The fundamental problem with standard [[RNN|RNN]]s is the ''vanishing gradient'': when you backpropagate through many time steps, the gradient signal gets multiplied by the same weight matrix over and over, shrinking exponentially until it effectively disappears. The network cannot learn long-range dependencies because the error signal from the future never reaches the distant past. This is explored in depth in [[Vanishing and Exploding Gradients|Vanishing and Exploding Gradients]].

LSTMs solve this with a deceptively simple architectural trick: they add a ''cell state'' — a separate path that runs through the entire sequence with only //additive// interactions. Because addition distributes gradients equally (the derivative of a sum is 1 for each term), the gradient can flow backward through hundreds of steps without vanishing. The cell state acts as a conveyor belt or highway: information placed on it at time step 5 can arrive at time step 500 essentially unchanged, unless a gate explicitly decides to modify it.

This is what makes LSTMs fundamentally different from vanilla RNNs. The hidden state in a plain RNN is overwritten at every step via a squashing nonlinearity (tanh), which compresses all information into a fixed-size vector with no mechanism for selective preservation. The LSTM separates ''what to remember long-term'' (cell state) from ''what to output right now'' (hidden state), and uses gates to mediate between them.

---

!! Gates: Forget, Input, and Output

An LSTM cell has three gates, each a sigmoid layer that outputs values between 0 and 1. Think of each gate as a dial: 0 means "block everything," 1 means "let everything through."

The ''forget gate'' $f_t$ looks at the current input $x_t$ and the previous hidden state $h_{t-1}$ and decides what to erase from the cell state. If the network is processing language and encounters the end of a paragraph, the forget gate might learn to clear out subject-verb information that is no longer relevant.

The ''input gate'' $i_t$ decides what new information to write onto the cell state. It works in tandem with a ''candidate vector'' $\tilde{c}_t$ (produced by a tanh layer) that proposes new values. The input gate scales these candidates: if a piece of information is important (gate near 1), it gets added; if irrelevant (gate near 0), it is ignored.

The ''output gate'' $o_t$ decides what part of the cell state to expose as the hidden state $h_t$. The cell state might contain far more information than is needed for the current prediction. The output gate filters it, producing a hidden state that is tailored to the immediate task — predicting the next word, classifying sentiment, or generating a musical note.

The interplay of these three gates is what gives LSTMs their remarkable flexibility. The network can learn to hold information for exactly as long as needed and release it precisely when it becomes useful.

---

!! What the Math Is Actually Doing

At each time step $t$, the LSTM receives the input $x_t$ and the previous hidden state $h_{t-1}$. The concatenation $[h_{t-1}, x_t]$ is fed into four parallel linear transformations (one per gate plus the candidate), each with its own weight matrix and bias:

$$f_t = \sigma(W_f [h_{t-1}, x_t] + b_f)$$

$$i_t = \sigma(W_i [h_{t-1}, x_t] + b_i)$$

$$\tilde{c}_t = \tanh(W_c [h_{t-1}, x_t] + b_c)$$

$$o_t = \sigma(W_o [h_{t-1}, x_t] + b_o)$$

The cell state is then updated by forgetting old content and adding new content:

$$c_t = f_t \odot c_{t-1} + i_t \odot \tilde{c}_t$$

Here $\odot$ denotes element-wise (Hadamard) multiplication. The forget gate $f_t$ scales down (or zeroes out) entries of the old cell state $c_{t-1}$, while the input gate $i_t$ scales the candidate $\tilde{c}_t$ before adding it in.

Finally, the hidden state is produced by passing the updated cell state through tanh and filtering with the output gate:

$$h_t = o_t \odot \tanh(c_t)$$

The critical gradient property: when backpropagating through $c_t = f_t \odot c_{t-1} + \ldots$, the gradient with respect to $c_{t-1}$ is simply $f_t$ — a learned diagonal matrix with entries in $(0,1)$. If the forget gate stays close to 1, the gradient flows backward through time with minimal decay. This is the mathematical mechanism that defeats the vanishing gradient problem.

The total parameter count for an LSTM layer with input dimension $d$ and hidden dimension $h$ is $4h(h + d) + 4h$ — four weight matrices of size $h \times (h+d)$ and four bias vectors of size $h$.

---

!! LSTMs in Practice

LSTMs dominated [[Sequence Modeling|Sequence Modeling]] from their introduction by Hochreiter and Schmidhuber in 1997 through roughly 2017, when [[Transformers|Transformers]] began to supplant them for many tasks.

''Language modeling and machine translation'' were the flagship applications. Before the Transformer, the state-of-the-art in neural machine translation was the encoder-decoder LSTM architecture, often augmented with attention. Google Translate switched from phrase-based statistical MT to a deep LSTM system in 2016, immediately improving translation quality across 100+ language pairs.

''Speech recognition'' systems at Google, Apple (Siri), and Amazon (Alexa) used stacked bidirectional LSTMs as their core acoustic models. The ability to capture long-range dependencies in audio spectrograms — where a sound's identity can depend on context hundreds of milliseconds away — made LSTMs far superior to previous HMM-based approaches.

''Time series forecasting'' remains a domain where LSTMs are competitive. Financial data, sensor readings, weather prediction, and medical monitoring all involve sequences where patterns at multiple time scales interact. LSTMs can simultaneously track short-term fluctuations (via the hidden state) and long-term trends (via the cell state).

The ''[[GRU|GRU]]'' (Gated Recurrent Unit) is a simplified variant that merges the forget and input gates into a single "update gate" and combines the cell state and hidden state. GRUs have fewer parameters and often perform comparably to LSTMs on shorter sequences, though LSTMs tend to have an edge on tasks requiring very long memory.

''Stacking'' multiple LSTM layers (deep LSTMs) and using ''bidirectional'' processing (one LSTM reads forward, another reads backward) were standard techniques that significantly improved performance. The hidden state of one layer becomes the input sequence for the next, allowing hierarchical feature extraction.

---

!! When LSTMs Break Down

''Very long sequences (thousands of steps):'' Despite their superior gradient flow compared to vanilla RNNs, LSTMs still process sequences //sequentially// — each time step depends on the previous one. For sequences beyond a few thousand steps, training becomes extremely slow because parallelism across time is impossible. Transformers, which process all positions in parallel via self-attention, are dramatically faster to train on long sequences. Additionally, even the cell state highway eventually degrades: with $T$ time steps, the gradient passes through $T$ multiplicative forget-gate factors, and if these are even slightly below 1, the effective memory still decays exponentially — just more slowly than in a vanilla RNN.

''Capturing global structure:'' LSTMs build representations by sweeping left-to-right (or right-to-left). They struggle with tasks where the relevant context is not sequential but //structural// — for instance, understanding that a pronoun in sentence 15 refers to a noun in sentence 2 requires the LSTM to have carried that information through 13 sentences of forget-gate filters. Attention mechanisms (and ultimately Transformers) solve this by allowing direct connections between any two positions, regardless of distance.

''High-dimensional inputs with spatial structure:'' LSTMs treat their input as a flat vector at each time step. For data with inherent spatial structure — images, video frames, 3D point clouds — this flattening discards critical information. Convolutional LSTMs (ConvLSTMs) partially address this for grid-structured data, but the fundamental sequential bottleneck remains. Modern architectures like Vision Transformers handle spatiotemporal data more naturally.

---

!! Technical Details

''Original paper:'' Hochreiter & Schmidhuber (1997), "Long Short-Term Memory," //Neural Computation// 9(8). This paper introduced the LSTM architecture with input and output gates. The forget gate was added later by Gers, Schmidhuber & Cummins (2000) and is now considered standard.

''Peephole connections:'' A variant where the gate layers receive the cell state $c_{t-1}$ as an additional input (in addition to $h_{t-1}$ and $x_t$). This allows gates to inspect the current cell state when making gating decisions. The equations become $f_t = \sigma(W_f [h_{t-1}, x_t, c_{t-1}] + b_f)$, etc. Peepholes improve performance on precise timing tasks but add parameters and complexity.

''Forget gate bias initialization:'' A critical practical detail. Jozefowicz et al. (2015) showed that initializing the forget gate bias $b_f$ to 1 or 2 (so the gate starts near 1, meaning "remember everything by default") significantly improves training stability and final performance. Without this, LSTMs can start by forgetting aggressively, losing information before the network has learned what to keep.

''Gradient clipping:'' Even with the LSTM's improved gradient flow, the //exploding// gradient problem can still occur (the forget gate mitigates vanishing, not exploding). Standard practice clips gradients to a maximum norm (typically 1.0 or 5.0) during training.

''Parameter count comparison:'' For hidden size $h = 512$ and input size $d = 300$: a single LSTM layer has $4 \times 512 \times (512 + 300) + 4 \times 512 = 1{,}663{,}488$ parameters. A [[GRU|GRU]] with the same dimensions has $3 \times 512 \times (512 + 300) + 3 \times 512 = 1{,}247{,}616$ — roughly 25% fewer.

''Bidirectional LSTMs:'' Two LSTMs process the sequence in opposite directions; their hidden states are concatenated at each time step, producing a representation that has context from both the past and the future. This doubles the parameter count and the hidden state dimension.

''CuDNN optimization:'' NVIDIA's cuDNN library provides a fused LSTM kernel that computes all four gate activations in a single matrix multiplication, achieving 5-10x speedup over naive implementations. This makes the LSTM the fastest recurrent architecture on GPU hardware.
